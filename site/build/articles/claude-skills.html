Claude Skills 마스터 학습 경로

초보자에서 마스터까지: 이 경로를 따라가면 Claude Skill을 완벽하게 만들 수 있습니다.

🎯 3가지 가이드와의 관계 {#3가지-가이드와의-관계}

Plain Text 
당신의 수준                    추천 가이드                        학습 시간
─────────────────────────────────────────────────────────────────────
❌ "Skill이 뭐죠?" →  Claude_Skills_초보자_가이드 →  30분
                   (5살 버전부터 상세 설명)


❓ "어떻게 만들어?" →  clipping-processor_실전_분석 →  45분
                    (실제 예제로 상세 분석)


✅ "지금 만들고 싶어" → 나의_첫_Skill_만드는_템플릿 →  1-2시간
                    (바로 쓸 수 있는 템플릿)
8 lines

📚 학습 단계별 가이드 {#학습-단계별-가이드}

🟢 Level 1: 기초 이해 (Day 1) {#level-1-기초-이해-day-1}

1-1. 개념 이해하기 (30분)

읽을 문서: Claude Skills가 뭔가요 (5살 버전)

배울 내용:

  • ✅ Skill의 정의
  • ✅ Skill과 일반 명령어의 차이
  • ✅ Skill이 필요한 이유
  • ✅ Skill의 기본 가치

확인 질문:

  • "Skill이 뭔가?"에 5살 아이도 이해할 수 있게 설명 가능한가?
  • "왜 Skill이 필요한가?"를 말할 수 있는가?

1-2. 실제 사례 보기 (30분)

읽을 문서: 스토리: 내가 Claude Skill을 만든 이유

배울 내용:

  • ✅ Skill을 만들어야 하는 상황
  • ✅ Before/After 비교
  • ✅ 시간 절감 효과

확인 질문:

  • 자신의 반복적인 일 중에 Skill로 자동화할 수 있는 게 있나?

1-3. 구조 이해하기 (20분)

읽을 문서: Skill의 구조 한눈에 보기

배울 내용:

  • ✅ SKILL.md의 역할
  • ✅ JSON 파일의 역할
  • ✅ 마크다운 가이드 파일의 역할

과제: Skill의 4가지 파일을 손가락으로 가리키면서 "이건 뭘 하는 파일인가?" 말하기

🟡 Level 2: 실제 분석 (Day 2) {#level-2-실제-분석-day-2}

2-1. clipping-processor Skill 분석 (60분)

읽을 문서: 이 Skill이 해결하는 문제 ~ 처리 과정: 실제 예제로 따라하기

배울 내용:

  • ✅ 실제 Skill이 뭘 하는지
  • ✅ 파일들이 어떻게 연결되는지
  • ✅ 처리 과정이 어떻게 작동하는지
  • ✅ 🆕 YouTube 영상 처리 기능 (Skill의 새로운 고도화 버전)

실습:

  1. clipping-processor의 SKILL.md 열어보기
  2. 가이드의 설명과 비교하면서 읽기
  3. "왜 이렇게 설계했을까?"를 생각하기
  4. 🆕 YouTube 영상 처리 워크플로우 섹션 분석하기
    • MCP YouTube Server와 yt-dlp의 차이 이해
    • 타임스탐프 기반 섹션 분할 로직 이해
    • 필러 단어 제거 및 톤 변환 규칙 학습

2-2. 설계 결정 이해하기 (30분)

읽을 문서: 왜 이렇게 설계했을까

배울 내용:

  • ✅ 파일을 4가지로 분리한 이유
  • ✅ JSON을 사용한 이유
  • ✅ 3단계 예제의 중요성
  • ✅ Feynman 기법의 가치

과제: 각 설계 결정의 장단점을 표로 정리하기

2-3. 패턴 인식하기 (30분)

읽을 문서: 나만의 Skill을 만들 때 활용하기

배울 내용:

  • ✅ 입력→처리→출력 패턴
  • ✅ 분류 규칙 정의 패턴
  • ✅ 품질 점수 시스템 패턴

과제: 자신의 미래 Skill이 따를 패턴 선택하기

🔴 Level 3: 나의 Skill 만들기 (Day 3-4) {#level-3-나의-skill-만들기-day-3-4}

3-1. 구상 단계 (30분)

준비:

  1. 자신이 반복적으로 하는 일 3가지 적어보기
    • 예: 파일 정렬, 텍스트 정리, 메타데이터 생성
  2. 각 일에 대해 다음 질문에 답하기:
    • 입력: 뭘 받을 건가?
    • 처리: 어떻게 변환할 건가?
    • 출력: 어디에 저장할 건가?
    • 분류: 여러 종류가 있나?
  3. 가장 간단한 것 1가지 선택

3-2. 파일 준비 (30분)

읽을 문서: 사전 준비

실행:

Bash 
mkdir -p ~/.claude/skills/your-skill-name
cd ~/.claude/skills/your-skill-name
touch SKILL.md
touch category_mapping.json
touch enhancement_rules.md
5 lines

3-3. SKILL.md 작성 (1시간)

읽을 문서: SKILL.md 템플릿

단계별 작성:

  1. 개요: 한 문장으로 Skill 설명
  2. 핵심 기능: 3가지 이상의 기능 나열
  3. 사용 방법: 구체적인 예제 3개 이상
  4. 처리 과정: Step by Step 설명
  5. 메타데이터: YAML 샘플 제공

확인:

  • 개요를 읽은 사람이 이해하나?
  • 사용 예제를 따라할 수 있나?
  • 처리 과정이 명확한가?

3-4. category_mapping.json 작성 (45분)

읽을 문서: JSON 템플릿

단계별 작성:

  1. 주요 키워드 3~5가지 생각하기
  2. 각 키워드별 폴더 결정
  3. 태그 정의
  4. priority 순서 정하기

검증:

Plain Text 
jsonlint.com에서 올바른 형식인지 확인
1 line

3-5. enhancement_rules.md 작성 (45분)

읽을 문서: Enhancement 템플릿

단계별 작성:

  1. Feynman 기법 규칙 정의
  2. Storytelling 규칙 정의
  3. 3단계 예제 만들기
  4. 품질 체크리스트 작성

3-6. 테스트 및 개선 (1시간)

테스트:

Plain Text 
직접 명령어로 Skill 사용해보기:
"[your-skill-name] skill으로 [입력]을 처리해줘"
2 lines

확인:

  • Skill이 제대로 작동하나?
  • 예상한 대로 처리되나?
  • 메타데이터가 생성되나?
  • 파일이 올바른 위치에 저장되나?

개선:

  • 안 되는 부분 찾아서 수정
  • 예제가 실제로 작동하는지 확인
  • 설명이 명확한지 재검토

📊 시간표 요약 {#시간표-요약}

날짜 학습 내용 시간 활동
Day 1 Claude Skills 개념 이해 1.5시간 읽기
Day 2 clipping-processor 분석 (+ YouTube 기능) 2시간 읽기 + 분석
Day 3 첫 번째 Skill 기획 및 작성 3시간 구상 + 파일 작성
Day 4 테스트 및 개선 1시간 실행 + 디버깅
선택 YouTube 기능 심화 학습 1~2시간 분석 + 설계
합계 7.5-8.5시간

🎓 각 단계별 체크리스트 {#각-단계별-체크리스트}

✅ Level 1 완료 확인 {#level-1-완료-확인}

  • Skill이 뭔지 설명할 수 있나?
  • Skill과 일반 명령어의 차이를 알나?
  • Skill의 4가지 구성 요소를 말할 수 있나?
  • 자신의 반복 업무 중 Skill로 자동화할 수 있는 것을 찾았나?

✅ Level 2 완료 확인 {#level-2-완료-확인}

  • clipping-processor가 뭘 하는지 설명 가능한가?
  • SKILL.md가 왜 필요한지 알나?
  • JSON 파일의 역할을 이해했나?
  • 처리 과정을 그림으로 그릴 수 있나?

✅ Level 3 완료 확인 {#level-3-완료-확인}

  • 첫 번째 Skill 폴더를 만들었나?
  • SKILL.md, JSON, 마크다운 파일을 모두 작성했나?
  • 직접 만든 Skill이 작동하나?
  • 피드백을 반영해서 개선했나?

📞 막혔을 때 확인사항 {#막혔을-때-확인사항}
