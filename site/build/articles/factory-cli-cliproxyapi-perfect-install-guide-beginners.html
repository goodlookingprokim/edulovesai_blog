이야기로 시작하기 - 넷플릭스와 극장의 차이 {#이야기로-시작하기-넷플릭스와-극장의-차이}

영화를 보는 두 가지 방법을 생각해보세요 🎬

  1. 극장 방식 (API 키): 영화 한 편 볼 때마다 티켓을 구매 (토큰당 과금)
  2. 넷플릭스 방식 (Claude Pro 구독): 월정액으로 무제한 시청

그런데 어떤 특별한 영화 편집 프로그램(Factory CLI)은 극장 티켓만 인식할 수 있습니다. 여기서 CLIProxyAPI가 등장합니다! 이것은 넷플릭스 구독을 극장 티켓처럼 보이게 만드는 마법의 변환기입니다.

실제 비용 차이 (2025년 기준) {#실제-비용-차이-2025년-기준}

  • 🎫 API 방식: 월 수십만원~수백만원 (사용량에 따라)
  • 📺 구독 방식: Claude Pro 월 $20 (약 3만원)
  • 💰 절약 효과: 월 90% 이상 비용 절감!

파인만 기법으로 이해하기 - 5살 아이도 이해할 수 있게 {#파인만-기법으로-이해하기-5살-아이도-이해할-수-있게}

"5살 아이에게 설명한다면..." {#5살-아이에게-설명한다면}

"너희 집 TV 리모컨 알지? 그런데 이 리모컨(Factory CLI)은 특별한 건전지(API 키)만 사용할 수 있어. 우리는 충전식 건전지(Claude Pro 구독)를 가지고 있는데, 이걸 사용하려면 특별한 어댑터(CLIProxyAPI)가 필요한 거야!"

왜 이런 복잡한 과정이 필요한가? {#왜-이런-복잡한-과정이-필요한가}

  1. Factory CLI는 원래 기업용으로 만들어져서 API 키만 인식
  2. Claude Pro는 개인 구독이라 브라우저 로그인 방식 사용
  3. CLIProxyAPI가 이 둘을 연결해주는 통역사 역할

실제 설치 과정 - 2025년 10월 11일 기준 {#실제-설치-과정-2025년-10월-11일-기준}

📦 사전 준비물 체크리스트 {#사전-준비물-체크리스트}

  • ✅ macOS (Monterey 이상) 또는 Linux/Windows WSL2
  • ✅ Claude Pro 또는 ChatGPT Plus 구독
  • ✅ 터미널 기본 사용법 이해
  • ✅ 약 500MB 여유 공간

Bash

출력 예시:

Go가 없다면 설치

💡 왜 Go가 필요한가?

CLIProxyAPI가 Go로 작성되어 있어서 컴파일하려면 Go가 필요합니다. 마치 한국 요리를 하려면 한국 조리도구가 필요한 것과 같습니다.

logging-to-file: false

proxy-url: ""
request-retry: 3
quota-exceeded:
  switch-project: true
  switch-preview-model: true
auth:
  providers: []
generative-language-api-key: []
EOF


echo "✅ 설정 파일 생성 완료!"

🌳 고급 예제: 완전 자동화 (프로 레벨) {#고급-예제-완전-자동화-프로-레벨}

Python 통합 스크립트

Python 
#!/usr/bin/env python3
# claude_automation.py


import subprocess
import json
import os
from datetime import datetime


class ClaudeAutomation:
    def init(self, model="claude-sonnet-4-5-20250929"):
        self.model = model
        self.check_proxy()


def check_proxy(self):
    &quot;&quot;&quot;CLIProxyAPI 실행 확인&quot;&quot;&quot;
    result = subprocess.run(
        &quot;ps aux | grep cli-proxy-api | grep -v grep&quot;,
        shell=True, capture_output=True
    )
    if not result.stdout:
        print(&quot;⚠️ CLIProxyAPI가 실행되지 않음. 시작 중...&quot;)
        self.start_proxy()

def start_proxy(self):
    &quot;&quot;&quot;프록시 자동 시작&quot;&quot;&quot;
    os.system(&quot;cd ~/CLIProxyAPI &amp;&amp; ./cli-proxy-api --config config.yaml &amp;&quot;)
    print(&quot;✅ CLIProxyAPI 시작됨&quot;)

def ask(self, prompt, context=&quot;&quot;):
    &quot;&quot;&quot;Claude에게 질문&quot;&quot;&quot;
    full_prompt = f&quot;{context}\n{prompt}&quot; if context else prompt
    cmd = f&#39;echo &quot;{full_prompt}&quot; | droid --model {self.model}&#39;

    result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
    return result.stdout

def daily_standup(self):
    &quot;&quot;&quot;일일 스탠드업 자동 생성&quot;&quot;&quot;
    yesterday_log = self.get_git_commits(1)
    today_plan = self.ask(
        f&quot;어제 작업: {yesterday_log}\n&quot;
        &quot;오늘 할 일을 3개 제안해줘&quot;
    )

    standup = f&quot;&quot;&quot;
    # Daily Standup - {datetime.now().strftime(&#39;%Y-%m-%d&#39;)}

    ## 어제 완료한 작업
    {yesterday_log}

    ## 오늘 계획
    {today_plan}

    ## 이슈/블로커
    - 없음
    &quot;&quot;&quot;

    with open(f&quot;standup_{datetime.now().strftime(&#39;%Y%m%d&#39;)}.md&quot;, &quot;w&quot;) as f:
        f.write(standup)

    return standup

def get_git_commits(self, days=1):
    &quot;&quot;&quot;최근 커밋 가져오기&quot;&quot;&quot;
    cmd = f&quot;git log --since=&#39;{days} days ago&#39; --oneline&quot;
    result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
    return result.stdout
