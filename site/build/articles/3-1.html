개요 {#개요}

  • 라이브 제목: 바이브코딩 라이브 | 커서맛피아가 대신 만들어드림 #3
  • 방송 시간: 약 3시간 (10,992초)
  • 주요 주제: Cursor AI를 활용한 실시간 SaaS 개발
  • 개발 프로젝트: WorkEasy - 알바생 스케줄 관리 서비스

주요 내용 {#주요-내용}

라이브 코딩 목표 {#라이브-코딩-목표}

  • SaaS 서비스의 핵심 기능을 실제로 구현
  • 기능이 "말이 되게" 완성하는 것에 중점
  • Cursor AI를 활용한 효율적인 개발 프로세스 시연

시청자 제보 서비스 {#시청자-제보-서비스}

  • 서비스명: WorkEasy (워크이지)
  • 유사 서비스: 워크잼(WorkJam)
  • 주요 기능:
    • 알바생 스케줄 자동 조정
    • 사내 채팅 기능
    • 근무 교대 시스템

WorkEasy 프로젝트 소개 {#workeasy-프로젝트-소개}

서비스 개요 {#서비스-개요}

  • 목적: 알바생들의 효율적인 근무 스케줄 관리
  • 타겟: 알바생이 많은 매장 (카페, 레스토랑, 편의점 등)
  • 핵심 가치: 근무 교대의 효율화

주요 기능 명세 {#주요-기능-명세}

  1. 스케줄 관리

    • 알바생별 근무 스케줄 조회
    • 매니저의 통합 스케줄 관리

  2. 스케줄 변경 및 교환

    • 알바생 간 근무 교대 요청
    • 자동 매칭 시스템

  3. 계정 체계

    • 매니저 계정: 스케줄 생성 및 관리 권한
    • 알바생 계정: 스케줄 조회 및 교대 요청 권한

  4. 부가 기능

    • 교육 지침 자료 공유
    • 사내 공지사항
    • PWA 설치 시 푸시 알림

사용자 플로우 설계 {#사용자-플로우-설계}

Plain Text 
매니저 가입 → 매장 생성 → 알바생 계정 발급 → 
알바생 초대 → 스케줄 등록 → 근무 교대 요청/승인
2 lines

EasyNext 프레임워크 활용 {#easynext-프레임워크-활용}

EasyNext란? {#easynext란}

  • 방송 진행자가 개발한 Next.js 기반 프레임워크
  • 빠른 프로젝트 초기 설정 지원
  • SaaS 개발에 최적화된 보일러플레이트

설치 및 설정 {#설치-및-설정}

1. 사전 요구사항

Bash 
# Node.js 버전 확인 (v20 이상 필요)
node -v
2 lines

2. EasyNext 설치

Bash 
# EasyNext CLI 설치
npm install -g easynext


또는 npx 사용


npx create-easynext@latest
4 lines

3. 프로젝트 생성

Bash 
# 프로젝트 생성 명령어
easynext create workeasy


폴더 이동


cd workeasy
4 lines

4. 프로젝트 실행

Bash 
# 개발 서버 실행
npm run dev
2 lines

구현 방법 {#구현-방법}

Cursor 활용 팁 {#cursor-활용-팁}

1. 프로젝트 열기

  • Cursor 실행 → Open Project → workeasy 폴더 선택

2. 주요 패널 소개

  • 파일 탐색기: 좌측 패널에서 프로젝트 파일 관리
  • AI 채팅 패널: 우측에서 AI에게 기능 요청
  • 터미널: Ctrl+J (Windows) 또는 Cmd+J (Mac)로 열기

3. AI 활용 예시

Plain Text 
"로그인 기능 만들어 줘"
"알바생 스케줄 관리 페이지 구현해 줘"
"근무 교대 요청 시스템 만들어 줘"
3 lines

개발 환경 구성 {#개발-환경-구성}

필수 도구

  • Node.js (v20 이상)
  • Cursor AI Editor
  • Git (버전 관리용)

터미널 명령어

Bash 
# 프로젝트 실행
npm run dev


의존성 설치


npm install


빌드


npm run build
6 lines

핵심 학습 포인트 {#핵심-학습-포인트}

1. AI 기반 개발 프로세스 {#1-ai-기반-개발-프로세스}

  • 자연어로 기능 요청
  • AI가 생성한 코드 검토 및 수정
  • 반복적인 개선 과정

2. SaaS 아키텍처 설계 {#2-saas-아키텍처-설계}

  • 다중 테넌트 구조 이해
  • 권한 기반 접근 제어
  • 실시간 데이터 동기화

3. 실무 개발 팁 {#3-실무-개발-팁}

  • 매니저 우선 가입 플로우의 중요성
  • 현실적인 사용 시나리오 고려
  • MVP(최소 기능 제품) 접근법

실무 적용점 {#실무-적용점}

프로젝트 관리 {#프로젝트-관리}

  1. 단계별 개발

    • 핵심 기능부터 구현
    • 점진적 기능 확장
    • 사용자 피드백 반영

  2. 효율적인 개발

    • AI 도구 적극 활용
    • 보일러플레이트 활용
    • 빠른 프로토타이핑

기술 스택 선택 {#기술-스택-선택}

  • 프론트엔드: Next.js + TypeScript
  • 스타일링: Tailwind CSS
  • 상태관리: React Context 또는 Zustand
  • 인증: NextAuth.js

확장 가능성 {#확장-가능성}

  • 급여 계산 기능
  • 출퇴근 기록 관리

  • 다중 매장 관리

구현 체크리스트 {#구현-체크리스트}

  • 프로젝트 초기 설정 (EasyNext)
  • 인증 시스템 구현
  • 매니저/알바생 계정 분리
  • 스케줄 CRUD 기능
  • 근무 교대 요청 시스템
  • 실시간 알림 기능
  • PWA 설정
  • 배포 환경 구성

연결된 노트 {#연결된-노트}

  • Cursor AI 활용 가이드
  • Next.js SaaS 개발 패턴
