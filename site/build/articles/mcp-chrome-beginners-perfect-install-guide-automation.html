MCP Chrome 초보자 완벽 설치 가이드 - 스토리텔링으로 배우는 브라우저 자동화

📌 중요 안내: 이 가이드는 Claude Code를 사용하여 MCP Chrome을 설치하고 활용하는 것을 기준으로 작성되었습니다. Claude Code는 터미널 기반 AI 어시스턴트로, 명령어 하나로 복잡한 설치 과정을 자동화할 수 있습니다. 일반 터미널 사용자도 따라할 수 있지만, Claude Code 사용자에게 최적화되어 있습니다.

🎯 개요 {#개요}

이야기의 시작: "왜 MCP Chrome이 필요할까?" {#이야기의-시작-왜-mcp-chrome이-필요할까}

상상해보세요. 여러분이 Claude Code에게 이렇게 말할 수 있다면 어떨까요?

"이 웹사이트에서 상품 정보를 수집해줘"
"폼을 자동으로 작성하고 제출해줘"
"웹페이지 스크린샷을 찍어서 분석해줘"

MCP Chrome은 이 모든 것을 가능하게 만드는 마법의 다리입니다. Claude Code가 Chrome 브라우저를 직접 제어할 수 있게 해주는 특별한 도구죠.

핵심 개념을 아이에게 설명하듯이 {#핵심-개념을-아이에게-설명하듯이}

**MCP (Model Context Protocol)**를 레고 블록이라고 생각해보세요:

  • Claude Code = 레고를 조립하는 아이
  • MCP 서버 = 특별한 기능을 가진 레고 블록
  • MCP Chrome = 자동차를 움직이게 하는 모터 블록

여러분은 오늘 이 "모터 블록"을 Claude Code에 연결하는 방법을 배우게 됩니다!

준비물 체크 - 요리를 시작하기 전에 {#준비물-체크-요리를-시작하기-전에}

🍳 필요한 재료들 {#필요한-재료들}

요리를 시작하기 전에 재료를 확인하듯, 설치에 필요한 것들을 하나씩 체크해봅시다:

  1. Chrome 브라우저

    • 🔍 확인 방법: 크롬을 열고 주소창에 chrome://version 입력
    • ✅ 버전이 나타나면 OK!

  2. Node.js (버전 18.19.0 이상)

    • 🔍 확인 방법: 터미널에서 node --version 입력
    • ❌ 없다면? nodejs.org에서 LTS 버전 다운로드

  3. Claude Code

    • 🔍 확인 방법: 터미널에서 claude --version 입력
    • ✅ 버전이 나타나면 준비 완료!

  4. 인터넷 연결

    • 설치 파일을 다운로드하기 위해 필요합니다

💡 초보자 팁 {#초보자-팁}

"터미널이 뭔가요?" 라고 생각하셨나요?
Mac에서는 Command + Space → "터미널" 검색
Windows에서는 시작 메뉴 → "명령 프롬프트" 또는 "PowerShell" 검색

Plain Text

1 line

스텝 1 - Chrome 확장 프로그램 다운로드 {#스텝-1-chrome-확장-프로그램-다운로드}

🎬 이야기: "보물찾기의 시작" {#이야기-보물찾기의-시작}

MCP Chrome을 설치하는 첫 단계는 GitHub라는 보물창고에서 Chrome 확장 프로그램을 찾는 것입니다.

📝 단계별 가이드 {#단계별-가이드}

  1. GitHub 릴리즈 페이지 방문

    Plain Text 
    https://github.com/hangwin/mcp-chrome/releases
    2 lines
    - 🖱️ 위 주소를 클릭하거나 복사해서 Chrome 주소창에 붙여넣기

  2. 최신 버전 찾기

    • 페이지 상단에 "Latest" 라벨이 붙은 버전을 찾습니다
    • 예: v0.0.6 또는 더 높은 버전

  3. 다운로드 파일 찾기

    • "Assets" 섹션 클릭 (화살표 모양)
    • chrome-mcp-server-0.0.6.zip 같은 파일 찾기
    • 클릭해서 다운로드!

🎯 중요한 순간! {#중요한-순간}

다운로드가 완료되면:

  1. 다운로드 폴더로 이동
  2. ZIP 파일을 더블클릭해서 압축 해제
  3. chrome-mcp-server-0.0.6 폴더가 생성됨

💡 초보자를 위한 꿀팁 {#초보자를-위한-꿀팁}

압축이 자동으로 풀리지 않나요?

  • Mac: 더블클릭하면 자동으로 압축 해제
  • Windows: 우클릭 → "압축 풀기" 또는 "Extract All" 선택

스텝 2 - Chrome에 확장 프로그램 설치하기 {#스텝-2-chrome에-확장-프로그램-설치하기}

🎬 이야기: "Chrome에게 새로운 능력 부여하기" {#이야기-chrome에게-새로운-능력-부여하기}

이제 다운로드한 확장 프로그램을 Chrome에 설치할 차례입니다. 마치 스마트폰에 앱을 설치하는 것과 비슷해요!

📝 단계별 가이드 {#단계별-가이드}

  1. Chrome 확장 프로그램 관리 페이지 열기

    • Chrome을 엽니다
    • 주소창에 chrome://extensions/ 입력하고 Enter
    • 또는: 메뉴(⋮) → 도구 더보기 → 확장 프로그램

  2. 개발자 모드 켜기

    • 페이지 오른쪽 상단의 "개발자 모드" 스위치를 켭니다
    • 🔄 스위치가 파란색으로 변하면 성공!

  3. 확장 프로그램 로드하기

    • "압축해제된 확장 프로그램을 로드합니다" 버튼 클릭
    • 아까 압축 해제한 폴더 선택
    • "선택" 또는 "폴더 선택" 클릭

✅ 설치 성공 확인 {#설치-성공-확인}

  • Chrome 툴바에 새로운 아이콘이 나타남
  • 확장 프로그램 목록에 "MCP Chrome Server" 표시
  • 오류 메시지가 없으면 성공!

🚨 주의사항 {#주의사항}

"오류가 발생했습니다" 메시지가 보이나요?

  • 올바른 폴더를 선택했는지 확인 (manifest.json 파일이 있는 폴더)
  • 폴더 경로에 한글이 없는지 확인

스텝 3 - mcp-chrome-bridge 설치하기 {#스텝-3-mcp-chrome-bridge-설치하기}

🎬 이야기: "다리 놓기" {#이야기-다리-놓기}

Chrome 확장 프로그램(섬)과 Claude Code(육지)를 연결하는 다리가 필요합니다. 그 다리가 바로 mcp-chrome-bridge입니다!

📝 단계별 가이드 {#단계별-가이드}

  1. 터미널 열기

    • Mac: Command + Space → "터미널" 검색
    • Windows: 시작 메뉴 → "명령 프롬프트" 또는 "PowerShell"

  2. npm으로 설치하기

    Bash 
    npm install -g mcp-chrome-bridge
    2 lines
    - 이 명령어를 복사해서 터미널에 붙여넣기 - Enter 키 누르기 - 설치가 진행되는 동안 잠시 기다리기 (1-3분)

✅ 설치 성공 신호 {#설치-성공-신호}

Plain Text 
added 156 packages in 3s
1 line
이런 메시지가 나타나면 성공!

🚨 자주 발생하는 문제들 {#자주-발생하는-문제들}

"npm이 인식되지 않습니다" 오류?

  • Node.js를 먼저 설치해야 합니다
  • nodejs.org에서 LTS 버전 다운로드

"권한이 거부되었습니다" 오류?

  • Mac/Linux: sudo npm install -g mcp-chrome-bridge 사용
  • Windows: 관리자 권한으로 터미널 실행

Bash

1 line

스텝 4 - Claude Code에 연결하기 {#스텝-4-claude-code에-연결하기}

🎬 이야기: "마법의 연결고리 만들기" {#이야기-마법의-연결고리-만들기}

이제 가장 중요한 순간입니다! Claude Code가 Chrome을 제어할 수 있도록 설정을 추가하는 거예요.

📝 단계별 가이드 {#단계별-가이드}

방법 1: Claude Code 명령어 사용 (가장 쉬운 방법!)

  1. Claude Code 세션에서 직접 설치

    Claude Code를 실행한 상태에서 다음 명령어를 입력하세요:

Plain Text 
/mcp install mcp-chrome
2 lines

Claude Code가 자동으로 설치를 진행합니다!

  1. 설치 확인
    Plain Text 
    /mcp list
    2 lines
    목록에 mcp-chrome이 보이면 성공!

방법 2: 터미널에서 Claude CLI 사용

  1. 터미널에서 실행

    Bash 
    claude mcp add mcp-chrome streamableHttp://127.0.0.1:12306/mcp
    2 lines

  2. JSON 형식으로 추가 (고급)

    Bash 
    claude mcp add-json chrome-mcp-server '{"type":"streamableHttp","url":"http://127.0.0.1:12306/mcp"}'
    2 lines

  3. 잘 안 되나요? 수동으로 추가해봅시다!

방법 3: 수동 설정 (더 확실한 방법)

  1. 설정 파일 위치 찾기

    • Mac/Linux: ~/.claude.json
    • Windows: C:\Users\[사용자명]\.claude.json

  2. 설정 파일 편집

    JSON 
    {
  "mcpServers": {
    "chrome-mcp-server": {
      "type": "streamableHttp",
      "url": "http://127.0.0.1:12306/mcp"
    }
  }
}
    9 lines

💡 초보자를 위한 상세 설명 {#초보자를-위한-상세-설명}

설정 파일을 어떻게 편집하나요?

  1. Mac에서:

    Bash 
    open ~/.claude.json
    2 lines
    기본 텍스트 편집기로 열립니다

  2. Windows에서:

    • 파일 탐색기 열기
    • 주소창에 %USERPROFILE% 입력
    • .claude.json 파일을 메모장으로 열기

⚠️ 중요한 주의사항 {#중요한-주의사항}

Chrome 확장 프로그램이 다른 이름을 요구하나요?

때로는 Chrome 확장 프로그램이 streamable-mcp-server라는 이름을 찾을 수 있습니다. 이 경우:

JSON 
{
  "mcpServers": {
    "chrome-mcp-server": {
      "type": "streamableHttp",
      "url": "http://127.0.0.1:12306/mcp"
    },
    "streamable-mcp-server": {
      "type": "streamableHttp",
      "url": "http://127.0.0.1:12306/mcp"
    }
  }
}
12 lines

두 개의 이름을 모두 추가하면 확실합니다!

문제 해결 - 막힐 때 확인할 것들 {#문제-해결-막힐-때-확인할-것들}

🔧 자주 발생하는 문제와 해결책 {#자주-발생하는-문제와-해결책}

1. "연결되지 않음" 문제

증상: MCP 서버 목록에 나타나지 않음

해결책:

  • Chrome 확장 프로그램이 활성화되어 있는지 확인
  • 포트 12306이 다른 프로그램에서 사용 중인지 확인
  • 방화벽이 차단하고 있지 않은지 확인

2. "Invalid configuration" 오류

증상: 설정 추가 시 오류 발생

해결책:

  • JSON 형식이 올바른지 확인 (쉼표, 따옴표)
  • 온라인 JSON 검증기 사용: jsonlint.com

3. Chrome 확장 프로그램 오류

증상: "매니페스트 파일을 읽을 수 없습니다"

해결책:

  • 올바른 폴더 선택 확인 (manifest.json이 있는 폴더)
  • 다시 다운로드 후 압축 해제

💡 디버깅 팁 {#디버깅-팁}

  1. 로그 확인하기

    • Chrome: 확장 프로그램 페이지에서 "오류" 확인
    • Claude Code: 에러 메시지 자세히 읽기

  2. 단계별로 다시 확인

    • 각 단계를 순서대로 다시 진행
