/clear 명령어 주의사항

/clear 명령어 사용 시 주의할 점들:

⚠️ 주요 주의사항 {#주요-주의사항}

1. 완전 삭제 - 복구 불가 {#1-완전-삭제-복구-불가}

  • 대화 내용이 완전히 사라짐
  • 되돌릴 수 없음
  • 백업 기능 없음

2. 컨텍스트 손실 {#2-컨텍스트-손실}

  • 프로젝트 정보 모두 삭제
  • /init으로 설정한 내용 사라짐
  • /memory로 저장한 정보 초기화

3. 진행 중인 작업 중단 {#3-진행-중인-작업-중단}

  • 현재 작업 맥락 손실
  • 단계별 진행사항 리셋
  • 이전 결정사항들 잊어버림

🤔 사용 전 확인사항 {#사용-전-확인사항}

체크리스트: {#체크리스트}

  • 정말로 대화를 지워야 하는가?
  • 중요한 정보가 포함되어 있지 않은가?
  • 진행 중인 작업이 완료되었는가?
  • /compact 명령어로 충분하지 않은가?

✅ 안전한 사용 시점 {#안전한-사용-시점}

권장 상황: {#권장-상황}

  • 새 프로젝트 시작 시
  • 민감한 정보 포함 시 (보안상 삭제 필요)
  • 대화가 너무 길어졌을 때 (성능 저하)
  • 완전히 다른 주제로 전환 시

비권장 상황: {#비권장-상황}

  • 프로젝트 진행 중
  • 중요한 설정이나 결정사항이 있을 때
  • 단순히 화면 정리 목적

💡 대안책 {#대안책}

/clear 대신 고려해볼 것들: {#clear-대신-고려해볼-것들}

1. /compact 사용

  • 대화 내용 압축
  • 중요 정보 보존
  • 성능 개선

2. /memory 활용

  • 중요 정보 미리 저장
  • /clear 후에도 정보 유지

3. 파일로 백업

  • 중요한 대화 내용을 별도 저장
  • 필요 시 참조 가능

🚨 실수 방지 팁 {#실수-방지-팁}

사용 전: {#사용-전}

Plain Text 
1. 중요 정보 → /memory 저장
2. 필요 시 → 대화 내용 복사
3. 확인 완료 → /clear 실행
3 lines

사용 후: {#사용-후}

Plain Text 
1. /init으로 프로젝트 재설정
2. /memory 내용 다시 확인
3. 필요한 컨텍스트 재구성
3 lines

핵심 {#핵심}

/clear는 강력한 명령어입니다. 신중하게 사용하고, 가능하면 /compact를 먼저 고려하세요.
