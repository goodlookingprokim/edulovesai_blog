4-6. /agents - 커스텀 AI 서브에이전트 관리

🎯 사용 시점 {#사용-시점}

  • 특화된 작업이 필요할 때
  • 커스텀 AI 서브에이전트가 필요할 때
  • 특정 작업에 맞춤형 도우미가 필요할 때

📝 기본 사용법 {#기본-사용법}

명령어 {#명령어}

Bash 
> /agents
1 line

공식 기능 {#공식-기능}

  • 커스텀 AI 서브에이전트 관리
  • 특화된 작업을 위한 커스텀 AI 서브에이전트 관리

🔧 상세 기능 {#상세-기능}

주요 특징 {#주요-특징}

  • 서브에이전트 관리: 특화된 작업을 위한 AI 서브에이전트 생성 및 관리
  • 작업 특화: 특정 작업에 맞춤화된 에이전트 운영
  • 커스텀 설정: 사용자의 필요에 따른 맞춤형 에이전트 구성

💡 실제 사용 예시 {#실제-사용-예시}

1. 에이전트 관리 인터페이스 접근 {#1-에이전트-관리-인터페이스-접근}

Bash 
# 에이전트 관리 화면으로 이동
> /agents


사용 가능한 작업:


- 기존 에이전트 목록 확인


- 새 에이전트 생성


- 에이전트 설정 수정


- 에이전트 삭제
7 lines

2. 특화 작업을 위한 에이전트 활용 {#2-특화-작업을-위한-에이전트-활용}

Bash 
# 특정 작업에 맞춤화된 에이전트 사용
> /agents


예시:


- 코드 리뷰 전용 에이전트


- 문서화 전용 에이전트


- 테스트 작성 전용 에이전트
6 lines

✅ 언제 사용하면 좋을까? {#언제-사용하면-좋을까}

최적의 사용 시점 {#최적의-사용-시점}

  • 반복적인 특화 작업이 있을 때
  • 일관된 작업 방식이 필요할 때
  • 특정 도메인 지식이 필요한 작업이 있을 때
  • 워크플로우 자동화를 원할 때

효과적인 활용 상황 {#효과적인-활용-상황}

Bash 
🎯 권장 사용 상황:
- 특정 코딩 스타일이나 패턴을 지속적으로 적용해야 하는 경우
- 반복되는 리뷰나 검증 작업이 있는 경우
- 특정 프레임워크나 라이브러리에 특화된 작업이 필요한 경우
- 팀의 일관된 작업 방식을 유지하고 싶은 경우
5 lines

⚠️ 중요한 한계사항 {#중요한-한계사항}

사용 제한사항 {#사용-제한사항}

  • 초기 설정 필요: 에이전트를 먼저 구성해야 함
  • 학습 시간 필요: 효과적인 에이전트 설정에는 시간이 소요됨
  • 관리 복잡성: 여러 에이전트를 관리하는 것은 복잡할 수 있음

대상 사용자별 적합성 {#대상-사용자별-적합성}

  • ✅ 반복 작업이 많은 개발자 - 매우 유용
  • ✅ 팀 리더 - 일관된 작업 방식 유지에 도움
  • ❌ 일회성 작업 중심 사용자 - 설정 대비 효과 낮음
  • ❌ 초보 사용자 - 설정이 복잡할 수 있음

🔄 대안 명령어 {#대안-명령어}

에이전트를 사용하지 않고도 유사한 작업이 가능합니다:

Bash 
/review      # 기본 코드 리뷰
/help        # 일반적인 도움말
/init        # 프로젝트 초기화
/config      # 기본 설정 관리
4 lines

📊 요약 {#요약}

구분 내용
장점 특화된 작업의 일관성 유지, 워크플로우 자동화
단점 초기 설정 복잡성, 관리 오버헤드
사용 빈도 상황별 (반복 작업이 많은 경우)
대체 명령어 기본 명령어들 (개별 작업별로)

결론: 반복되는 특화 작업이 많다면 유용하지만, 일회성 작업에는 과도할 수 있습니다!

⚠️ 불확실한 정보 안내 {#불확실한-정보-안내}

주의: 이 명령어의 정확한 작동 방식과 세부 기능은 공식 문서에 상세히 나와있지 않습니다. 실제 사용 시 기대와 다를 수 있으니 직접 테스트해보시기 바랍니다.
