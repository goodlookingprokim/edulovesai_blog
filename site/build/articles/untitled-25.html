AI에게 '똑똑하게' 일 시키는 법 (요리사 비유)

엄청나게 유능하지만, 말귀를 곧이곧대로만 알아듣는 최고의 요리사(AI)가 우리 팀에 새로 왔다고 상상해 보세요. 이 요리사에게 어떻게 주문해야 우리가 원하는 완벽한 요리를 매번 만들어낼 수 있을까요? 여기서 요리사는 AI, 레시피는 프롬프트입니다.

1단계: 대충 주문하기 vs. 완벽한 레시피 주기 {#1단계-대충-주문하기-vs-완벽한-레시피-주기}

나쁜 예시 👎: "요리사님, 맛있는 파스타 하나 만들어주세요."

  • 이렇게 말하면 요리사는 자기 마음대로 크림 파스타를 만들 수도, 토마토 파스타를 만들 수도 있습니다. 우리가 원하던 요리가 아닐 확률이 높죠.

  • 개발로 치면, "로그인 기능 만들어주세요." 라고 AI에게 막연하게 지시하는 것과 같습니다.

좋은 예시 👍: 체계적인 '레시피'를 전달한다.

최고의 요리사에게는 아래 4가지 항목이 포함된 완벽한 레시피를 줘야 합니다.

  1. 요구사항 (Requirement): 손님의 관점에서 어떤 요리인지 설명합니다.

    • 요리 예시: "채식주의자 손님을 위한, 살짝 매콤한 토마토 파스타"

    • 개발 예시: "사용자는 아이디와 비밀번호로 로그인할 수 있다."

  2. 구현 상세 (Implementation Detail): 주방장(개발자)의 관점에서 구체적인 지시를 합니다.

    • 요리 예시: "파스타 면은 '데체코' 브랜드를 쓰고, 토마토는 '산 마르지노' 품종을 써. 마늘 먼저 볶다가 면을 넣어."

    • 개발 예시: "로그인 로직은 AuthController 안에 login 메소드로 구현해. 데이터베이스 모델은 User를 사용하고, 비밀번호는 bcrypt로 암호화해."

  3. 제약 조건 (Constraint): 절대 하지 말아야 할 것을 명확히 알려줍니다.

    • 요리 예시: "치즈나 우유 같은 유제품은 절대 넣으면 안 돼."

    • 개발 예시: "로그인 시 사용자의 비밀번호를 평문으로 로그에 남기면 절대 안 돼."

  4. 성공 조건 (Acceptance Tests): 어떤 상태가 되면 '요리 성공'인지 알려줍니다.

    • 요리 예시: "면은 알덴테로 익어야 하고, 소스는 따뜻해야 하며, 매콤한 맛이 나야 성공이야."

    • 개발 예시: "테스트 계정으로 로그인이 성공해야 하고, 틀린 비밀번호를 입력하면 에러 메시지가 나와야 해."

이렇게 구체적인 레시피(프롬프트)를 주면, 요리사(AI)는 우리가 원하는 결과물을 거의 완벽하게 만들어냅니다.

2단계: 머릿속으로 요리 시뮬레이션 돌려보기 (프리모템) {#2단계-머릿속으로-요리-시뮬레이션-돌려보기-프리모템}

레시피를 요리사에게 주기 전에, 잠시 눈을 감고 상상해 보는 겁니다. "이 레시피대로 요리했을 때, 혹시 실수가 나올 만한 부분이 있을까?"

  • 요리 예시: "레시피에 '소금을 적당히 넣으세요'라고 썼네. 근데 이 요리사는 '적당히'를 잘 모르지. 짜게 만들 수도 있겠다. '천일염 5g을 넣으세요'라고 고쳐야겠다."

  • 개발 예시: 프롬프트에 "미리 정의된 사용자 역할을 적용하세요"라고 썼더니, AI가 이걸 어떻게 구현할지 헷갈릴 수 있겠네요. "사용자 역할은 환경 변수에 설정된 값을 읽어서 적용하세요"라고 명확하게 수정합니다.

이렇게 AI가 헷갈리거나 실수할 만한 부분을 미리 찾아내서 레시피를 더 명확하게 만드는 과정을 **프리모템(Pre-mortem)**이라고 합니다.

3단계: 완성된 요리를 맛보고 레시피 수정하기 (포스트모템) {#3단계-완성된-요리를-맛보고-레시피-수정하기-포스트모템}

요리사가 레시피대로 요리를 가져왔습니다. 거의 완벽한데, 바질 향이 조금 부족하네요. 이때 그냥 "바질 좀 더 넣어주세요"라고 말하고 끝내는 게 아닙니다.

원래 전달했던 '마스터 레시피'에 빨간 펜으로 수정사항을 기록하는 겁니다.

  • (추가 요청) "바질 잎 3장을 마지막에 찢어서 넣어주세요."

  • (레시피 업데이트) 마스터 레시피의 마지막 단계에 해당 내용을 추가합니다.

이렇게 하면 나중에 다른 요리사가 이 레시피로 요리해도 똑같은 실수를 반복하지 않겠죠. 실제 AI와 작업하며 추가로 요청했던 내용을 원래 프롬프트에 다시 반영해서 개선하는 이 과정을 **포스트모템(Post-mortem)**이라고 합니다.

아주 중요한 원칙: '한 장짜리 완벽한 레시피'를 지향하세요 (싱글턴 프롬프트) {#아주-중요한-원칙-한-장짜리-완벽한-레시피를-지향하세요-싱글턴-프롬프트}

요리사 옆에 서서 "자, 이제 양파를 썰고... 아니, 더 잘게 썰어요. 이제 마늘을 넣고... 아, 저쪽 팬 말고요." 와 같이 계속 말을 걸면서 요리를 시키면 요리사는 헷갈리고 결과물도 엉망이 됩니다.

가장 좋은 방법은, 수정과 보완이 끝난 완벽한 레시피 한 장을 딱 주고, 요리사가 그것만 보고 요리에 집중하게 하는 것입니다. AI에게 일을 시킬 때도 여러 번에 걸쳐 대화하기보다, 한 번의 지시(프롬프트)로 원하는 결과물을 얻는 '싱글턴(Singleton)' 방식을 지향해야 하는 이유입니다.

보너스 팁: 주방을 항상 깨끗하게 유지하세요 (컨텍스트 관리) {#보너스-팁-주방을-항상-깨끗하게-유지하세요-컨텍스트-관리}

요리사가 일하는 주방(AI가 코드를 참고하는 범위)이 깨끗해야 좋은 요리가 나옵니다.

  1. 오래된 재료는 버리세요 (레거시 파일 지우기): 냉장고에 썩은 채소가 있으면 요리사가 실수로 쓸 수도 있습니다. 마찬가지로, 프로젝트에 지금은 안 쓰는 옛날 코드(레거시)가 남아있으면 AI가 그걸 참고해서 엉뚱한 코드를 만들 수 있습니다. 과감히 지우세요!

  2. 재료 이름표를 정확히 붙이세요 (정확한 변수명/경로 쓰기): '설탕' 통에 소금이 들어있으면 요리가 망하겠죠? AI에게 지시할 때도 두루뭉술하게 "인증 관련 파일에 추가해줘"라고 하지 말고, "AuthController.ts 파일에 login 메소드를 추가해줘" 와 같이 실제 파일명, 클래스명, 메소드명을 정확히 언급해야 AI가 길을 잃지 않습니다.

결론 {#결론}

AI를 유능한 개발 동료로 만들기 위한 핵심은, AI와의 작업을 '대화'가 아닌 '체계적인 개발 프로세스'로 만드는 것입니다.

완벽한 레시피(프롬프트)를 작성하고, 동료와 함께 리뷰하고, 실패 경험을 통해 레시피를 계속 발전시켜 나가세요. 이렇게 팀의 노하우가 담긴 레시피들이 쌓이면, AI라는 최고의 요리사는 항상 우리 팀이 원하는 최고의 결과물을 만들어 줄 겁니다.
