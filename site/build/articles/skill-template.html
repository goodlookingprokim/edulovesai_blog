나의 첫 Skill 만드는 템플릿

이 문서는 복사해서 바로 쓸 수 있는 템플릿입니다. 자신의 Skill을 만들 때 이 템플릿을 기초로 수정하면서 배우세요.

📋 목차 {#목차}

  1. 사전 준비
  2. 파일 1: SKILL.md 템플릿
  3. 파일 2: category_mapping.json 템플릿
  4. 파일 3: enhancement_rules.md 템플릿
  5. 체크리스트
  6. 자주 하는 실수

사전 준비 {#사전-준비}

Step 1: 폴더 만들기 {#step-1-폴더-만들기}

터미널에서 다음을 실행:

Bash 
# 1. 폴더 생성
mkdir -p ~/.claude/skills/my-skill-name


2. 폴더로 이동


cd ~/.claude/skills/my-skill-name


3. 파일 생성


touch SKILL.md
touch category_mapping.json
touch enhancement_rules.md
8 lines

Step 2: 템플릿 결정 {#step-2-템플릿-결정}

자신의 Skill이 할 일은?

  • 텍스트 변환? (예: 마크다운 → HTML)
  • 파일 분류? (예: 파일을 주제별로 정렬)
  • 콘텐츠 향상? (예: 원본 텍스트 → 정리된 노트)
  • 메타데이터 생성? (예: 파일에 YAML 헤더 추가)

난이도 선택:

  • 🟢 쉬움: 텍스트 변환 (2시간)
  • 🟡 중간: 콘텐츠 향상 + 분류 (4시간)
  • 🔴 어려움: 다국어 지원 + 로깅 (8시간)

📝 JSON 작성 팁 {#json-작성-팁}

JSON

JSON 
❌ {"key": "value",}  // 마지막에 쉼표 있음

❌ {'key': 'value'}   // 홑따옴표 사용

Plain Text

실수 2: JSON 형식 오류 {#실수-2-json-형식-오류}

JSON

  // 주석 (JSON에서 안 됨!)
}
4 lines

✅ 올바른 형식:

JSON

}
4 lines

실수 3: 우선순위(priority) 설정 실수 {#실수-3-우선순위priority-설정-실수}

❌ 같은 priority:

JSON 
{"priority": 1, "keywords": ["Claude Code", "MCP"]},
{"priority": 1, "keywords": ["Claude"]}
2 lines
→ 둘 다 priority 1이면 어느 것을 먼저 적용할까? 혼란!

✅ 명확한 priority:

JSON 
{"priority": 1, "keywords": ["Claude Code"]},  // 가장 구체적
{"priority": 2, "keywords": ["Claude"]},      // 그 다음
