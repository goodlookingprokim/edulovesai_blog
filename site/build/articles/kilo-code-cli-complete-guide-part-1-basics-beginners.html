📚 프로그래밍 입문자

  • 상황: "코딩을 배우고 싶은데 뭐부터 해야 할지 모르겠어요"
  • 이 도구가 도움이 되는 이유: AI가 친절한 선생님처럼 코드를 설명해줍니다

🤔 코드 이해가 어려운 분

  • 상황: "다른 사람이 쓴 코드를 봐도 무슨 말인지 모르겠어요"
  • 이 도구가 도움이 되는 이유: 복잡한 코드를 쉬운 말로 풀어서 설명해줍니다

🐛 에러 메시지가 두려운 분

  • 상황: "빨간 글씨만 보면 머리가 아파요"
  • 이 도구가 도움이 되는 이유: 에러를 한국어로 설명하고 해결 방법을 알려줍니다

💻 터미널이 낯선 분

  • 상황: "검은 화면이 무서워요"
  • 이 도구가 도움이 되는 이유: 터미널을 친근한 대화 창으로 만들어줍니다

Kilo Code CLI란 무엇인가요? {#kilo-code-cli란-무엇인가요}

🎯 일상 비유로 이해하기 {#일상-비유로-이해하기}

비유 1: AI 개인 코딩 선생님

Plain Text 
상황을 상상해보세요...


당신은 수학 문제를 풀고 있습니다.
옆에 친절한 선생님이 앉아 계십니다.


"선생님, 이 문제 어떻게 푸는 거예요?"
→ 선생님이 차근차근 설명해줍니다.


"여기서 실수한 것 같은데, 어디가 틀렸을까요?"
→ 선생님이 틀린 부분을 찾아 고쳐줍니다.


Kilo Code CLI는 바로 이런 선생님입니다!
단, 수학이 아니라 '코딩'을 가르쳐주는 선생님이죠.
9 lines

비유 2: 번역기 + 도우미

Plain Text 
외국어 책을 읽는다고 생각해보세요.


어려운 단어가 나올 때마다
→ 번역기에게 물어봅니다.


문장 구조가 이해 안 될 때
→ 번역기가 쉽게 풀어서 설명해줍니다.


Kilo Code CLI는 '코드 번역기'입니다!
복잡한 코드를 쉬운 말로 번역해줍니다.
7 lines

🔍 정확한 정의 {#정확한-정의}

Kilo Code CLI는:

  1. 터미널에서 실행되는 프로그램입니다
  2. AI를 활용하여 코드 작성을 도와줍니다
  3. 대화하듯이 질문하고 답변을 받을 수 있습니다
  4. 수백 개의 AI 모델을 사용할 수 있습니다

왜 Kilo Code를 배워야 할까요? {#왜-kilo-code를-배워야-할까요}

💡 초보자에게 특히 좋은 이유 {#초보자에게-특히-좋은-이유}

1️⃣ 학습 속도가 빨라집니다

기존 방법 (도구 없이 배우기):

Plain Text 
1. 에러 발생
2. 구글 검색
3. Stack Overflow에서 비슷한 질문 찾기
4. 영어로 된 답변 읽기
5. 내 코드에 적용해보기
6. 또 에러 발생
7. 1번으로 돌아가기
⏰ 소요 시간: 30분~1시간
8 lines

Kilo Code 사용:

Plain Text 
1. 에러 발생
2. Kilo Code에게 물어보기
3. 한국어로 설명 듣기
4. 제안된 해결책 적용하기
⏰ 소요 시간: 5분
5 lines

2️⃣ 혼자서도 문제를 해결할 수 있습니다

실제 이야기:

Plain Text 
민수는 코딩 부트캠프에 다니고 있습니다.
밤 11시, 과제를 하다가 막혔습니다.


선택지:
A) 내일 아침까지 기다려서 선생님께 질문하기
B) 친구에게 카톡으로 물어보기 (답장이 올까...?)
C) Kilo Code에게 즉시 물어보고 해결하기 ✅


민수는 C를 선택했고, 5분 만에 문제를 해결했습니다!
7 lines

3️⃣ 실수가 두렵지 않게 됩니다

심리적 장벽 제거:

Plain Text 
사람에게 질문할 때:
😰 "이런 걸 물어봐도 될까?"
😓 "바보 같은 질문이면 어쩌지?"
😔 "이미 세 번이나 물어봤는데..."


AI에게 질문할 때:
😊 "100번 물어봐도 괜찮아!"
🤗 "어떤 질문이든 친절하게 답해줘!"
💪 "실수해도 부끄럽지 않아!"
8 lines

Plain Text 
✅ Mac: macOS 10.15 이상
✅ Windows: Windows 10 이상
✅ Linux: Ubuntu 18.04 이상

2. Node.js (노드 제이에스)

용어 설명: JavaScript를 컴퓨터에서 실행할 수 있게 해주는 프로그램

비유:

  • 게임을 하려면 게임기가 필요하듯이
  • Node.js 프로그램을 실행하려면 Node.js가 필요합니다

설치 확인하기:

Bash 
# 터미널을 열고 이 명령어를 입력하세요

v18.17.0 (숫자는 다를 수 있습니다)


이렇게 나오면 설치가 필요합니다:


command not found: node
6 lines

없다면 설치하기:

  1. https://nodejs.org 접속
  2. "LTS" 버전 다운로드 (추천)
  3. 다운로드한 파일 실행
  4. "다음", "동의", "설치" 버튼만 누르면 끝!

Plain Text

터미널이란 무엇인가요? {#터미널이란-무엇인가요}

🖥️ 쉬운 설명 {#쉬운-설명}

비유: 마법의 주문을 외우는 곳

Plain Text 
옛날 옛적, 컴퓨터는 마우스가 없었습니다.
모든 것을 '글자로 명령'해야 했죠.


터미널은 그때의 방식입니다.
하지만 아직도 쓰는 이유가 있어요!


왜냐하면...


    

  • 빠르고 정확하기 때문
    • 

  • 자동화하기 쉽기 때문
    • 

  • 개발자들이 선호하기 때문
8 lines

실생활 비유

Plain Text 
마우스로 파일 10개 복사하기:
1. 마우스로 파일 선택
2. 드래그
3. 폴더로 이동
4. 붙여넣기
5. 1-4 반복 10번
⏰ 1분 소요


터미널로 파일 10개 복사하기:
cp file*.txt /destination/
⏰ 1초 소요!
10 lines

📝 기본 터미널 명령어 {#기본-터미널-명령어}

필수 명령어 5가지

1. pwd (Print Working Directory)

Bash 
pwd
# 출력 예시: /Users/yourname/Documents
2 lines
**뜻**: "지금 내가 어디에 있는지 보여줘" **비유**: 내비게이션에서 "현재 위치" 보기

2. ls (List)

Bash 
ls
# 출력 예시:
# file1.txt  file2.txt  폴더1  폴더2
3 lines
**뜻**: "여기 있는 파일과 폴더를 보여줘" **비유**: 서랍 열어서 안에 뭐가 있나 보기

3. cd (Change Directory)

Bash 
cd Documents
# Documents 폴더로 이동
2 lines
**뜻**: "이 폴더로 이동해" **비유**: 방에서 거실로 이동하기

4. mkdir (Make Directory)

Bash 
mkdir my-project
# my-project라는 폴더 생성
2 lines
**뜻**: "새 폴더 만들어" **비유**: 새 서랍 만들기

5. clear

Bash 
clear
# 화면이 깨끗해집니다
2 lines
**뜻**: "화면 지워" **비유**: 칠판 지우기

쉬운 설명

Plain Text 
CLI = 명령줄 인터페이스


'인터페이스'란?
→ 사람과 컴퓨터가 소통하는 방법


두 가지 종류:


    

  1. GUI (그래픽): 마우스로 클릭 (우리가 익숙한 방식)
    2. 

  2. CLI (글자): 키보드로 명령 타이핑
6 lines

비교 표

특징 GUI (그래픽) CLI (명령줄)
예시 바탕화면, 아이콘 터미널, 검은 화면
사용 방법 마우스 클릭 키보드 타이핑
장점 직관적, 쉬움 빠름, 정확함
단점 느림, 반복작업 힘듦 배우기 어려움
비유 그림책 읽기 소설책 읽기

왜 개발자들은 CLI를 좋아할까?
