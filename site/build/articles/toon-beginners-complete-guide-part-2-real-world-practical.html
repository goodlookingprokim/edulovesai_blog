🏗️ 객체 마스터하기 {#객체-마스터하기}

개념: 객체란? {#개념-객체란}

객체는 하나의 대상이 여러 속성을 가지는 경우입니다.

일상 비유: 운전면허증

  • 이름
  • 생년월일
  • 주소
  • 발급 기관
  • 유효 기간

기본 형태 {#기본-형태}

Code 
속성이름1: 값1
속성이름2: 값2
속성이름3: 값3
3 lines

예제 1: 직원 정보 (초급) {#예제-1-직원-정보-초급}

Code 
직원이름: 김철수
직급: 부장
부서: 개발팀
입사일: 2020-01-15
급여: 5000000
5 lines

이 객체가 담고 있는 정보:

  • 누구?: 김철수
  • 어떤 위치?: 부장
  • 어디서?: 개발팀
  • 언제부터?: 2020년 1월 15일
  • 얼마나?: 500만원

예제 2: 책 정보 (중급) {#예제-2-책-정보-중급}

Code 
제목: 혼자 공부하는 파이썬
저자: 윤인성
출판사: 한빛미디어
출판년도: 2024
페이지수: 588
평점: 8.5
6 lines

예제 3: 중첩 객체 (고급) {#예제-3-중첩-객체-고급}

때로는 객체 안에 또 다른 객체가 있을 수 있습니다.

Code 
학생:
  이름: 김철수
  나이: 25
  주소:
    시도: 서울
    시군구: 강남구
    상세: 테헤란로 123
  연락처:
    휴대폰: 010-1234-5678
    이메일: kim@email.com
10 lines

구조 분석:

Plain Text 
학생 (객체)
├── 이름: 김철수
├── 나이: 25
├── 주소 (중첩된 객체)
│   ├── 시도: 서울
│   ├── 시군구: 강남구
│   └── 상세: 테헤란로 123
└── 연락처 (중첩된 객체)
    ├── 휴대폰: 010-1234-5678
    └── 이메일: kim@email.com
10 lines

들여쓰기의 규칙:

  • 부모 객체는 들여쓰기 없음
  • 자식 속성은 2칸 들여쓰기
  • 중첩이 깊어질수록 더 들여씀

💡 체크리스트: 객체 이해 확인 {#체크리스트-객체-이해-확인}

  • 간단한 학생 정보를 TOON으로 표현할 수 있나요?
  • 중첩된 주소 정보를 들여쓰기로 표현할 수 있나요?
  • 들여쓰기를 2칸으로 해야 하는 이유를 알 수 있나요?

📦 배열 완전 정복 {#배열-완전-정복}

개념: 배열이란? {#개념-배열이란}

배열은 여러 항목을 순서대로 나열한 것입니다.

일상 비유:

  • 쇼핑 목록 (우유, 계란, 빵, 치즈)
  • 방문 순서 (1번, 2번, 3번 방)
  • 계절 순서 (봄, 여름, 가을, 겨울)

배열의 3가지 형태 {#배열의-3가지-형태}

형태 1: 텍스트 배열

형식:

Code 
배열이름[개수]: 항목1,항목2,항목3
1 line

예제 1: 계절 목록

Code 
계절[4]: 봄,여름,가을,겨울
1 line

예제 2: 좋아하는 영화

Code 
영화[3]: 기생충,오펜하이머,아바타
1 line

예제 3: 친구 목록

Code 
친구들[5]: 이순신,세종대왕,을지문덕,신사임당,장보고
1 line

형태 2: 숫자 배열

Code 
시험점수[4]: 90,85,88,92
1 line

여기서 [4]의 의미: 4개의 점수가 있어요

형태 3: 혼합 배열

Code 
상품[3]: 우유,계란,100원
1 line

❌ 주의: 이렇게 섞으면 헷갈립니다! 같은 종류로 정렬하세요.

배열 예제들 {#배열-예제들}

예제 1: 쇼핑 목록 (초급)

Code 
구매목록[6]: 우유,계란,빵,치즈,소시지,버터
1 line

시각화:

Plain Text 
항목 1: 우유
항목 2: 계란
항목 3: 빵
항목 4: 치즈
항목 5: 소시지
항목 6: 버터
6 lines

예제 2: 시험 점수들 (중급)

Code 
1차시험점수[4]: 85,90,78,92
2차시험점수[4]: 88,92,80,95
2 lines

예제 3: 여행 일정 (중급)

Code 
여행지[5]: 서울,강릉,속초,강릉,서울
방문기간[5]: 1일,2일,3일,2일,1일
2 lines

💡 배열 만들 때 주의사항 {#배열-만들-때-주의사항}

규칙 설명 예제
쉼표로 분리 항목들은 쉼표(,)로 구분 항목1,항목2,항목3
[개수] 명시 몇 개인지 정확히 적기 [3]: a,b,c
일관성 같은 종류의 항목들만 포함 ❌ [2]: 숫자,텍스트
순서가 중요 첫 번째가 항목 1, 두 번째가 항목 2 월,화,수,목,금

📊 표 형식 데이터 다루기 {#표-형식-데이터-다루기}

개념: 표란? {#개념-표란}

표는 여러 행(사람/물건)이 같은 정보(항목)를 가지는 경우입니다.

쉽게 말해: 엑셀 스프레드시트처럼!

표의 구조 {#표의-구조}

Code 
표이름[행개수]{항목1,항목2,항목3}:
  값1,값2,값3
  값1,값2,값3
  값1,값2,값3
4 lines

예제 1: 학생 성적표 (초급) {#예제-1-학생-성적표-초급}

Code 
학생[3]{이름,국어,수학}:
  김철수,90,85
  이영희,95,92
  박민준,88,90
4 lines

엑셀 시트로 표현:

이름 국어 수학
김철수 90 85
이영희 95 92
박민준 88 90

예제 2: 상품 정보 (중급) {#예제-2-상품-정보-중급}

Code 
상품[3]{상품명,가격,재고}:
  노트북,1500000,15
  마우스,50000,50
  키보드,150000,30
4 lines

엑셀 형태:

상품명 가격 재고
노트북 1500000 15
마우스 50000 50
키보드 150000 30

예제 3: 영화 정보 (중급) {#예제-3-영화-정보-중급}

Code 
영화[3]{제목,감독,개봉년도,평점}:
  기생충,봉준호,2019,8.5
  오펜하이머,크리스토퍼 놀란,2023,8.4
  아바타,제임스 카메론,2009,7.8
4 lines

예제 4: 직원 정보 (고급) {#예제-4-직원-정보-고급}

Code 
직원[4]{사원번호,이름,직급,부서,급여}:
  001,김철수,부장,개발팀,5000000
  002,이영희,과장,디자인팀,4000000
  003,박민준,대리,마케팅팀,3000000
  004,최지현,사원,영업팀,2500000
5 lines

표 만들 때 중요한 규칙 {#표-만들-때-중요한-규칙}

규칙 1: 모든 행이 같은 항목을 가져야 함

✅ 올바른 예:

Code 
학생[2]{이름,나이,학년}:
  김철수,20,3
  이영희,19,2
3 lines

❌ 잘못된 예:

Code 
학생[2]{이름,나이,학년}:
  김철수,20,3
  이영희,19  # ← 항목 개수가 다름!
3 lines

규칙 2: 항목의 순서는 일관되어야 함

✅ 올바른 예:

Code 
영화[2]{제목,감독,연도}:
  기생충,봉준호,2019
  아바타,제임스 카메론,2009
3 lines

❌ 잘못된 예:

Code 
영화[2]{제목,감독,연도}:
  기생충,봉준호,2019
  아바타,2009,제임스 카메론  # ← 순서가 다름!
3 lines

규칙 3: 행의 개수가 정확해야 함

Code 
학생[3]{이름,나이}:  # ← 3명이라고 선언
  김철수,20
  이영희,19
  박민준,21
4 lines

[3]의 의미: "이 표는 정확히 3명의 데이터를 가지고 있어요"

💡 체크리스트: 표 형식 이해 확인 {#체크리스트-표-형식-이해-확인}

  • 표의 구조 ([행]{항목}:) 를 알 수 있나요?
  • 엑셀의 행과 열을 TOON 표에서 찾을 수 있나요?
  • 항목 이름을 중괄호 안에 정렬할 수 있나요?
  • 5개 항목을 가진 3행 표를 만들 수 있나요?

💬 인용 규칙 쉽게 배우기 {#인용-규칙-쉽게-배우기}

인용(따옴표)이 필요한 경우 {#인용따옴표이-필요한-경우}

때로는 데이터 앞뒤에 **따옴표("")**를 붙여야 합니다.

언제 따옴표를 붙일까? {#언제-따옴표를-붙일까}

규칙 1: 공백이 있을 때

❌ 따옴표 없음 (잘못됨):

Code 
이름: 김 철수  # ← 공백이 있어서 문제 발생
1 line

✅ 따옴표 있음 (올바름):

Code 
이름: "김 철수"
1 line

규칙 2: 특수 기호가 있을 때

❌ 따옴표 없음:

Code 
주소: 서울, 강남구  # ← 쉼표가 헷갈림
1 line

✅ 따옴표 있음:

Code 
주소: "서울, 강남구"
1 line

규칙 3: 숫자처럼 보이는 텍스트

❌ 따옴표 없음:

Code 
학번: 2024001  # ← 숫자인지 텍스트인지 헷갈림
1 line

✅ 따옴표 있음:

Code 
학번: "2024001"
1 line

규칙 4: 콜론(:)을 포함하는 경우

❌ 따옴표 없음:

Code 
시간: 14:30:45  # ← 콜론이 구분자로 오해됨
1 line

✅ 따옴표 있음:

Code 
시간: "14:30:45"
1 line

인용 규칙 예제들 {#인용-규칙-예제들}

예제 1: 주소 표현

Code 
주소: "서울시 강남구 테헤란로 123"
1 line

예제 2: 전화번호

Code 
휴대폰: "010-1234-5678"
1 line

예제 3: 이메일

Code 
이메일: "kim@company.com"
1 line

예제 4: 표에서 인용 필요

Code 
직원[2]{이름,직책,이메일}:
  "김 철수",개발팀 부장,"kim@company.com"
  "이 영희",디자인팀 과장,"lee@company.com"
3 lines

🔄 타입 변환 이해하기 {#타입-변환-이해하기}

타입이란? {#타입이란}

데이터의 종류를 의미합니다.

종류들:

  • 숫자: 30, 3.14, -5
  • 텍스트: "안녕하세요"
  • 참/거짓: true, false
  • 없음: null

TOON이 다루는 기본 타입 {#toon이-다루는-기본-타입}

1️⃣ 문자열 (String)

Code

이메일: kim@email.com
2 lines

2️⃣ 숫자 (Number)

Code 
나이: 30
급여: 5000000
키: 175.5
3 lines

3️⃣ 참/거짓 (Boolean)

Code 
활성화: true
휴면계정: false
2 lines

4️⃣ 널 (Null) - "없음" 표현

Code

프로필사진: null
2 lines

특수한 값들의 변환 {#특수한-값들의-변환}

TOON은 일부 특수한 데이터를 자동으로 변환합니다.

변환 규칙표

입력 TOON 변환 설명
undefined null 정의되지 않은 값 → null로
NaN null 유효하지 않은 숫자 → null로
Infinity null 무한대 → null로
-Infinity null 음의 무한대 → null로
Date "2025-11-18T..." 날짜 → ISO 형식 문자열로
BigInt "123456789" 큰 정수 → 문자열로
function null 함수 → null로

명령어를 입력해서 JSON을 TOON으로 바꿀 수 있는 도구입니다.

설치 (터미널에 입력) {#설치-터미널에-입력}

npm install @toon-format/cli
1 line

기본 사용법 {#기본-사용법}

JSON 파일을 TOON으로 변환

Bash 
npx @toon-format/cli input.json -o output.toon
1 line

의미:

  • input.json: 원본 파일 (JSON 형식)
  • -o output.toon: 결과 파일 (TOON 형식)

터미널에 직접 입력

Bash 
echo '{"이름":"김철수","나이":30}' | npx @toon-format/cli

나이: 30

Bash 
npx @toon-format/cli input.json --stats -o output.toon

- JSON: 250 tokens

🌟 핵심 정리 {#핵심-정리}

객체 vs 배열 vs 표 {#객체-vs-배열-vs-표}

구분 용도 예제
객체 한 대상의 여러 정보 1명의 학생 정보
배열 여러 항목 나열 학생 이름 목록
표 같은 정보를 가진 여러 대상 모든 학생의 성적

TOON 작성 5단계 체크리스트 {#toon-작성-5단계-체크리스트}

  1. ✅ 데이터 구조 결정: 객체? 배열? 표?
  2. ✅ 항목 정하기: 어떤 정보를 포함할 건가?
  3. ✅ 형식 따라하기: [개수], {항목}, : 규칙
  4. ✅ 따옴표 확인: 공백이나 특수문자 체크
  5. ✅ 개수 확인: 선언한 개수가 맞는지 확인
