4-7. Agent vs Subagent 차이점과 활용법

🤖 Agent vs Subagent 개념 이해 {#agent-vs-subagent-개념-이해}

Main Agent (Claude Code) {#main-agent-claude-code}

  • 역할: 기본 Claude Code 인터페이스
  • 특징: 범용적이고 저수준, 특정 워크플로우를 강요하지 않음
  • 컨텍스트: 메인 대화 컨텍스트 사용
  • 용도: 전반적인 코딩 작업과 조정

Subagent (서브에이전트) {#subagent-서브에이전트}

  • 역할: 특정 작업에 특화된 AI 어시스턴트
  • 특징: 맞춤형 시스템 프롬프트와 도구 권한을 가진 전문가
  • 컨텍스트: 독립적인 컨텍스트 윈도우 사용
  • 용도: 특정 도메인의 전문 작업

🎯 주요 장점 {#주요-장점}

컨텍스트 격리 {#컨텍스트-격리}

  • 각 서브에이전트는 독립적인 대화 컨텍스트에서 작동
  • 메인 대화의 "컨텍스트 오염" 방지
  • 긴 대화에서 성능 저하 방지

전문화 {#전문화}

  • 특정 개발 도메인을 위한 맞춤형 시스템 프롬프트
  • 지정된 작업에서 더 높은 성공률
  • 재사용 가능한 작업별 워크플로우

유연한 도구 권한 {#유연한-도구-권한}

  • 서브에이전트별로 필요한 도구만 선택적 부여
  • 보안과 효율성 향상

🔧 서브에이전트 생성 방법 {#서브에이전트-생성-방법}

1. 기본 생성 과정 {#1-기본-생성-과정}

Bash

2. 가능한 서브에이전트 유형들 (추정) {#2-가능한-서브에이전트-유형들-추정}

  • 코드 리뷰어: 코드 품질 검토
  • 테스트 실행자: 테스트 자동화
  • 성능 분석가: 성능 최적화
  • 보안 검토자: 보안 취약점 분석
  • 문서화 담당: API 문서 생성
  • 데이터 분석가: 데이터 분석 및 시각화

주의: 구체적인 구현 방법과 시스템 프롬프트 내용은 사용자가 직접 정의해야 하며, 위 유형들은 추정입니다.

🔄 호출 방법 {#호출-방법}

1. 자동 위임 (공식 문서 확인됨) {#1-자동-위임-공식-문서-확인됨}

  • 작업 설명을 기반으로 적절한 서브에이전트를 자동 선택
  • description 필드를 매칭 기준으로 사용

2. 명시적 호출 (공식 문서 확인됨) {#2-명시적-호출-공식-문서-확인됨}

  • 서브에이전트 이름을 직접 언급하여 호출 가능

주의: 구체적인 호출 방법과 매칭 알고리즘은 공식 문서에 상세히 나와있지 않습니다.

🚀 팀 협업 패턴 {#팀-협업-패턴}

협업 패턴 (공식 문서에서 언급됨) {#협업-패턴-공식-문서에서-언급됨}

  • 순차적 패턴: 에이전트들이 순서대로 작업 수행
  • 병렬 처리: 여러 에이전트가 동시에 다른 작업 수행
  • 자동 조정: Claude Code가 에이전트 간 작업을 자동으로 조정

주의: 구체적인 패턴과 작동 방식은 추정이며, 실제 구현 방법은 확실하지 않습니다.

✅ 모범 사례 {#모범-사례}

효과적인 서브에이전트 생성 {#효과적인-서브에이전트-생성}

  1. 단일 책임 원칙: 명확하고 단일한 책임을 가진 서브에이전트 생성
  2. 능동적 사용 장려: description에 "능동적으로" 또는 "즉시" 포함
  3. 적절한 도구 권한: 필요한 도구만 선택적으로 부여
  4. 구체적인 설명: 언제 사용해야 하는지 명확히 명시

사용 시 최적화 팁 {#사용-시-최적화-팁}

  • 복잡한 문제에서 서브에이전트 적극 활용
  • 대화 초기나 작업 시작 시 세부사항 검증용으로 활용
  • 메인 컨텍스트 보존을 위해 조사나 검증 작업에 활용
  • 장기 프로젝트에서 일관된 품질 유지를 위해 활용

⚠️ 주의사항 {#주의사항}

제한사항 {#제한사항}

  • 초기 설정 필요: 효과적인 서브에이전트 구성에 시간 투자 필요
  • 복잡성 증가: 여러 서브에이전트 관리의 복잡성
  • 학습 곡선: 최적의 서브에이전트 활용법 습득 필요

대상 사용자별 적합성 {#대상-사용자별-적합성}

  • ✅ 팀 프로젝트: 일관된 품질과 워크플로우 유지
  • ✅ 복잡한 프로젝트: 전문화된 작업 분담
  • ✅ 반복 작업이 많은 경우: 자동화와 효율성 증대
  • ❌ 간단한 일회성 작업: 설정 오버헤드가 비효율적

📊 활용 시나리오별 추천 {#활용-시나리오별-추천}

시나리오 추천 서브에이전트 사용 목적
코드 품질 관리 code-reviewer, linter-expert 일관된 코드 품질 유지
성능 최적화 performance-analyzer, profiler 병목 지점 식별 및 개선
테스트 자동화 test-runner, test-generator 테스트 커버리지 향상
보안 검토 security-auditor, vulnerability-scanner 보안 취약점 사전 발견
문서화 docs-writer, api-documenter 프로젝트 문서 자동 생성
데이터 분석 data-scientist, visualization-expert 데이터 인사이트 도출

🆕 최신 기능 안내 {#최신-기능-안내}

서브에이전트는 Claude Code의 최신 기능으로, 복잡한 코딩 작업을 전문화된 AI 지원을 통해 개선하는 도구입니다.

확인된 주요 특징 {#확인된-주요-특징}

  • 독립적인 컨텍스트에서 작동
  • 특정 작업에 특화된 AI 어시스턴트
  • 마크다운 파일로 정의
  • /agents 명령어로 관리

주의: 정확한 출시일과 세부 기능은 공식 문서에서 확인이 필요합니다.

결론: 서브에이전트를 통해 Claude Code의 능력을 특정 작업에 맞게 전문화하여 더 높은 품질과 효율성을 달성할 수 있습니다.
