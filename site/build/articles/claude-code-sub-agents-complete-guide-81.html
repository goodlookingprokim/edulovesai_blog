Sub-Agent란? {#sub-agent란}

Sub-Agent는 Claude Code가 특정 작업을 위임할 수 있는 사전 구성된 AI 전문가들입니다. 각 Sub-Agent는:

  • 전문 영역: 특정 목적과 전문 분야를 가짐
  • 독립 컨텍스트: 메인 대화와 분리된 자체 컨텍스트 윈도우 사용
  • 도구 접근 권한: 사용할 수 있는 특정 도구들로 구성 가능
  • 맞춤 시스템 프롬프트: 행동을 안내하는 사용자 정의 시스템 프롬프트 포함

핵심 이점 {#핵심-이점}

🧠 컨텍스트 보존

각 Sub-Agent는 자체 컨텍스트에서 작동하여 메인 대화의 오염을 방지합니다. 상위 목표에 집중할 수 있도록 대화 흐름을 유지합니다.

🎯 전문화된 전문성

특정 도메인에 대한 세부 지침으로 미세 조정이 가능합니다. 지정된 작업에서 더 높은 성공률을 달성합니다.

♻️ 재사용성

한 번 생성하면 여러 프로젝트에서 재사용이 가능합니다. 팀과 공유하여 일관된 워크플로우를 구축할 수 있습니다.

🔐 유연한 권한 관리

각 Sub-Agent마다 다른 도구 접근 수준 설정이 가능합니다. 강력한 도구를 특정 Sub-Agent 유형에만 제한할 수 있습니다.

3. Sub-Agent 정의

  • 권장사항: Claude로 먼저 생성한 후 개인화하여 커스터마이징
  • Sub-Agent에 대한 상세 설명 및 사용 시점 기술

Sub-Agent는 YAML frontmatter가 포함된 Markdown 파일로 다음 두 위치에 저장됩니다:

유형 위치 범위 우선순위

💡 참고: Sub-Agent 이름이 충돌할 경우, 프로젝트 레벨 Sub-Agent가 사용자 레벨 Sub-Agent보다 우선합니다.

구체적인 지침, 모범 사례, Sub-Agent가 따라야 할 제약사항을 포함하세요.

도구 접근 권한 {#도구-접근-권한}

Sub-Agent는 Claude Code의 모든 내부 도구에 대한 접근 권한을 부여받을 수 있습니다.

💡 권장사항: /agents 명령어를 사용하여 도구 접근을 수정하세요. MCP 서버 도구를 포함한 모든 사용 가능한 도구를 나열하는 대화형 인터페이스를 제공하여 필요한 도구를 쉽게 선택할 수 있습니다.

도구 구성을 위한 두 가지 옵션: {#도구-구성을-위한-두-가지-옵션}

1. tools 필드 생략

메인 스레드의 모든 도구(MCP 도구 포함) 상속 (기본값)

2. 개별 도구 지정

쉼표로 구분된 목록으로 세부 제어 (수동 편집 또는 /agents를 통해 편집 가능)

MCP 도구 {#mcp-도구}

Sub-Agent는 구성된 MCP 서버의 MCP 도구에 접근할 수 있습니다. tools 필드가 생략되면 메인 스레드에서 사용 가능한 모든 MCP 도구를 상속받습니다.

/agents 명령어는 Sub-Agent 관리를 위한 종합적인 인터페이스를 제공합니다:

Bash 
/agents
1 line

이 명령어로 열리는 대화형 메뉴에서 다음을 수행할 수 있습니다:

  • 사용 가능한 모든 Sub-Agent 보기 (내장, 사용자, 프로젝트)
  • 가이드된 설정으로 새 Sub-Agent 생성
  • 도구 접근 권한을 포함한 기존 사용자 정의 Sub-Agent 편집
  • 사용자 정의 Sub-Agent 삭제
  • 중복이 있을 때 활성화된 Sub-Agent 확인
  • 사용 가능한 도구의 완전한 목록으로 도구 권한을 쉽게 관리

전통적인 LLM 사용 방식:

  • 질문 → 답변 (일회성)
  • 단방향 상호작용
  • 품질 개선 제한적

Agent 워크플로우 방식:

  • 질문 → 사고 → 연구 → 수정 → 개선 (반복적)
  • 자율적인 품질 향상
  • 실제 팀워크와 유사한 협업

핵심 구성 요소 {#핵심-구성-요소}

1. Reflection (반성/검토)

  • 자신의 작업 결과를 스스로 검토
  • 개선점 자동 발견 및 수정
  • 품질 기준에 따른 자가 평가

2. Tool Use (도구 활용)

  • 필요한 도구를 자율적으로 선택
  • 작업에 맞는 권한 관리
  • 효율적인 리소스 활용

3. Planning (계획 수립)

  • 작업 순서 자율 결정
  • 우선순위 동적 조정
  • 의존성 관계 파악

4. Multi-agent Collaboration (다중 에이전트 협업)

  • CTO, 개발자, 테스터 등 역할 분담
  • 단일 LLM이 여러 전문가 역할 수행
  • 실제 개발팀처럼 상호 피드백

MCP 통합 {#mcp-통합}

MCP 서버 타입 {#mcp-서버-타입}

  • stdio: 표준 입출력을 통한 로컬 프로세스 통신
  • SSE (Server-Sent Events): HTTP 기반 실시간 이벤트 스트림
  • HTTP: REST API 스타일의 통신

서버 스코프 레벨 {#서버-스코프-레벨}

  1. Local: 개인용, 프로젝트별 구성
  2. Project: .mcp.json 파일을 통한 팀 공유 구성
  3. User: 프로젝트 간 접근 가능한 서버

FAQ 및 트러블슈팅 {#faq-및-트러블슈팅}

일반적인 질문 {#일반적인-질문}

Q: Claude Code와 GitHub Copilot의 가장 큰 차이점은 무엇인가요? A: Claude Code는 파일을 직접 읽고 쓸 수 있으며, Agent 시스템을 통해 여러 전문가가 협업하는 방식으로 작동합니다. Copilot은 코드 완성에 특화되어 있지만, Claude Code는 전체 프로젝트 라이프사이클을 관리할 수 있습니다.

Q: Agent를 몇 개까지 만들 수 있나요? A: 이론적으로는 제한이 없지만, 실제로는 5-7개 이내가 관리하기 좋습니다. 너무 많으면 역할이 중복되거나 대화가 복잡해질 수 있어요.

Q: Sub-Agent 설정을 다른 프로젝트에서도 사용할 수 있나요? A: Personal Agent는 모든 프로젝트에서 사용 가능하고, Project Agent는 해당 프로젝트에서만 사용할 수 있습니다. .claude/agents/ 폴더를 복사해서 다른 프로젝트에 적용할 수도 있어요.

기술적 질문 {#기술적-질문}

Q: Agent 간에 데이터를 공유할 수 있나요? A: 네! Agent들은 같은 프로젝트 디렉토리를 공유하므로, 파일을 통해 데이터를 전달할 수 있습니다. 또한 한 Agent가 작업한 내용을 다른 Agent가 이어받아 수정하거나 확장할 수 있어요.

Q: API 비용이 많이 나오나요? A: Agent를 사용하면 일반적인 대화보다 더 많은 토큰을 사용합니다. 하지만 작업 효율성이 크게 향상되기 때문에 비용 대비 효과는 충분합니다. Claude Pro 또는 API 사용량을 모니터링하며 사용하세요.

활용 팁 {#활용-팁}

Q: Agent 이름을 어떻게 짓는 게 좋을까요? A: 짧고 기억하기 쉬운 이름이 좋습니다. 예: "민호", "준호", "진수" 등. "코드작성전문가민호"처럼 긴 이름은 매번 입력하기 번거롭습니다. 또한 "민호야", "준호야" 등 친근한 호칭을 사용하면 더 자연스럽게 사용할 수 있어요.

Q: 한 번에 여러 Agent를 동시에 호출할 수 있나요? A: 아니요, Claude Code는 한 번에 하나의 Agent만 활성화됩니다. 하지만 순차적으로 여러 Agent를 호출해서 마치 팀이 협업하는 것처럼 사용할 수 있습니다.

🔧 트러블슈팅 {#트러블슈팅}

Agent 생성 관련

  • "Agent not found": /agents list로 Agent 목록 확인, 정확한 이름 입력
  • "Tool access denied": Agent 생성 시 필요한 도구 권한 누락, Agent 삭제 후 재생성 필요
  • "Multiple agents with same name": 자동으로 숫자 추가됨 (민호2, 민호3)

성능 관련

  • 응답 속도 저하: 너무 많은 도구 권한 제거, Sub-Agent별 역할 명확히 분리
  • 메모리 사용량 증가: 불필요한 Agent 삭제, 컨텍스트 정리

🎯 최종 요약 {#최종-요약}

Claude Code Sub-Agents는 개발 워크플로우를 혁신적으로 개선할 수 있는 강력한 도구입니다. 전문화된 AI 어시스턴트를 통해 작업을 효율적으로 분배하고, 컨텍스트를 보존하며, 팀 전체의 생산성을 향상시킬 수 있습니다.

핵심 요점 {#핵심-요점}

  • 전문화: 각 Agent가 특정 영역에 특화되어 높은 품질의 결과 제공
  • 협업: 실제 개발팀처럼 여러 Agent가 순차적으로 협업
  • 효율성: 컨텍스트 보존과 역할 분담으로 전체적인 효율성 향상
  • 유연성: 프로젝트와 팀의 요구사항에 맞게 커스터마이징 가능

이 가이드를 통해 Sub-Agent 시스템을 최대한 활용하여 개발 프로세스를 최적화하세요!
