Plain Text

🌈 5가지 색깔로 기억하기 {#5가지-색깔로-기억하기}

Plain Text 
1xx (파란색 💙) = Information (정보)
   → "잠깐만요, 처리 중이에요"


2xx (초록색 💚) = Success (성공)
   → "완벽해요! 성공했어요!"


3xx (노란색 💛) = Redirect (리다이렉션)
   → "저기로 가세요! 길 안내해줄게요"


4xx (주황색 🧡) = Client Error (클라이언트 에러)
   → "이런, 당신이 뭔가 잘못했어요"


5xx (빨간색 ❤️) = Server Error (서버 에러)
   → "죄송해요, 저희가 고장났어요"
10 lines

💼 비즈니스에게 {#비즈니스에게}

  • 고객 지원 비용 감소: 정확한 에러 메시지 → 빠른 문제 해결

🤔 이게 뭐예요? {#이게-뭐예요}

100 Continue - 계속하세요

언제 사용하나요?

  • 큰 파일을 업로드할 때
  • 서버가 "계속 보내도 돼요!"라고 확인해줄 때

실생활 비유:

Plain Text 
당신: "100MB 영상 파일 보내도 될까요?" 🎥
서버: "100 Continue - 네, 보내세요!" ✅


당신: "그럼 파일 전송 시작합니다!" 📤
서버: "200 OK - 잘 받았어요!" ✅
4 lines

예제 코드:

JavaScript 
// 큰 파일 업로드 전 확인
fetch('/api/upload/large-video', {
  method: 'POST',
  headers: {
    'Expect': '100-continue',
    'Content-Length': '104857600' // 100MB
  }
})
.then(response => {
  if (response.status === 100) {
    console.log("서버가 파일을 받을 준비가 됐어요!");
    // 실제 파일 전송 시작
  }
});

HTTP 관련 {#http-관련}

HTTP (HyperText Transfer Protocol)

웹 브라우저와 서버가 대화하는 규칙이에요. 마치 외국인과 대화할 때 영어를 쓰는 것처럼, 컴퓨터들이 서로 이해하려면 HTTP라는 공통 언어를 사용해요.

Status Code (상태 코드)

서버가 브라우저에게 보내는 "결과 번호표"예요.

  • 200번: "성공했어요!"
  • 404번: "못 찾았어요!"
  • 500번: "고장났어요!"

Request (요청)

브라우저가 서버에게 "이거 주세요!"라고 부탁하는 것

  • 예: "ID 123번 사용자 정보 주세요"

Response (응답)

서버가 브라우저에게 "여기 있어요!" 또는 "없어요!"라고 대답하는 것

  • 상태 코드 + 데이터

Endpoint (엔드포인트)

API의 "주소"예요.

  • /api/users = 사용자 목록 주소
  • /api/posts/123 = 123번 게시글 주소

인증/권한 관련 {#인증권한-관련}

Authentication (인증)

"당신이 누구인지" 확인하는 것

  • 로그인 = 인증
  • 비유: 신분증 제시

Authorization (권한)

"당신이 이것을 할 수 있는지" 확인하는 것

  • 관리자 권한 확인 = 권한
  • 비유: VIP 카드 확인

Token (토큰)

로그인 후 받는 "입장권" 같은 것

  • JWT (JSON Web Token)가 가장 흔함
  • 매 요청마다 토큰을 함께 보냄

Bearer Token

토큰 앞에 "Bearer"를 붙여서 보내는 방식

Plain Text 
> Authorization: Bearer eyJhbGci0iJIUzI1NiIs...
>

API 설계 관련 {#api-설계-관련}

REST API

웹 서비스를 만드는 일종의 "규칙"

  • URL로 자원을 표현
  • HTTP 메서드로 행동을 표현

CRUD

데이터의 4가지 기본 작업

  • Create (생성) → POST
  • Read (읽기) → GET
  • Update (수정) → PUT/PATCH
  • Delete (삭제) → DELETE

Idempotent (멱등성)

같은 요청을 여러 번 해도 결과가 같은 것

  • GET: 멱등 (여러 번 조회해도 같음)
  • DELETE: 멱등 (이미 삭제되면 계속 없음)
  • POST: 비멱등 (매번 새로 생성됨)

네트워크 관련 {#네트워크-관련}

CORS (Cross-Origin Resource Sharing)

다른 도메인의 리소스를 안전하게 사용하는 방법

  • 브라우저의 보안 정책
  • 서버에서 허용해줘야 함

Cache (캐시)

자주 쓰는 데이터를 미리 저장해두는 것

  • 304 Not Modified가 캐시 관련

Proxy (프록시)

중간에서 요청을 대신 전달해주는 서버

  • 502, 504가 프록시 관련 에러

Timeout (타임아웃)

너무 오래 기다리면 포기하는 것

  • 기본: 30초~60초

보안 관련 {#보안-관련}

Rate Limiting (속도 제한)

너무 많은 요청을 막는 것

  • 429 Too Many Requests
  • DDoS 공격 방어

HTTPS

HTTP의 보안 버전

  • 데이터 암호화
  • 중간에서 훔쳐보기 불가능

XSS (Cross-Site Scripting)

악의적인 스크립트를 주입하는 공격

  • 입력 값 검증으로 방어

SQL Injection

데이터베이스에 악의적인 쿼리를 주입하는 공격

  • 파라미터 바인딩으로 방어
