🔤 기술 용어 {#기술-용어}

A-E

API (Application Programming Interface)

Plain Text 
초급: 프로그램들이 서로 대화하는 방법
중급: 소프트웨어 간 통신을 위한 규칙과 정의
고급: 특정 기능이나 데이터에 접근하기 위한
      프로토콜, 도구, 정의의 집합


예시:
날씨 앱이 기상청 서버에 "오늘 날씨 알려줘"라고 요청
→ API를 통해 데이터 받아옴
7 lines

대역폭 (Bandwidth)

Plain Text 
초급: 인터넷 속도
중급: 일정 시간 동안 전송 가능한 데이터양
고급: 네트워크에서 주어진 시간 동안 전송할 수 있는
      최대 데이터 전송률 (bps, Mbps)


비유:
수도관의 두께 = 대역폭
물의 양 = 데이터
7 lines

컨테이너 (Container)

Plain Text 
초급: 프로그램을 포장한 상자
중급: 애플리케이션과 모든 필요한 것을 포함한 패키지
고급: OS 수준의 가상화를 이용하여 격리된 환경에서
      애플리케이션을 실행하는 기술


Docker 컨테이너 = 가장 유명한 컨테이너 시스템
5 lines

도메인 (Domain)

Plain Text 
초급: 웹사이트 주소 (예: google.com)
중급: IP 주소를 사람이 읽기 쉬운 이름으로 변환
고급: 인터넷에서 컴퓨터나 네트워크를 식별하기 위한
      고유한 이름 체계


예시:
meet.jit.si → 도메인
64.233.177.113 → IP 주소 (컴퓨터는 이걸 씀)
7 lines

E2EE (End-to-End Encryption)

Plain Text 
초급: 나랑 상대방만 볼 수 있는 암호
중급: 데이터가 송신자에서 수신자까지 암호화된 상태 유지
고급: 통신 양 끝단에서만 암호화/복호화가 일어나며,
      중간 서버는 내용을 알 수 없는 암호화 방식


WhatsApp, Signal이 E2EE 사용

F-O

방화벽 (Firewall)

Plain Text 
초급: 나쁜 사람들이 못 들어오게 하는 문지기
중급: 네트워크 트래픽을 감시하고 제어하는 보안 시스템
고급: 미리 정의된 보안 규칙에 따라 들어오고 나가는
      네트워크 트래픽을 모니터링하고 제어하는 시스템


집의 현관문 = 방화벽
초인종 누르면 확인하고 문 열어줌

HTTPS (HTTP Secure)

Plain Text 
초급: 안전한 웹사이트 (자물쇠 표시)
중급: 암호화된 HTTP 통신
고급: SSL/TLS 프로토콜을 사용하여 데이터를
      암호화하는 HTTP의 보안 버전


주소창에 자물쇠 🔒 = HTTPS
5 lines

IP 주소 (IP Address)

Plain Text 
초급: 인터넷상의 집 주소
중급: 네트워크에 연결된 기기의 고유 식별자
고급: 인터넷 프로토콜을 사용하는 네트워크에서
      기기를 식별하기 위한 숫자 주소


예시:
IPv4: 192.168.0.1 (옛날 방식)
IPv6: 2001:0db8:85a3::8a2e:0370:7334 (최신)
7 lines

지연시간 (Latency)

Plain Text 
초급: 말하고 나서 상대방이 듣기까지 걸리는 시간
중급: 데이터 전송 요청부터 응답까지의 시간
고급: 패킷이 출발지에서 목적지까지 도달하는 데
      걸리는 시간 (RTT - Round Trip Time)


낮을수록 좋음!
게임: <50ms (매우 좋음)
화상회의: <150ms (괜찮음)
7 lines

오픈소스 (Open Source)

Plain Text 
초급: 소스 코드를 누구나 볼 수 있는 프로그램
중급: 코드가 공개되어 자유롭게 사용, 수정, 배포 가능
고급: 특정 라이선스 하에 소스 코드를 공개하여
      협업적 개발과 투명성을 촉진하는 소프트웨어


Jitsi Meet = 오픈소스!
Windows = 오픈소스 아님 (proprietary)
6 lines

P-Z

P2P (Peer-to-Peer)

Plain Text 
초급: 서버 거치지 않고 직접 연결
중급: 중앙 서버 없이 기기 간 직접 통신
고급: 분산 네트워크에서 각 노드가 동등한 위치에서
      자원을 공유하는 통신 방식


비유:
일반 방식: A → 서버 → B (우체국 경유)
P2P: A ↔ B (직접 전달)


장점: 빠르고, 서버 부하 적음
단점: 방화벽 문제 가능
9 lines

포트 (Port)

Plain Text 
초급: 집의 여러 개 문 (현관, 뒷문...)
중급: 특정 프로그램으로 데이터를 보내는 번호
고급: TCP/IP 네트워크에서 프로세스를 구분하기 위한
      논리적 연결 지점 (0-65535)


예시:
HTTP: 80번 포트
HTTPS: 443번 포트
Jitsi: 443, 10000 포트 등
8 lines

SSL/TLS

Plain Text 
초급: 인터넷을 안전하게 만드는 기술
