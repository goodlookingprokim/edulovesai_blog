add-dir 명령어 사용 가이드

주요 사용 시점 {#주요-사용-시점}

1. 새 프로젝트 폴더 추가 {#1-새-프로젝트-폴더-추가}

  • 기존 작업 중인 프로젝트에 관련 폴더를 추가할 때
  • 멀티 프로젝트 환경에서 다른 프로젝트 폴더 접근이 필요할 때

2. 외부 라이브러리/모듈 분석 {#2-외부-라이브러리모듈-분석}

  • node_modules, vendor 폴더 등 의존성 코드 분석 필요 시
  • 서브모듈이나 외부 코드 참조가 필요할 때

3. 프로젝트 구조 확장 {#3-프로젝트-구조-확장}

  • 초기에 작은 범위로 시작했지만 더 넓은 컨텍스트가 필요할 때
  • 관련 설정 파일들이 있는 상위/하위 디렉토리 접근 시

4. 협업 프로젝트 {#4-협업-프로젝트}

  • 팀 공유 폴더나 공통 라이브러리 접근 시
  • 문서화 폴더나 리소스 폴더 추가 시

5. 문제 해결 시 {#5-문제-해결-시}

  • 에러 원인 분석을 위해 더 많은 파일 접근이 필요할 때
  • 설정 파일이나 로그 파일이 다른 디렉토리에 있을 때

실제 사용 예시 {#실제-사용-예시}

1. React 프로젝트에서 백엔드 폴더 추가 {#1-react-프로젝트에서-백엔드-폴더-추가}

Bash 
/add-dir ../backend
1 line
**상황**: 프론트엔드 작업 중 API 연동 오류 발생 **이유**: 백엔드 API 스펙을 확인하고 프론트엔드 코드를 수정해야 함

2. 모노레포 환경에서 공유 라이브러리 접근 {#2-모노레포-환경에서-공유-라이브러리-접근}

Bash 
/add-dir ../../shared/components
/add-dir ../../shared/utils
2 lines
**상황**: 여러 앱이 공통 컴포넌트를 사용하는 환경 **이유**: 공유 컴포넌트 업데이트가 현재 앱에 미치는 영향 분석 필요

3. 설정 파일 문제 해결 {#3-설정-파일-문제-해결}

Bash 
/add-dir ~/.config/myapp
/add-dir /etc/myapp
2 lines
**상황**: 앱이 제대로 실행되지 않음 **이유**: 사용자별/시스템 설정 파일 확인 및 수정 필요

4. 외부 라이브러리 소스 분석 {#4-외부-라이브러리-소스-분석}

Bash 
/add-dir node_modules/react
/add-dir node_modules/@types
2 lines
**상황**: TypeScript 타입 오류나 라이브러리 버그 의심 **이유**: 실제 라이브러리 소스코드와 타입 정의 확인 필요

5. 문서화 폴더 추가 {#5-문서화-폴더-추가}

Bash 
/add-dir ../docs
/add-dir ../examples
2 lines
**상황**: 새로운 기능 개발 중 **이유**: 기존 문서와 예제 코드를 참고하여 일관성 있는 구현 필요

6. 배포 관련 폴더 접근 {#6-배포-관련-폴더-접근}

Bash 
/add-dir ../deployment
/add-dir ../scripts
2 lines
**상황**: 프로덕션 배포 준비 **이유**: 배포 스크립트 수정이나 환경 설정 업데이트 필요

add-dir의 실제 동작 방식 {#add-dir의-실제-동작-방식}

❌ 하지 않는 것: {#하지-않는-것}

  • 새로운 폴더/디렉토리 생성
  • 파일 시스템 구조 변경
  • 물리적인 폴더 생성

✅ 실제로 하는 것: {#실제로-하는-것}

  • 기존 폴더를 Claude Code가 읽고 분석할 수 있도록 권한 부여
  • Claude Code의 작업 컨텍스트에 해당 폴더 포함
  • 해당 폴더의 파일들을 검색하고 편집할 수 있게 함

예시로 설명 {#예시로-설명}

Bash 
/add-dir ../backend
1 line

이 명령어는:

  • ../backend 폴더가 이미 존재한다고 가정
  • 해당 폴더를 Claude Code가 접근할 수 있는 범위에 추가
  • 이제 Claude Code가 backend 폴더의 파일들을 읽고 분석 가능

만약 ../backend 폴더가 존재하지 않으면 오류가 발생합니다.

요약 {#요약}

add-dir 명령어는 **"폴더 생성"이 아니라 "기존 폴더에 대한 접근 권한 추가"**입니다. 현재 작업 디렉토리 외부의 관련 코드나 설정에 접근해야 할 때 사용하며, Claude Code가 접근할 수 있는 범위를 확장하여 더 포괄적인 코드 분석과 작업을 가능하게 합니다.
