4-3. Vim 모드 사용 가이드

개요 {#개요}

Claude Code의 Vim 모드는 Vim 에디터에 익숙한 사용자들을 위한 키 바인딩 모드입니다. 텍스트 편집을 키보드만으로 효율적으로 할 수 있게 해줍니다.

Vim 모드 활성화 방법 {#vim-모드-활성화-방법}

설정에서 활성화 {#설정에서-활성화}

Plain Text 
/config 명령어 실행 → Vim mode 옵션 활성화
1 line

기본 Vim 키 바인딩 {#기본-vim-키-바인딩}

모드 전환 {#모드-전환}

Plain Text 
- ESC: Normal 모드로 전환
- i: Insert 모드 (입력 모드)
- a: 커서 뒤에서 Insert 모드
- o: 새 줄 생성 후 Insert 모드
4 lines

커서 이동 {#커서-이동}

Plain Text 
- h,j,k,l: 좌,하,상,우 이동
- w,b: 단어 단위 앞/뒤 이동
- 0,$: 줄 시작/끝으로 이동
- gg: 문서 시작으로 이동
- G: 문서 끝으로 이동
5 lines

편집 명령어 {#편집-명령어}

Plain Text 
- dd: 현재 줄 삭제
- yy: 현재 줄 복사
- p: 붙여넣기
- x: 현재 문자 삭제
- u: 실행 취소 (Undo)
5 lines

Claude Code 전용 Vim 명령어 {#claude-code-전용-vim-명령어}

특수 명령어 {#특수-명령어}

Plain Text 
- :q: 현재 대화 종료
- :w: 입력 내용 전송 (Enter와 동일)
- :wq: 전송 후 종료
3 lines

실제 사용 예시 {#실제-사용-예시}

예시 1: 긴 코드 편집 {#예시-1-긴-코드-편집}

Plain Text 
1. Vim 모드에서 코드 입력
2. ESC → Normal 모드로 전환
3. dd로 잘못된 줄 삭제
4. i로 다시 Insert 모드
5. :w로 전송
5 lines

예시 2: 빠른 수정 {#예시-2-빠른-수정}

Plain Text 
문제: 타이핑 실수로 중간에 오타 발견
해결: ESC → w로 단어 이동 → x로 문자 삭제 → i로 수정
2 lines

예시 3: 블록 편집 {#예시-3-블록-편집}

Plain Text 
1. 여러 줄 선택이 필요한 경우
2. V (Visual Line 모드)로 줄 단위 선택
3. yy로 복사 후 p로 붙여넣기
3 lines

일반 모드 vs Vim 모드 비교 {#일반-모드-vs-vim-모드-비교}

일반적인 요청시 Vim 모드의 단점 {#일반적인-요청시-vim-모드의-단점}

불편한 이유

  1. 학습 곡선

    • ESC, i, h,j,k,l 키 조합 외워야 함
    • 실수로 명령어 모드에서 타이핑하면 이상한 동작
    • Vim 초보자는 계속 헤맴

  2. 일반 질문에는 오버스펙

    • "오늘 날씨 어때?" 같은 간단한 질문
    • "이 코드 설명해줘" 같은 일회성 요청
    • 채팅하듯 대화하기에는 부적합

  3. 마우스 사용 제한

    • 클릭으로 커서 이동 안 됨
    • 드래그로 텍스트 선택 제한
    • 스크롤 동작도 다름

Vim 모드 사용 권장 상황 {#vim-모드-사용-권장-상황}

✅ 사용해야 하는 구체적인 상황들 {#사용해야-하는-구체적인-상황들}

1. 대용량 코드 파일 편집

Plain Text 
상황: 500줄 이상의 긴 코드를 Claude에게 보낼 때
예시: 전체 React 컴포넌트 파일 수정 요청
Vim 장점: dd, yy, p로 빠른 줄 단위 편집
3 lines

2. 반복적인 텍스트 조작

Plain Text 
상황: 여러 줄에서 동일한 패턴 수정
예시: 10개 함수명을 일괄 변경
Vim 장점: :%s/old/new/g 같은 일괄 치환
3 lines

3. 정밀한 코드 리팩토링 설명

Plain Text 
상황: 코드의 특정 부분만 지적하며 수정 요청
예시: "35번째 줄 if문을 switch문으로 바꿔줘"
Vim 장점: :35로 정확한 줄 번호 이동
3 lines

4. 복잡한 정규식이나 설정 파일

Plain Text 
상황: nginx.conf, .vimrc 같은 설정 파일 편집
예시: 여러 블록을 복사하고 수정
Vim 장점: 블록 단위 선택과 이동이 편함
3 lines

5. 키보드만 사용해야 하는 환경

Plain Text 
상황: 마우스가 없거나 터미널 작업 중
예시: SSH로 원격 서버에서 작업
Vim 장점: 모든 조작을 키보드로 가능
3 lines

❌ 사용하지 말아야 할 상황 {#사용하지-말아야-할-상황}

Plain Text 
- "안녕, 오늘 뭐해?" 같은 일상 대화
- "이 에러 뭐야?" 같은 간단한 질문  
- 10줄 미만의 짧은 코드 snippets
- 처음 Claude Code 사용하는 경우
- 마우스로 편리하게 작업할 수 있는 환경
5 lines

사용 권장 기준 {#사용-권장-기준}

핵심 판단 기준 {#핵심-판단-기준}

텍스트 길이 > 100줄 + Vim 숙련도 높음 = Vim 모드 ON

그 외 모든 경우 = 일반 모드가 편함

사용자별 추천 {#사용자별-추천}

  • 일반 사용자: Vim 모드 OFF (질문하기, 채팅, 간단한 코드 요청 등)
  • Vim 고수: Vim 모드 ON (복잡한 코드 편집, 대량 텍스트 작업 등)

결론 {#결론}

90% 사용자는 일반 모드가 훨씬 편합니다.

Vim에 익숙하지 않다면 일반 모드를 사용하는 것을 강력히 권장합니다. Vim 모드는 이미 Vim 에디터에 숙련된 개발자들이 대용량 텍스트 편집 작업을 할 때만 사용하는 것이 좋습니다.
