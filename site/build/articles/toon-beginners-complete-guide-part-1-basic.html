🤔 TOON이 뭔가요? {#toon이-뭔가요}

한 문장 설명 {#한-문장-설명}

TOON(Token-Oriented Object Notation) 은 AI에게 데이터를 더 효율적으로 전달하기 위한 새로운 데이터 포맷입니다.

조금 더 자세히 {#조금-더-자세히}

컴퓨터가 이해할 수 있는 데이터를 만들 때, 어떤 "언어"를 사용하는지가 중요합니다. 지금까지는 JSON이라는 형식을 많이 사용했는데, TOON은 JSON의 좋은 점은 유지하면서 더 간결하고 효율적으로 만들었습니다.

💡 왜 TOON을 배워야 하나요? {#왜-toon을-배워야-하나요}

문제 상황: JSON의 한계 {#문제-상황-json의-한계}

AI(특히 ChatGPT, Claude 같은 LLM)에게 데이터를 전달할 때, 데이터가 차지하는 토큰(Token) 개수가 중요합니다. 왜냐하면:

  1. 돈이 든다: AI API를 사용할 때 토큰 수에 따라 요금이 결정됨
  2. 속도가 느려진다: 토큰이 많을수록 처리 시간이 오래 걸림
  3. 정확도가 떨어진다: 불필요한 정보가 많으면 AI가 헷갈림

TOON의 해결책 {#toon의-해결책}

TOON은 같은 정보를 30~60% 더 적은 토큰으로 표현할 수 있습니다.

예시: 100명의 직원 정보를 저장하는 경우

  • JSON 형식: 5,000 토큰 필요
  • TOON 형식: 2,000 토큰만 필요 (60% 절감!)

📖 일상 비유로 이해하는 TOON {#일상-비유로-이해하는-toon}

비유 1: 택배 배송 문장 {#비유-1-택배-배송-문장}

JSON 방식 (현재 방식):

Plain Text 
"우리는" "상자에" "책을" "포장했고" "상자에" "책을" "넣었고" "상자에"
"책을" "배송했습니다"
2 lines

TOON 방식 (새로운 방식):

Plain Text 
상자[3]: 책, 책, 책
배송: 완료
2 lines

같은 정보인데 훨씬 짧죠? 이게 바로 TOON이 하는 일입니다.

비유 2: 엑셀 시트 {#비유-2-엑셀-시트}

사무직 직원들의 정보를 저장한다고 생각해봅시다:

JSON 방식 - 이렇게 저장:

Plain Text 
{
  "직원": {
    "이름": "김철수",
    "직급": "부장",
    "부서": "개발팀"
  }
},
{
  "직원": {
    "이름": "이영희",
    "직급": "대리",
    "부서": "디자인팀"
  }
}
14 lines

TOON 방식 - 엑셀처럼 저장:

Plain Text 
직원[2]{이름,직급,부서}:
  김철수,부장,개발팀
  이영희,대리,디자인팀
3 lines

마치 엑셀 시트처럼 정렬되어 있어서 더 이해하기 쉽고 짧습니다!

비유 3: 편지와 표 {#비유-3-편지와-표}

편지 형식 (JSON)

Plain Text 
"안녕하세요, 제 이름은 김철수입니다.
저는 개발팀에서 부장으로 일하고 있습니다.
제 이메일은 kim@company.com입니다."
3 lines

표 형식 (TOON)

Plain Text 
이름: 김철수
부서: 개발팀
직급: 부장
이메일: kim@company.com
4 lines

똑같은 정보인데 표가 더 깔끔하고 읽기 쉬운 것처럼, TOON도 정보를 더 체계적으로 정렬합니다.

📊 JSON과 TOON의 차이 {#json과-toon의-차이}

JSON이란? {#json이란}

JSON은 현재 가장 많이 사용되는 데이터 포맷입니다. 많은 프로그래밍 언어가 JSON을 지원합니다.

문제점 {#문제점}

JSON은 너무 **장황(verbose)**합니다. 같은 내용을 여러 번 반복해서 적어야 합니다.

비교: 하이킹 데이터 예제 {#비교-하이킹-데이터-예제}

JSON 방식 (길고 비효율적)

JSON 
{
  "context": {
    "task": "우리가 함께 한 산행들",
    "location": "서울",
    "season": "봄 2025"
  },
  "friends": ["이순신", "세종대왕", "을지문덕"],
  "hikes": [
    {
      "id": 1,
      "name": "남산 트래킹",
      "distanceKm": 7.5,
      "elevationGain": 320,
      "companion": "이순신",
      "wasSunny": true
    },
    {
      "id": 2,
      "name": "북한산 등반",
      "distanceKm": 9.2,
      "elevationGain": 540,
      "companion": "세종대왕",
      "wasSunny": false
    }
  ]
}
26 lines

토큰 수: 약 400개

TOON 방식 (짧고 효율적)

Code 
context:
  task: 우리가 함께 한 산행들
  location: 서울
  season: 봄 2025


friends[3]: 이순신,세종대왕,을지문덕


hikes[2]{id,name,distanceKm,elevationGain,companion,wasSunny}:
  1,남산 트래킹,7.5,320,이순신,true
  2,북한산 등반,9.2,540,세종대왕,false
8 lines

토큰 수: 약 240개 (40% 절감!)

🎨 TOON의 핵심 개념 {#toon의-핵심-개념}

1️⃣ 핵심 개념: YAML + CSV의 만남 {#1-핵심-개념-yaml-csv의-만남}

특징 YAML CSV TOON
읽기 쉬움 ✅ ✅ ✅
중첩 구조 ✅ ❌ ✅
효율성 ❌ ✅ ✅
데이터 유효성 ❌ ❌ ✅

TOON = YAML의 읽기 쉬움 + CSV의 효율성 + 데이터 유효성 검증

2️⃣ TOON의 세 가지 구성 요소 {#2-toon의-세-가지-구성-요소}

① 객체 (Object) - 속성을 가진 데이터

Code 
이름: 김철수
나이: 30
직급: 부장
3 lines

일상 비유: 주민등록증 (한 사람의 정보)

② 배열 (Array) - 여러 항목의 목록

Code 
친구들[3]: 이순신,세종대왕,을지문덕
1 line

일상 비유: 장을 보는 쇼핑 목록

③ 표 (Tabular Array) - 같은 구조의 항목들

Code 
직원[2]{이름,부서,직급}:
  김철수,개발팀,부장
  이영희,디자인팀,대리
3 lines

일상 비유: 학생 성적표 (모두 같은 항목들)

3️⃣ TOON의 문법 규칙 (3가지만 기억하세요!) {#3-toon의-문법-규칙-3가지만-기억하세요}

규칙 1: 객체는 들여쓰기로 표현

Code 
사람:
  이름: 김철수
  직급: 부장
3 lines

들여쓰기란: 줄의 앞에 스페이스를 추가하는 것

규칙 2: 배열은 [개수] 형식

Code 
친구들[3]: 친구1,친구2,친구3
1 line

괄호 안 숫자는: 몇 개인지 표시

규칙 3: 표는 헤더와 데이터 행으로 표현

Code 
직원[2]{이름,부서}:
  김철수,개발팀
  이영희,디자인팀
3 lines

중괄호 안: 항목의 이름들 (헤더) 콜론 뒤: 데이터 행들

👶 첫 번째 TOON 예제 {#첫-번째-toon-예제}

예제 1: 나의 정보 (초급) {#예제-1-나의-정보-초급}

목적: 자신의 프로필을 TOON으로 작성해봅시다.

❌ JSON 방식

JSON 
{
  "name": "김철수",
  "age": 30,
  "city": "서울"
}
5 lines

✅ TOON 방식

Code 
name: 김철수
age: 30
city: 서울
3 lines

설명:

  • : (콜론) 앞은 항목 이름
  • 콜론 뒤는 값(데이터)

예제 2: 학교 반 정보 (초급) {#예제-2-학교-반-정보-초급}

목적: 학반의 학생 목록을 작성합니다.

Code 
반: 3학년 2반
학급담임: 김선생님


학생[4]: 김철수,이영희,박민준,최지현
3 lines

설명:

  • 학생[4]는 "4명의 학생이 있어요"라는 뜻
  • 이름들은 쉼표(,)로 분리

예제 3: 학생 성적표 (중급) {#예제-3-학생-성적표-중급}

목적: 여러 학생의 성적을 정렬합니다.

Code 
학생[3]{이름,국어,수학,영어}:
  김철수,90,85,88
  이영희,95,92,90
  박민준,88,90,92
4 lines

설명:

  • [3]은 학생 3명
  • {이름,국어,수학,영어}는 4개의 항목
  • 콜론 뒤에는 각 학생의 성적을 쉼표로 구분

시각화:

이름 국어 수학 영어
김철수 90 85 88
이영희 95 92 90
박민준 88 90 92

예제 4: 영화 정보 (중급) {#예제-4-영화-정보-중급}

목적: 여러 영화의 정보를 저장합니다.

Code 
영화[2]{제목,감독,개봉년도,평점}:
  기생충,봉준호,2019,8.5
  아바타,제임스 카메론,2009,7.8
3 lines

설명:

  • 각 행은 하나의 영화
  • 모든 영화는 같은 정보를 가짐 (제목, 감독, 연도, 평점)

⚠️ TOON을 사용하지 말아야 할 경우 {#toon을-사용하지-말아야-할-경우}

중요: TOON이 만능은 아닙니다! {#중요-toon이-만능은-아닙니다}

TOON은 매우 효율적이지만, 모든 상황에 적합한 것은 아닙니다. 어떤 경우에는 JSON이나 다른 형식이 더 나을 수 있습니다.

상황 1: 깊게 중첩된 데이터 {#상황-1-깊게-중첩된-데이터}

예: 설정 파일, 복잡한 계층 구조

JSON 
{
  "database": {
    "connection": {
      "host": {
        "primary": "localhost",
        "fallback": "127.0.0.1"
      }
    }
  }
}
10 lines

❌ TOON으로 하면 복잡함:

Code 
database:
  connection:
    host:
      primary: localhost
      fallback: 127.0.0.1
5 lines

✅ JSON이 더 나음

상황 2: 비균일한 배열 (다른 구조의 항목들) {#상황-2-비균일한-배열-다른-구조의-항목들}

예: 여러 종류의 이벤트

Code 
events[3]:
  - type: "로그인"
    user: "김철수"
    timestamp: "2025-11-18"
  - type: "에러"
    error_code: 500
    message: "서버 오류"
  - type: "알림"
    content: "업데이트 완료"
9 lines

❌ TOON의 효율성이 떨어짐

✅ JSON이 더 나음

상황 3: 매우 간단한 데이터 {#상황-3-매우-간단한-데이터}

Code 
이름: 김철수
1 line

분석:

  • JSON: {"이름":"김철수"} = 약 20 토큰
  • TOON: 이름: 김철수 = 약 10 토큰
  • 절감: 50%

❌ 이렇게 작은 데이터는 절감 효과가 미미함

상황 4: 레이턴시(응답 시간)가 최우선인 경우 {#상황-4-레이턴시응답-시간가-최우선인-경우}

상황: 매우 빠른 응답이 필요한 경우

Plain Text 
JSON: 5,000 토큰 → 2초 (빠름)
TOON: 2,000 토큰 → 2.5초 (느림)
2 lines

일부 로컬 모델(Ollama 등)에서는 토큰 수가 적어도 처리 시간이 더 오래 걸릴 수 있습니다.

⚠️ 항상 벤치마크하세요!

상황 5: 기존 JSON 도구와 호환성 필요 {#상황-5-기존-json-도구와-호환성-필요}

Plain Text 
상황: 이미 JSON을 처리하는 도구가 많이 있음
문제: TOON 변환으로 인한 추가 복잡성
2 lines

✅ JSON 유지하기

판단 기준표 {#판단-기준표}

상황 TOON JSON CSV
균일한 테이블 (100+행) ✅ 최고 ⭐ ⭐⭐
깊게 중첩된 구조 ❌ ✅ 최고 ❌
매우 간단한 데이터 ⭐ ✅ ⭐
비균일한 배열 ⭐ ✅ 최고 ❌
완전히 평탄한 표 ⭐⭐ ✅ ✅ 최고

💡 선택 가이드 {#선택-가이드}

TOON을 사용하세요:

  • ✅ 100개 이상의 균일한 행
  • ✅ 토큰 비용을 절감해야 함
  • ✅ AI에 데이터를 전달
  • ✅ 구조화된 검증이 필요

JSON을 사용하세요:

  • ✅ 깊게 중첩된 데이터
  • ✅ 비균일한 구조
  • ✅ 기존 도구 호환성 필요
  • ✅ 데이터가 매우 작음

CSV를 사용하세요:

  • ✅ 완전히 평탄한 테이블
  • ✅ 엑셀에서 열 것
  • ✅ 스프레드시트 도구 필요

🎯 핵심 메시지 {#핵심-메시지}

"올바른 도구를 올바른 상황에 사용하세요." TOON은 강력하지만 만능은 아닙니다. 항상 데이터 구조를 먼저 분석한 후 선택하세요!

📚 용어 사전 (Glossary) {#용어-사전-glossary}

A~Z {#az}

Array (배열)

  • 여러 항목을 순서대로 나열한 것
  • 예: 친구들[3]: 이순신, 세종대왕, 을지문덕
  • 쉽게: 쇼핑 목록처럼 항목들을 나열하는 것

Colon (콜론) - :

  • 항목 이름과 값을 구분하는 기호
  • 예: 이름: 김철수
  • 콜론 왼쪽은 "무엇을", 오른쪽은 "값"

CSV (Comma-Separated Values)

  • 쉼표로 구분된 값들
  • 엑셀에서 저장할 수 있는 형식
  • 예: 김철수,부장,개발팀

Delimiter (구분자)

  • 데이터를 나누는 기호
  • TOON에서는 주로 쉼표(,)를 사용
  • 다른 구분자: 탭, 파이프(|)

Field (필드)

  • 데이터의 한 가지 항목
  • 표의 "열"과 같은 개념
  • 예: "이름", "나이", "직급"은 필드

Header (헤더)

  • 표의 맨 위 행에 적힌 항목 이름들
  • 예: {이름,나이,직급}

Indentation (들여쓰기)

  • 줄의 앞에 스페이스(빈 칸)를 추가하는 것
  • 계층 관계를 표현하는 데 사용
  • 예:
    Code 
    사람:
  이름: 김철수  ← 이 부분이 들여쓰기
  나이: 30
    4 lines

JSON (JavaScript Object Notation)

  • 현재 가장 인기 있는 데이터 포맷
  • 프로그래밍 언어들이 쉽게 읽을 수 있음
  • TOON보다 장황함 (많은 기호 사용)

Key (키)

  • 데이터의 이름 (항목 이름)
  • 예: 이름: 김철수에서 이름이 key

Nested (중첩된)

  • 데이터 안에 또 다른 데이터가 들어 있는 것
  • 예:
    Code 
    사람:
  이름: 김철수
  주소:
    시: 서울
    구: 강남구
    6 lines

Object (객체)

  • 여러 속성을 가진 하나의 데이터
  • 예:
    Code 
    사람:
  이름: 김철수
  나이: 30
    4 lines

Row (행)

  • 표에서 가로로 나열된 데이터
  • 한 줄의 정보
  • 예: 김철수,90,85,88는 한 행

Tabular (표 형식의)

  • 행과 열로 정렬된 표 모양
  • 엑셀 같은 형태
  • TOON의 가장 효율적인 부분

Token (토큰)

  • AI가 처리하는 데이터의 최소 단위
  • 단어나 부호 하나를 대략 1~2 토큰으로 계산
  • 많을수록 AI 처리 시간이 오래 걸리고 비용이 많이 듦

Value (값)

  • 데이터가 실제로 가진 내용
  • 예: 이름: 김철수에서 김철수가 value

YAML (YAML Ain't Markup Language)

  • 사람이 읽기 쉬운 데이터 포맷
  • 들여쓰기를 이용해 계층 관계 표현
  • 하지만 효율성은 낮음

🎯 Key Point 정리 {#key-point-정리}

🌟 TOON을 사용해야 하는 이유 3가지 {#toon을-사용해야-하는-이유-3가지}

이유 설명
💰 비용 절감 같은 정보를 30~60% 더 적게 표현
⚡ 속도 향상 토큰이 적으면 AI가 더 빨리 답함
🎯 정확도 향상 불필요한 정보가 적으면 AI가 더 정확히 답함

📖 TOON의 3가지 기본 형태 {#toon의-3가지-기본-형태}

  1. 객체: 한 가지 대상의 여러 속성

    Code 
    이름: 김철수
나이: 30
    3 lines

  2. 배열: 여러 항목의 목록

    Code 
    친구들[3]: 철수,영희,민준
    2 lines

  3. 표: 같은 구조의 여러 항목

    Code 
    학생[3]{이름,나이}:
  김철수,30
  이영희,28
  박민준,32
    5 lines
