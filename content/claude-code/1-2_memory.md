# `/memory` 명령어 사용 가이드

`/memory` 명령어는 다음과 같은 상황에서 사용합니다:

## 주요 사용 시점

### 1. 긴 프로젝트 작업 시
- 며칠에 걸친 개발 작업
- 복잡한 기능 구현 과정
- 단계별 진행 상황 기억 필요

### 2. 중요한 정보 저장
- 프로젝트 요구사항
- 설계 결정사항
- 버그 해결 방법
- 특별한 설정값들

### 3. 반복 작업 방지
- 같은 질문 재질문 방지
- 이미 논의된 내용 참조
- 결정된 사항 재확인

## 실제 사용 예시

### 예시 1: 프로젝트 설정 기억
```
사용자: /memory 이 프로젝트는 React + TypeScript + Tailwind CSS 조합이야
Claude: 프로젝트 기술 스택을 기억했습니다.
```

### 예시 2: 중요한 결정사항 저장
```
사용자: /memory 로그인은 JWT 방식으로, 상태관리는 Zustand 사용하기로 했어
Claude: 인증 방식과 상태관리 라이브러리 선택을 기억했습니다.
```

### 예시 3: 버그 해결법 저장
```
사용자: /memory CORS 에러는 proxy 설정으로 해결했어
Claude: CORS 문제 해결 방법을 기억했습니다.
```

## 언제 사용하면 좋을까?

### ✅ 사용 권장:
- 장기 프로젝트
- 복잡한 요구사항
- 팀 규칙이나 컨벤션
- 특별한 해결 방법

### ❌ 불필요한 경우:
- 일회성 간단한 질문
- 일반적인 프로그래밍 문법
- 공개된 문서 내용

## 📋 메모리 내용 확인 방법

### 1. **직접 확인 명령어**
```
/memory
```
- 현재 저장된 모든 메모리 내용 표시
- 가장 직접적인 방법

### 2. **질문을 통한 확인**
```
사용자: 이 프로젝트 기술 스택이 뭐였지?
Claude: 기억하고 있는 내용에 따르면 React + TypeScript + Tailwind CSS입니다.
```

### 3. **특정 주제별 확인**
```
사용자: 앞서 논의한 인증 방식이 뭐였어?
Claude: JWT 방식으로 하기로 결정했습니다.
```

### 실제 활용 예시

#### 프로젝트 재시작 시:
```
1. /memory                    # 전체 메모리 확인
2. "설정된 규칙이 뭐였지?"    # 구체적 질문
3. "사용하기로 한 라이브러리는?" # 세부사항 확인
```

#### 작업 중간에:
```
1. "이전에 어떤 방식으로 하기로 했지?"
2. "버그 해결 방법이 뭐였어?"
3. "정한 컨벤션이 있었나?"
```

## ⚠️ 메모리 관리 주의사항

### 메모리 손실 상황:
- `/clear` 사용 후 → 모든 메모리 삭제
- 새 세션 시작 → 이전 메모리 없음
- 너무 오래된 대화 → 메모리 제한

### 메모리 한계:
- 무제한 저장 불가
- 중요한 것만 선별 저장 필요
- 주기적 정리 권장

## 🔄 효과적인 메모리 관리 팁

### 사용 방법:
```
1. 중요 정보만 저장
2. 주기적으로 /memory로 확인
3. 불필요한 내용 정리
4. 새로운 결정사항 즉시 추가
```

### 백업 방법:
- 중요한 메모리 내용을 파일로 따로 저장
- 프로젝트 README나 문서에 기록
- 개발 일지에 정리

**나중에 다시 참조할 중요한 정보**를 `/memory`로 저장하고, **`/memory`만 입력해서 저장된 내용을 정기적으로 확인**하세요. 마치 개발 일지나 메모장처럼 활용하는 것입니다.
