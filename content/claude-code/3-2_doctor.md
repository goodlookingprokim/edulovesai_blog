# `/doctor` 명령어 사용 가이드

`/doctor` 명령어를 사용하는 상황들:

## 🚨 **문제 발생 시 첫 진단 도구**

### 1. **Claude Code 작동 오류**
```
- Claude Code가 실행되지 않을 때
- 응답이 느리거나 멈출 때
- 예상과 다르게 동작할 때
- 갑자기 기능이 안 될 때
```

### 2. **환경 설정 문제**
```
- 새로운 컴퓨터에서 설치 후
- Node.js나 Python 환경 변경 후
- 시스템 업데이트 후 문제 발생
- 패키지 설치/업데이트 후 오류
```

### 3. **네트워크/연결 문제**
```
- API 호출 실패
- "연결할 수 없습니다" 오류
- 타임아웃 에러 지속
- 인증 문제 발생
```

## 🔧 **정기적 시스템 점검**

### 주기적 사용:
- **주 1회**: 정기 건강 검진처럼
- **월 1회**: 전체 시스템 점검
- **대형 업데이트 후**: 환경 변경 시

## 💡 **실제 사용 시나리오**

### 시나리오 1: 갑작스런 오류
```
사용자: Claude Code가 갑자기 안 돼요
단계: /doctor → 문제 진단 → 해결 방법 확인
```

### 시나리오 2: 새 환경 설정
```
사용자: 새 컴퓨터에 Claude Code 설치했어요  
단계: /doctor → 환경 검증 → 누락된 설정 보완
```

### 시나리오 3: 성능 저하
```
사용자: 응답이 너무 느려졌어요
단계: /doctor → 성능 진단 → 최적화 제안 확인
```

## 🔍 **`/status`와의 차이점**

### `/status`: **현재 상태 확인**
- "지금 어떤 상태인가?"
- 기본 정보 표시

### `/doctor`: **문제 진단 및 해결**  
- "뭔가 문제가 있나?"
- 상세 분석 + 해결책 제시

## 🎯 **핵심 사용법**

**문제가 생겼을 때 가장 먼저 실행하는 명령어**
```
문제 발생 → /doctor → 진단 결과 확인 → 제안된 해결책 실행
```
## 핵심
**`/doctor`는 컴퓨터의 "문제 해결사" 역할을 하는 명령어입니다. 문제가 생겼을 때 가장 먼저 실행하여 무엇이 잘못되었는지 진단하고 해결 방법을 찾아줍니다.**
