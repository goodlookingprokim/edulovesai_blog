원본: $ARGUMENTS

## 진행

1. **분석**: 유형(코딩/개념/디버깅/설계/일반), 의도, 문제점 파악 후 추천 스타일 결정

2. **스타일 선택**: AskUserQuestion으로 선택받기. 추천에 "(추천)" 표시
   - 정밀형: 코딩/기술 (환경, 상황, 문제, 시도, 기대결과 명시)
   - 구조화형: 복잡한 질문 (배경→목표→질문→제약 구조)
   - 간결형: 단순 질문 (핵심만, 1-2문장)
   - 맥락형: 프로젝트 연계 (현재 컨텍스트 반영)
   - 다중 버전형: 간결/상세/구조화 3가지 동시 출력
   - 대화형: 목적 먼저 확인 후 맞춤 (AskUserQuestion 사용)

3. **개선 실행**: 선택된 스타일로 개선

4. **출력**:
   - 분석: 의도 | 문제점 | 적용 스타일
   - 개선된 질문 (한국어)
   - 개선 포인트
   - English 버전

5. **마무리**:
   - pbcopy로 영어 버전 클립보드 복사
   - AskUserQuestion: 영어로 실행 / 한국어로 실행 / 완료
