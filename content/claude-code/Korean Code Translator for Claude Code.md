Claude Code에서 한국어 입력을 감지하여 한국어로 응답하는 Output Style입니다.

## 기능

### 한국어 입력 감지
다음 조건에서 한국어로 응답합니다:
- 입력에 50% 이상 한국어 문자가 포함된 경우 (코드/기술 용어 제외)
- 한국어 동작 동사가 포함된 경우: 만들어줘, 고쳐줘, 설명해줘, 추가해줘, 삭제해줘, 테스트해줘, 배포해줘, 최적화해줘, 분석해줘
- 한국어 기술 요청: "컴포넌트 만들어", "API 연결해", "디버그해줘"

### 혼합 언어 지원
한국어와 영어가 섞인 입력에 대해 한국어로 응답하되 영어 용어는 보존합니다.

### 번역하지 않는 요소
- 코드 블록, 파일 경로, 함수명, 변수명
- 명령어, URL, 패키지명, git 명령어
- 환경변수, 설정 키
- 에러 메시지 (원본 표시 + 한국어 설명)

### 기술 용어 표기
`한국어(English)` 형식을 사용합니다:
- 컴포넌트(component), 훅(hook), 상태(state)
- 데이터베이스(database), 서버(server), API(API)
- 라우터(router), 미들웨어(middleware)

## 사용 예시

**혼합 언어 입력**:
- 입력: "디버그 info 보여줘"
- 응답: 디버그(debug) 정보를 보여드리겠습니다.

- 입력: "React component 만들어줘"
- 응답: React 컴포넌트(component)를 만들어드리겠습니다.

## 설치

1. 프로젝트에 `.claude/output-styles/` 폴더 생성
2. `korean-code-translator.md` 파일을 해당 폴더에 복사
3. Claude Code에서 output style 활성화

### 폴더 생성
Windows: `mkdir .claude\output-styles`
macOS/Linux: `mkdir -p .claude/output-styles`

## 한계점

### Output Style의 역할과 제약
- **언어 변경**: 응답 언어를 한국어로 변경 (✅ 가능)
- **지식 품질**: Claude의 기본 지식 품질 향상 (❌ 불가능)
- **훈련 데이터**: 언어별 훈련 데이터 차이 해결 (❌ 불가능)

### 언어별 응답 품질 차이
**영어가 더 나은 경우:**
- 복잡한 알고리즘 설명
- 최신 프레임워크 사용법
- 성능 최적화 기법
- 상세한 기술 문서

**한국어가 적절한 경우:**
- 기본 개념 설명
- 학습 가이드
- 일반적인 코딩 질문
- 에러 해결 과정

## 현실적 사용법

### 상황별 권장 사용법

**🎯 한국어로 질문하기 좋은 경우:**
```
- "Git 기본 사용법 알려줘"
- "React 컴포넌트가 뭐야?"
- "이 에러 메시지 뜻이 뭐야?"
- "프로그래밍 개념 설명해줘"
```

**🎯 영어로 질문하기 좋은 경우:**
```
- "Implement advanced React patterns with TypeScript"
- "Optimize performance for large datasets"
- "Debug complex async/await issues"
- "Latest best practices for..."
```

**🎯 하이브리드 접근법:**
```
1. 복잡한 기술 질문: 영어로 질문
2. 답변 이해 어려울 시: "위 내용을 한국어로 쉽게 설명해줘"
3. 기본 학습: 한국어로 질문
```

## 권장사항

### 이 도구를 사용하면 좋은 사용자
- 프로그래밍 입문자 또는 초보자
- 영어 기술 문서 읽기가 부담스러운 개발자
- 기본적인 개념 학습이 목적인 경우
- 빠른 참고용 질문이 필요한 경우

### 영어 사용을 고려해야 하는 경우
- 최신 기술이나 고급 주제
- 정확성이 매우 중요한 프로덕션 코드
- 복잡한 아키텍처 설계
- 성능 최적화나 디버깅

### 결론
Korean Code Translator는 **편의성과 접근성**을 제공하는 도구입니다. **완벽한 한국어 기술 지원**이 아닌 **한국어 사용자의 개발 경험 개선**을 목적으로 사용하는 것이 적절합니다.
