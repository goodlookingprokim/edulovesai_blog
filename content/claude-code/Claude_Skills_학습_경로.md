# Claude Skills 마스터 학습 경로

> **초보자에서 마스터까지**: 이 경로를 따라가면 Claude Skill을 완벽하게 만들 수 있습니다.

---

## 🎯 3가지 가이드와의 관계

```
당신의 수준                    추천 가이드                        학습 시간
─────────────────────────────────────────────────────────────────────
❌ "Skill이 뭐죠?" →  Claude_Skills_초보자_가이드 →  30분
                   (5살 버전부터 상세 설명)

❓ "어떻게 만들어?" →  clipping-processor_실전_분석 →  45분
                    (실제 예제로 상세 분석)

✅ "지금 만들고 싶어" → 나의_첫_Skill_만드는_템플릿 →  1-2시간
                    (바로 쓸 수 있는 템플릿)
```

### 🔴 Level 3: 나의 Skill 만들기 (Day 3-4)

#### 3-1. 구상 단계 (30분)

**준비:**
1. 자신이 반복적으로 하는 일 3가지 적어보기
   - 예: 파일 정렬, 텍스트 정리, 메타데이터 생성
2. 각 일에 대해 다음 질문에 답하기:
   - 입력: 뭘 받을 건가?
   - 처리: 어떻게 변환할 건가?
   - 출력: 어디에 저장할 건가?
   - 분류: 여러 종류가 있나?
3. **가장 간단한 것 1가지 선택**

#### 3-2. 파일 준비 (30분)

**읽을 문서:**
[[나의_첫_Skill_만드는_템플릿#사전 준비|사전 준비]]

**실행:**
```bash
mkdir -p ~/.claude/skills/your-skill-name
cd ~/.claude/skills/your-skill-name
touch SKILL.md
touch category_mapping.json
touch enhancement_rules.md
```

#### 3-3. SKILL.md 작성 (1시간)

**읽을 문서:**
[[나의_첫_Skill_만드는_템플릿#파일 1: SKILL.md 템플릿|SKILL.md 템플릿]]

**단계별 작성:**
1. **개요**: 한 문장으로 Skill 설명
2. **핵심 기능**: 3가지 이상의 기능 나열
3. **사용 방법**: 구체적인 예제 3개 이상
4. **처리 과정**: Step by Step 설명
5. **메타데이터**: YAML 샘플 제공

**확인:**
- [ ] 개요를 읽은 사람이 이해하나?
- [ ] 사용 예제를 따라할 수 있나?
- [ ] 처리 과정이 명확한가?

#### 3-4. category_mapping.json 작성 (45분)

**읽을 문서:**
[[나의_첫_Skill_만드는_템플릿#파일 2: category_mapping.json 템플릿|JSON 템플릿]]

**단계별 작성:**
1. 주요 키워드 3~5가지 생각하기
2. 각 키워드별 폴더 결정
3. 태그 정의
4. priority 순서 정하기

**검증:**
```
jsonlint.com에서 올바른 형식인지 확인
```

#### 3-5. enhancement_rules.md 작성 (45분)

**읽을 문서:**
[[나의_첫_Skill_만드는_템플릿#파일 3: enhancement_rules.md 템플릿|Enhancement 템플릿]]

**단계별 작성:**
1. Feynman 기법 규칙 정의
2. Storytelling 규칙 정의
3. 3단계 예제 만들기
4. 품질 체크리스트 작성

#### 3-6. 테스트 및 개선 (1시간)

**테스트:**
```
직접 명령어로 Skill 사용해보기:
"[your-skill-name] skill으로 [입력]을 처리해줘"
```

**확인:**
- [ ] Skill이 제대로 작동하나?
- [ ] 예상한 대로 처리되나?
- [ ] 메타데이터가 생성되나?
- [ ] 파일이 올바른 위치에 저장되나?

**개선:**
- 안 되는 부분 찾아서 수정
- 예제가 실제로 작동하는지 확인
- 설명이 명확한지 재검토

## 💡 학습 팁

### Tip 1: 동시에 3가지를 읽지 말 것

**❌ 하지 말 것:**
```
1. 초보자 가이드
2. 실전 분석
3. 템플릿
을 모두 동시에 읽기 → 머리가 복잡함
```

**✅ 올바른 방법:**
```
1. 초보자 가이드 완전히 읽기
2. 하루 쉬기 (뇌가 정보를 정리하는 시간)
3. 실전 분석 읽기
4. 하루 쉬기
5. 템플릿으로 직접 만들어보기
```

### Tip 2: 읽으면서 손으로 따라하기

**❌ 하지 말 것:**
```
화면에서만 읽기
```

**✅ 올바른 방법:**
```
읽은 내용을 직접 타이핑하면서 파일 만들기
손가락으로 가리키면서 "이건 뭐다" 말하기
```

### Tip 3: 처음엔 완벽함보다 완성을 목표

**❌ 하지 말 것:**
```
"완벽한 Skill을 만들어야지"
→ 너무 고민하다가 아무것도 못 함
```

**✅ 올바른 방법:**
```
"일단 가장 간단한 버전을 만들고 시작하자"
→ 기본 Skill 완성 후 계속 개선
```

### Tip 4: 실수해도 괜찮다

**흔한 실수들:**
```
- JSON 형식이 틀림 → 수정하면 됨
- YAML 메타데이터가 이상함 → 온라인 검증기로 확인
- 파일이 잘못된 폴더에 저장됨 → category_mapping.json 수정
```

**해결:**
모두 **수정 가능**하고, **수정하면서 배움**

## 🏆 완료 후 당신은...

이 경로를 모두 마치면:

✅ **이해**
- Skill이 뭔지 완벽하게 알게 됨
- 왜 필요한지 알게 됨
- 어떻게 작동하는지 알게 됨

✅ **만들기**
- 간단한 Skill을 혼자서 만들 수 있음
- 복잡한 Skill의 설계를 이해할 수 있음
- 남의 Skill을 분석할 수 있음

✅ **개선**
- 기존 Skill을 커스터마이징할 수 있음
- 더 나은 규칙을 추가할 수 있음
- 팀과 함께 Skill을 관리할 수 있음
