# 4-6. /agents - 커스텀 AI 서브에이전트 관리

## 🎯 사용 시점
- 특화된 작업이 필요할 때
- 커스텀 AI 서브에이전트가 필요할 때
- 특정 작업에 맞춤형 도우미가 필요할 때

## 📝 기본 사용법

### 명령어
```bash
> /agents
```

### 공식 기능
- **커스텀 AI 서브에이전트 관리**
- 특화된 작업을 위한 커스텀 AI 서브에이전트 관리

## 🔧 상세 기능

### 주요 특징
- **서브에이전트 관리**: 특화된 작업을 위한 AI 서브에이전트 생성 및 관리
- **작업 특화**: 특정 작업에 맞춤화된 에이전트 운영
- **커스텀 설정**: 사용자의 필요에 따른 맞춤형 에이전트 구성

## 💡 실제 사용 예시

### 1. 에이전트 관리 인터페이스 접근
```bash
# 에이전트 관리 화면으로 이동
> /agents

# 사용 가능한 작업:
# - 기존 에이전트 목록 확인
# - 새 에이전트 생성
# - 에이전트 설정 수정
# - 에이전트 삭제
```

### 2. 특화 작업을 위한 에이전트 활용
```bash
# 특정 작업에 맞춤화된 에이전트 사용
> /agents

# 예시:
# - 코드 리뷰 전용 에이전트
# - 문서화 전용 에이전트
# - 테스트 작성 전용 에이전트
```

## ✅ 언제 사용하면 좋을까?

### 최적의 사용 시점
- **반복적인 특화 작업**이 있을 때
- **일관된 작업 방식**이 필요할 때
- **특정 도메인 지식**이 필요한 작업이 있을 때
- **워크플로우 자동화**를 원할 때

### 효과적인 활용 상황
```bash
🎯 권장 사용 상황:
- 특정 코딩 스타일이나 패턴을 지속적으로 적용해야 하는 경우
- 반복되는 리뷰나 검증 작업이 있는 경우
- 특정 프레임워크나 라이브러리에 특화된 작업이 필요한 경우
- 팀의 일관된 작업 방식을 유지하고 싶은 경우
```

## ⚠️ 중요한 한계사항

### 사용 제한사항
- **초기 설정 필요**: 에이전트를 먼저 구성해야 함
- **학습 시간 필요**: 효과적인 에이전트 설정에는 시간이 소요됨
- **관리 복잡성**: 여러 에이전트를 관리하는 것은 복잡할 수 있음

### 대상 사용자별 적합성
- ✅ **반복 작업이 많은 개발자** - 매우 유용
- ✅ **팀 리더** - 일관된 작업 방식 유지에 도움
- ❌ **일회성 작업 중심 사용자** - 설정 대비 효과 낮음
- ❌ **초보 사용자** - 설정이 복잡할 수 있음
