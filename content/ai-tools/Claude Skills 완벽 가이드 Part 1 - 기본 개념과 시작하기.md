## 스킬이란 무엇일까요? - 실생활 이야기로 시작하기

### 📖 생활 속 이야기

여러분이 새로운 요리사를 고용했다고 상상해보세요. 이 요리사는 매우 똑똑하고 빠르게 배우지만, **여러분 가족만의 특별한 레시피**는 모릅니다.

- **일반적인 요리법**은 잘 알고 있어요 (파스타 끓이기, 스테이크 굽기)
- 하지만 **할머니의 비밀 김치찌개 레시피**나 **우리 집만의 특별한 볶음밥 만드는 법**은 모릅니다

이럴 때 어떻게 하면 좋을까요?

**해결책**: 레시피 카드를 만들어서 주방 벽에 붙여놓는 거예요! 📝

```
🍲 할머니의 김치찌개 레시피
━━━━━━━━━━━━━━━━━━
1. 묵은지 한 포기를 큼직하게 썰기
2. 돼지고기는 한입 크기로...
3. 된장 1스푼을 꼭 넣기!
...
```

**Claude Skills는 바로 이 "레시피 카드"와 같아요!**

- **Claude = 똑똑한 요리사** (AI 비서)
- **Skills = 레시피 카드** (특정 작업을 위한 상세한 지침)
- **결과 = 여러분이 원하는 완벽한 요리** (맞춤형 AI 작업 수행)

## Claude Skills의 탄생 배경

### 왜 Skills가 필요했을까요?

#### 문제 상황 1: "매번 같은 설명을 반복하는 피로감"

**이전 방식**:
```
사용자: "내 회사 브랜드 색상으로 프레젠테이션을 만들어줘"
Claude: "어떤 색상을 사용하면 될까요?"
사용자: "빨강 #FF0000, 파랑 #0000FF... (매번 반복)"
```

**Skills 사용 후**:
```
사용자: "brand-guidelines 스킬을 써서 프레젠테이션 만들어줘"
Claude: "알겠습니다! 미리 등록된 브랜드 가이드라인으로 작업할게요 ✅"
```

#### 문제 상황 2: "복잡한 작업의 일관성 부족"

웹 애플리케이션 테스트를 매번 할 때마다:
- 어떤 브라우저를 사용할지
- 어떤 순서로 테스트할지
- 어떤 형식으로 보고서를 작성할지

매번 다시 설명해야 했어요.

**Skills의 해결책**:
```yaml
# webapp-testing Skill이 미리 정의됨
- 브라우저: Playwright 사용
- 테스트 순서: 로그인 → 핵심 기능 → 에러 처리
- 보고서: 표준화된 형식
```

## Skills의 핵심 개념 - 쉽게 이해하기

### Skill의 3가지 핵심 요소

#### 1️⃣ **지침 (Instructions)** - "어떻게 할지"를 알려주는 설명서

마치 IKEA 가구 조립 설명서처럼, 단계별로 무엇을 해야 하는지 알려줍니다.

```markdown
# 예시: PDF 문서 생성 Skill
## 작업 순서
1. 사용자가 원하는 내용 파악하기
2. 적절한 레이아웃 선택하기
3. 폰트와 색상 적용하기
4. PDF 파일로 변환하기
```

#### 2️⃣ **리소스 (Resources)** - "필요한 재료"들

요리할 때 필요한 재료들처럼, 작업에 필요한 파일, 데이터, 도구들입니다.

```
📁 brand-guidelines/
  ├── logo.png         (회사 로고 이미지)
  ├── colors.json      (브랜드 색상 코드)
  └── fonts.css        (폰트 스타일시트)
```

#### 3️⃣ **스크립트 (Scripts)** - "자동화된 동작"

버튼 하나로 여러 작업을 자동으로 처리하는 프로그램들입니다.

```python
# 예시: 이미지 자동 최적화 스크립트
def optimize_image(image_path):
    # 1. 이미지 크기 조정
    # 2. 압축 적용
    # 3. 웹 최적화
    return optimized_image
```

### 🌱 기초 예제: "Hello World" Skill

가장 간단한 Skill부터 만들어볼까요?

```yaml
---
name: hello-world
description: 사용자 이름을 받아서 친근하게 인사하는 Skill
---

# Hello World Skill

## 목적
사용자의 이름을 받으면 한국어로 친근하게 인사합니다.

## 사용 방법
사용자가 이름을 말하면 다음 형식으로 인사하세요:

"안녕하세요, [이름]님! 😊
오늘 하루도 좋은 일만 가득하시길 바랍니다!"

## 예시
입력: "내 이름은 민수야"
출력: "안녕하세요, 민수님! 😊 오늘 하루도 좋은 일만 가득하시길 바랍니다!"
```

**이것은 마치**: 편지 쓰는 템플릿을 미리 만들어둔 것과 같아요. 이름만 바꿔서 넣으면 항상 똑같이 멋진 인사말이 완성됩니다!

### 🌿 중급 예제: "회사 보고서 작성" Skill

실제 업무에서 자주 쓰이는 패턴입니다.

```yaml
---
name: company-report
description: 표준화된 형식의 회사 보고서를 자동으로 생성
---

# 회사 보고서 작성 Skill

## 보고서 표준 구조
1. **제목**: [프로젝트명] - [날짜] 진행 보고
2. **요약**: 3-5줄로 핵심 내용 요약
3. **진행 상황**:
   - 완료된 작업 (✅)
   - 진행 중인 작업 (🔄)
   - 예정된 작업 (📅)
4. **이슈 및 해결방안**
5. **다음 단계**

## 스타일 가이드
- 문체: 간결하고 명확하게
- 용어: 회사 표준 용어 사용
- 길이: A4 2페이지 이내

## 예시 출력
[실제 보고서 샘플 첨부]
```

**실용성**: 매주 쓰는 보고서 형식을 한 번만 정의해두면, 다음부터는 내용만 입력하면 자동으로 완벽한 형식의 보고서가 완성됩니다!

### 🌳 고급 예제: "웹앱 자동 테스트" Skill

프로페셔널한 개발 환경에서 사용하는 복잡한 Skill입니다.

```yaml
---
name: webapp-testing
description: Playwright를 사용한 체계적인 웹 애플리케이션 테스트
---

# 웹앱 자동 테스트 Skill

## 테스트 시나리오 구조

### 1단계: 환경 설정
```python
# Playwright 브라우저 초기화
browser = playwright.chromium.launch(headless=False)
context = browser.new_context(
    viewport={'width': 1920, 'height': 1080},
    locale='ko-KR'
)
```

### 2단계: 테스트 케이스 실행
- **로그인 테스트**: 정상/실패 케이스 모두 확인
- **핵심 기능 테스트**: 사용자 여정(User Journey) 시뮬레이션
- **에러 처리 테스트**: 예상되는 오류 상황 테스트

### 3단계: 결과 보고
```markdown
# 테스트 결과 리포트
- 실행 일시: 2025-10-17 14:30
- 총 테스트: 25개
- 성공: 23개 ✅
- 실패: 2개 ❌
- 상세 로그: [링크]
```

## 고급 기능
- 스크린샷 자동 캡처
- 성능 메트릭 측정
- 이전 테스트 결과와 비교
```

**심화 활용**: 이 Skill 하나로 매일 밤 자동으로 전체 웹사이트를 테스트하고, 문제가 발견되면 개발팀에 자동으로 알림을 보낼 수 있습니다.

## 어디서 어떻게 사용하나요?

### Skills를 사용할 수 있는 3가지 방법

#### 방법 1: Claude.ai 웹사이트 (가장 쉬움!)

**누구에게 추천**: 프로그래밍 경험이 없는 일반 사용자

**장점**:
- 클릭 몇 번으로 바로 사용 가능
- 브라우저만 있으면 OK
- Anthropic에서 제공하는 Skills 즉시 이용

**사용 방법**:
```
1. claude.ai 접속
2. 유료 플랜 가입 (Pro, Team, Enterprise)
3. 설정 → Skills 메뉴
4. 원하는 Skill 활성화 또는 업로드
```

**🤔 생각해보기**:
- 회사에서 자주 쓰는 문서 형식이 있나요?
- 그것을 Skill로 만들면 시간을 얼마나 절약할 수 있을까요?

---

#### 방법 2: Claude Code (개발자용)

**누구에게 추천**: 코드를 작성하는 개발자

**장점**:
- 코드 에디터와 통합
- 프로그래밍 작업에 최적화
- GitHub와 직접 연동

**사용 방법**:
```bash
# 터미널에서 실행
/plugin marketplace add anthropics/skills

# 특정 Skill 사용
pdf 스킬을 사용하여 보고서.pdf 생성해줘
```

**실전 예시**:
```
개발자: "webapp-testing 스킬로 내 로컬 앱 테스트해줘"
Claude Code: "알겠습니다! Playwright 실행 중..."
[자동으로 브라우저가 열리고 테스트 진행]
Claude Code: "✅ 테스트 완료! 23/25 성공"
```

---

#### 방법 3: Claude API (전문가용)

**누구에게 추천**: 자동화 시스템을 구축하는 엔지니어

**장점**:
- 완전 자동화 가능
- 기존 시스템에 통합
- 대규모 작업 처리

**간단한 예시**:
```python
import anthropic

# API 클라이언트 초기화
client = anthropic.Anthropic(api_key="your-api-key")

# Skill 사용하여 메시지 전송
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    skills=["pdf"],  # PDF Skill 활성화
    messages=[{
        "role": "user",
        "content": "2025년 3분기 보고서를 PDF로 만들어줘"
    }]
)
```

**고급 활용**:
- 매일 밤 자동으로 데이터 분석 보고서 생성
- 고객 문의를 자동으로 분류하고 답변
- 대량의 문서를 자동으로 처리

## 첫 번째 Skill 사용해보기 - 단계별 실습

### 실습 1: Claude.ai에서 미리 만들어진 Skill 사용하기

**목표**: PDF Skill을 사용하여 간단한 이력서 PDF 만들기

**준비물**:
- Claude.ai Pro 계정
- 브라우저

**단계별 진행**:

#### Step 1: Skill 활성화
```
1. claude.ai 로그인
2. 왼쪽 하단 ⚙️ 설정 클릭
3. "Skills" 탭 선택
4. "pdf" 스킬 찾아서 활성화 토글 ON
```

#### Step 2: Skill 사용하기
```
새 대화 시작하고 다음과 같이 입력:

"pdf 스킬을 사용해서 내 이력서를 만들어줘.

이름: 김민수
직업: 프론트엔드 개발자
경력: 3년
주요 기술: React, TypeScript, Tailwind CSS
학력: OO대학교 컴퓨터공학과 졸업"
```

#### Step 3: 결과 확인
```
Claude가 자동으로:
1. 전문적인 이력서 레이아웃 선택
2. 내용을 적절히 배치
3. PDF 파일 생성
4. 다운로드 링크 제공
```

**⚠️ 주의사항**:
- 첫 사용 시 Skill 로딩에 2-3초 걸릴 수 있어요
- 무료 플랜에서는 일부 Skill만 사용 가능합니다
- 대화 시작 시 Skill을 명시하면 더 정확해요

   3. [...]

   ### 배운 점
   [...]

   💭 한줄 회고: "좋은 사람들과 함께하는 시간이 가장 소중하다"
   ```

## 용어 사전 (Glossary)

복잡한 용어들을 **5살 아이도 이해할 수 있게** 풀어썼어요!

### 📘 기본 용어

#### **Skill (스킬)**
**쉬운 설명**: AI 비서에게 주는 "일하는 방법 안내서"
**비유**: 레고 조립 설명서처럼, 단계별로 어떻게 할지 알려주는 메뉴얼
**예시**: "브랜드 색상으로 디자인하는 방법" 스킬

#### **SKILL.md 파일**
**쉬운 설명**: Skill의 설명서가 적힌 텍스트 파일
**비유**: 레시피 카드 (재료와 만드는 방법이 적혀있음)
**예시**: `pdf-generator.md` - PDF 만드는 방법 설명서

#### **YAML 프론트매터 (Frontmatter)**
**쉬운 설명**: 문서 맨 위에 적는 기본 정보
**비유**: 책 표지 (제목, 저자, 출판일 등)
**예시**:
```yaml
---
name: my-skill         # 이름
description: 설명      # 무엇을 하는지
---
```

### 🔧 기술 용어

#### **API (Application Programming Interface)**
**쉬운 설명**: 프로그램들이 서로 대화하는 방법
**비유**: 레스토랑의 메뉴판 (원하는 것을 주문하는 방식)
**예시**: "Claude API에 '보고서 만들어줘' 요청 → Claude가 처리 → 결과 받기"

#### **Playwright**
**쉬운 설명**: 웹사이트를 자동으로 테스트하는 도구
**비유**: 로봇 손님이 웹사이트의 모든 버튼을 눌러보고 작동하는지 확인
**사용 예시**: 쇼핑몰 웹사이트의 로그인, 장바구니, 결제 기능 자동 테스트

#### **MCP 서버 (Model Context Protocol Server)**
**쉬운 설명**: Claude가 외부 서비스와 연결하는 다리
**비유**: 통역사 (Claude와 다른 프로그램 사이에서 의사소통 도와줌)
**예시**: Claude가 날씨 정보를 가져오려면 날씨 MCP 서버를 통해 기상청과 연결

#### **Repository (저장소)**
**쉬운 설명**: 코드와 파일을 저장하는 온라인 보관함
**비유**: 구글 드라이브의 폴더
**예시**: `anthropics/skills` GitHub 저장소

### 🎨 Skills 관련 용어

#### **Algorithmic Art (알고리즘 아트)**
**쉬운 설명**: 컴퓨터 프로그램으로 만드는 예술 작품
**비유**: 자동으로 그림을 그리는 로봇 화가
**Skill 예시**: p5.js로 움직이는 패턴 생성

#### **Brand Guidelines (브랜드 가이드라인)**
**쉬운 설명**: 회사 디자인 규칙 (색상, 로고, 폰트 등)
**비유**: 학교 교복 규정 (정해진 스타일 따라야 함)
**Skill 적용**: 모든 문서에 회사 색상 자동 적용

#### **Artifact (아티팩트)**
**쉬운 설명**: Claude가 만들어낸 결과물 (문서, 코드, 이미지 등)
**비유**: 공장에서 만든 완성된 제품
**예시**: PDF 보고서, HTML 웹페이지, Python 스크립트

#### **Template (템플릿)**
**쉬운 설명**: 내용만 바꿔 넣으면 완성되는 빈 양식
**비유**: 색칠하기 책 (윤곽선만 있고 색칠은 직접)
**예시**: 이력서 템플릿, 보고서 템플릿

---

### 📂 파일 형식

#### **.md (마크다운)**
**쉬운 설명**: 간단한 기호로 서식을 만드는 텍스트 파일
**장점**: 메모장으로도 편집 가능, 읽기 쉬움
**예시**:
```markdown
# 제목 (샵 하나는 큰 제목)
## 소제목 (샵 두 개는 작은 제목)
**굵게** (별표 두 개)
```

#### **.json**
**쉬운 설명**: 데이터를 저장하는 구조화된 형식
**비유**: 서류 정리함 (항목별로 깔끔하게 정리)
**예시**:
```json
{
  "이름": "김민수",
  "나이": 25,
  "기술": ["Python", "React"]
}
```

#### **.yaml**
**쉬운 설명**: JSON보다 더 읽기 쉬운 데이터 형식
**특징**: 들여쓰기로 구조 표현
**예시**:
```yaml
이름: 김민수
나이: 25
기술:
  - Python
  - React
```

### 🔐 계정 및 플랜

#### **Pro Plan (프로 플랜)**
**설명**: Claude.ai 유료 구독 (월 $20)
**혜택**: 더 많은 메시지, 모든 Skills 사용 가능
**비유**: 넷플릭스 프리미엄 구독

#### **API Key (API 키)**
**쉬운 설명**: 프로그램이 Claude를 사용할 수 있게 해주는 비밀번호
**주의**: 절대 공개하면 안 됨 (돈 빠져나갈 수 있음)
**형태**: `sk-ant-api03-...` 같은 긴 문자열

#### **Token (토큰)**
**쉬운 설명**: AI가 처리하는 글자의 단위 (요금 계산에 사용)
**대략**: 한국어 1글자 ≈ 2-3 토큰
**비유**: 전기 요금처럼 사용량에 따라 과금
