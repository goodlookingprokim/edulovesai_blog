## 핵심 학습 내용

### 1. 프롬프트 엔지니어링 기초 원리

#### 효과적인 프롬프트의 4가지 핵심 요소
- **명확성 (Clarity)**: 모호함 없는 명확한 지시
- **구체성 (Specificity)**: 구체적이고 세부적인 요구사항
- **맥락성 (Context)**: 충분한 배경 정보 제공
- **구조성 (Structure)**: 논리적이고 체계적인 구성

#### 기본 프롬프트 구조
```
[역할 설정] + [작업 정의] + [맥락 제공] + [출력 형식] + [제약 조건]
```

### 2. 역할 기반 프롬프팅 (Role Prompting)

#### 효과적인 역할 부여 방법
- **전문가 역할**: "당신은 마케팅 전문가입니다"
- **특정 페르소나**: "당신은 10년 경력의 데이터 분석가입니다"
- **상황적 역할**: "당신은 고객 서비스 담당자로서..."

#### 역할 설정 베스트 프랙티스
- 구체적인 전문 분야 명시
- 경험 수준 및 배경 정보 포함
- 역할에 맞는 톤앤매너 설정

### 3. 데이터와 지시사항 분리

#### 구조화된 프롬프트 작성
```xml
<instructions>
다음 데이터를 분석하여 핵심 인사이트를 도출하세요.
</instructions>

<data>
[실제 데이터 내용]
</data>

<format>
1. 핵심 발견사항
2. 트렌드 분석
3. 권장사항
</format>
```

#### 분리의 장점
- **혼동 방지**: 데이터와 지시사항 구분 명확
- **재사용성**: 동일한 구조로 다양한 데이터 처리
- **정확성**: 잘못된 해석 가능성 감소

## 기본 프롬프트 기법

### 1. 명확하고 직접적인 프롬프팅

#### DO: 명확한 지시
```
"다음 텍스트를 3개의 핵심 포인트로 요약해 주세요."
```

#### DON'T: 모호한 지시
```
"이 텍스트에 대해 뭔가 해주세요."
```

### 2. 출력 포맷 지정

#### JSON 형태 출력
```json
{
  "요약": "핵심 내용 요약",
  "키워드": ["키워드1", "키워드2", "키워드3"],
  "감정": "긍정적/부정적/중립적"
}
```

#### 표 형태 출력
```
| 항목 | 내용 | 중요도 |
|------|------|--------|
| 항목1 | 설명1 | 높음 |
| 항목2 | 설명2 | 중간 |
```

### 3. 단계별 사고 프롬프팅

#### Chain of Thought (CoT) 기법
```
"다음 문제를 단계별로 풀어보세요:
1. 먼저 문제를 이해하고
2. 필요한 정보를 파악하고
3. 논리적으로 해결하세요"
```

#### 사고 과정 표시
```
<thinking>
이 문제를 해결하기 위해서는...
</thinking>

<answer>
최종 답변
</answer>
```

## 고급 프롬프트 기법

### 1. Few-Shot 프롬프팅

#### 예시 기반 학습
```
예시 1:
입력: "오늘 날씨가 좋아요"
출력: 감정: 긍정적, 주제: 날씨

예시 2:
입력: "교통이 너무 막혀요"
출력: 감정: 부정적, 주제: 교통

이제 다음을 분석하세요:
입력: "새로운 프로젝트가 시작되었어요"
```

### 2. 환각 방지 기법

#### 불확실성 인정 방법
```
"확실하지 않은 정보는 '확실하지 않음'으로 표시하세요."
"추측이나 가정은 명확히 구분하여 표시하세요."
```

#### 인용 및 출처 요구
```
"모든 사실적 정보는 출처를 명시하거나 '출처 불명'으로 표시하세요."
```

### 3. 복잡한 프롬프트 작성

#### 다단계 작업 분할
```
1단계: 텍스트 분석
2단계: 핵심 주제 추출
3단계: 감정 분석
4단계: 종합 결론 도출
```

#### 조건부 로직 활용
```
IF 긍정적 감정 > 70% THEN "긍정적 리뷰"
ELSE IF 부정적 감정 > 70% THEN "부정적 리뷰"
ELSE "중립적 리뷰"
```

## 부록 및 확장 기법

### 1. 프롬프트 체이닝 (Prompt Chaining)

#### 순차적 프롬프트 연결
```
프롬프트 1: 데이터 수집 및 정리
↓
프롬프트 2: 데이터 분석 및 패턴 파악
↓
프롬프트 3: 인사이트 도출 및 권장사항
```

#### 체이닝 설계 원칙
- 각 단계의 출력이 다음 단계의 입력
- 단계별 검증 및 품질 관리
- 오류 발생 시 롤백 메커니즘

### 2. 도구 사용 (Tool Use)

#### 함수 호출 기법
```python
def analyze_sentiment(text):
    # 감정 분석 함수
    pass

def extract_keywords(text):
    # 키워드 추출 함수
    pass
```

#### API 연동 활용
- 외부 데이터 소스 연결
- 실시간 정보 업데이트
- 다양한 서비스 통합

### 3. 성능 평가 및 측정

#### 평가 지표
- **정확성**: 올바른 답변 비율
- **완전성**: 요구사항 충족 정도
- **일관성**: 동일한 입력에 대한 일관된 출력
- **효율성**: 응답 시간 및 리소스 사용량

#### A/B 테스트 방법
```
버전 A: 기본 프롬프트
버전 B: 개선된 프롬프트
→ 성능 비교 분석
```
