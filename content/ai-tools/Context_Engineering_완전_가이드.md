## 🔧 Context Engineering 핵심 원리

### 🎯 정의 및 개념
**Context Engineering이란?**
- AI가 각 단계에서 효율적이고 효과적으로 작업을 수행할 수 있도록 필요한 정보를 신중하게 선택, 구성, 관리하는 기술
- 과도한 정보 제공이나 중요한 세부사항 누락 없이 적절한 균형 유지
- 단순한 프롬프트를 넘어선 종합적인 컨텍스트 관리

### 🧠 컨텍스트 구성 요소
**완전한 컨텍스트의 7가지 요소**
- **상태(State)**: 현재 프로젝트 상황
- **히스토리(History)**: 이전 작업 이력
- **사용자 프롬프트(User Prompt)**: 현재 요청사항
- **사용 가능한 도구(Available Tools)**: 활용 가능한 리소스
- **RAG 지침(RAG Instructions)**: 검색 증강 생성 가이드
- **장기 메모리(Long-term Memory)**: 축적된 지식
- **구조화된 출력**: 최적화된 결과물

### 💡 핵심 철학
- **과학과 직관의 조합**: 체계적 접근과 경험적 판단의 결합
- **시스템 설계**: AI와의 협업을 위한 구조적 프레임워크
- **실세계 적용**: 단순한 ChatGPT 래퍼를 넘어선 실용적 시스템

## 📚 실전 구현 가이드

### 🚀 1단계: 환경 설정
**필수 준비사항**
1. **Git 설치**: 리포지토리 클론을 위한 버전 관리 도구
2. **Node.js 설치**: Claude Code 기능 활용을 위한 런타임
3. **Claude Code 설치**: 명령줄에서 `claude` 명령어로 설치
4. **API 키 설정**: Claude AI 서비스 인증

### 🎯 2단계: 템플릿 설정
**리포지토리 클론 및 설정**
```bash
git clone [템플릿 리포지토리 URL]
cd context-engineering-template
```

**핵심 파일 구성**
- **claude.md**: 프로젝트 전역 규칙 정의
- **initial.md**: 초기 기능 요청사항 명세
- **commands/**: 커스텀 슬래시 명령어 정의
- **examples/**: 참조용 예제 파일들

### 🔧 3단계: 글로벌 규칙 설정
**claude.md 파일 구성**
- **코드 구조**: 프로젝트 코딩 표준 정의
- **테스팅**: 테스트 방법론 및 도구 설정
- **신뢰성**: 코드 품질 보장 기준
- **작업 완료**: 완료 조건 및 검증 방법
- **시스템 및 대화**: AI와의 상호작용 규칙

### 📝 4단계: 기능 요청 명세
**initial.md 파일 작성**
- **Features**: 구현하고자 하는 기능의 명확한 설명
- **Examples**: 참조할 파일들과 컨텍스트 제공
- **Documentation**: 관련 문서, API, MCP 서버 리소스 링크
- **Additional Considerations**: 엣지 케이스 및 특별 요구사항

## 🛠️ 템플릿 활용법

### ⚡ PRP 생성 프로세스
**Product Requirement Prompt 자동 생성**
```bash
/generate-prp initial.md
```

**PRP의 핵심 기능**
- **철저한 계획**: 코드 작성 전 상세한 계획 수립
- **API 연구**: 관련 API 및 라이브러리 사전 조사
- **코드베이스 분석**: 기존 프로젝트 구조 파악
- **예제 검토**: 제공된 참조 자료 분석
- **문서 검토**: 관련 문서 및 가이드라인 확인

### 🎯 PRP 실행 및 구현
**자동 코드 생성 실행**
```bash
/execute-prp [PRP_폴더] [PRP_파일명.md]
```

**실행 과정의 특징**
- **체계적 접근**: 단계별 작업 분해
- **컨텍스트 참조**: 모든 관련 정보를 종합적으로 활용
- **품질 보장**: 환각 현상 최소화
- **효율성**: 토큰 사용량 최적화
