## 개요와 배경

### 왜 이 가이드가 필요한가?
상상해 보세요. 당신은 프로그래밍을 막 시작한 초보 개발자입니다. AI가 마법처럼 코드를 작성해주지만, 어떻게 지시해야 할지 모르겠어요. 마치 아이가 장난감을 가지고 놀 때 "이렇게 해!"라고 말해야 장난감이 제대로 움직이듯이, AI에게는 '프롬프트'라는 지시가 필요합니다.

페이만 기법처럼, 이 가이드는 복잡한 AI 개념을 마치 10살 아이에게 설명하듯 간단히 풀어갑니다. 스토리: "옛날에 한 마을에 AI 마법사가 살았어요. 마법사는 주인의 말만 잘 들으면 황금 알을 낳았지만, 말투가 엉망이면 엉뚱한 걸 만들었죠. 당신이 그 마법사의 주인이 되는 이야기예요!"

Lovable은 코드 없이 AI 앱을 만드는 플랫폼입니다. 이 가이드에서 4단계 프롬프트를 배우며, 초보에서 고급까지 성장하세요.

## 프롬프트가 무엇일까? 스토리텔링으로 이해하기

프롬프트는 AI에게 주는 '명령어'예요. 하지만 단순한 명령이 아니라, 대화처럼요. 스토리: "AI는 똑똑한 요리사예요. '밥 해라'라고 하면 밥만 나오지만, '맛있는 비빔밥을 매운 맛으로, 채소 듬뿍 넣어 만들어줘'라고 하면 완벽한 요리가 나와요."

초보 수준 예시: "안녕, AI야. 'hello world'를 출력하는 Python 코드를 써줘." → AI가 간단한 코드를 줍니다.

중급: "초보자를 위한 'hello world' 설명과 함께 코드를 작성해. 왜 이 코드가 중요한지 설명해."

고급: "Python 초보자를 가르치는 튜토리얼 형식으로 'hello world' 코드를 작성해. 단계별로 설명하고, 흔한 실수를 피하는 팁 추가."

## 페이만 기법으로 기본 배우기: 훈련 바퀴 프롬프트

페이만 기법: "아이에게 설명할 수 있으면 진짜 이해한 거야." 훈련 바퀴 프롬프트는 템플릿처럼 AI가 어떻게 답할지 미리 보여주는 거예요. 자전거에 보조 바퀴를 달아 안 넘어지게 하듯이.

스토리: "첫 자전거를 타는 아이가 템플릿(지도) 없이 길을 잃지 않게 도와줘요."

초보 예시: 입력: "다음 문장을 요약해." 출력 예시: "이 문장은 ...에 관한 거야." Lovable에서 온보딩 봇 만들기.

상세 설명: 템플릿 사용으로 예측 가능. 예: 이메일 생성기 - "친구에게 생일 축하 이메일을 써. 형식: 인사 + 축하 + 마무리."

## 중급: 훈련 바퀴 없이 자유롭게 프롬프트 작성

이제 보조 바퀴를 떼고 직접 타보세요. AI에게 역할 부여하고, 자유롭게 지시.

스토리: "자전거를 자유롭게 타며 모험하는 이야기. 하지만 길을 잃지 않게 지도(명확한 지시) 필요."

중급 예시: "너는 전문 작가야. 이 문단을 설득력 있게 다시 써." 형식 지정: "불릿 포인트로."

고급 예시: API 호출과 결합 - "최신 날씨 데이터를 가져와 여행 계획을 세워줘."

## 고급: 메타 프롬프트와 역메타 프롬프트

메타: AI에게 프롬프트를 개선하게 함. 스토리: "마법사가 주인의 주문을 더 좋게 만들어주는 거예요."

예시 (초보): "이 프롬프트를 더 좋게 만들어: '계약 요약해'."
AI: "구체적으로: '법적 계약을 정중한 톤으로 요약해. 주요 조항 3가지 나열.'"

역메타: AI가 프롬프트를 비판. 고급 예시: "이 프롬프트의 문제점 분석해."

## 프롬프트 라이브러리 활용: 재사용의 힘

라이브러리는 저장된 프롬프트 모음. 스토리: "요리 레시피북처럼, 한 번 만들면 여러 번 써요."

예시: "너는 {{역할}}이야. {{주제}}에 대해 {{톤}}으로 작성해." 변수로 재사용.

초보: 간단 Q&A 라이브러리.
중급: 팀 공유 템플릿.

## 실전 사례: 초보자부터 고급까지 예시

초보 사례: 간단 요약 봇 - "문단 요약해." (훈련 바퀴 사용)

중급: 고객 지원 - "고객 메시지 분석하고 답변 작성. 공감적으로."

고급: CI/CD 모니터링 봇 - API 로그 분석, Slack 알림.

페이만 적용: 각 사례를 "왜 이게 중요한지" 아이에게 설명하듯.

## 용어 사전 (Glossary)
- **프롬프트 (Prompt)**: AI에게 주는 지시어. 마치 레시피처럼 구체적일수록 좋음.
- **Lovable**: 코드 없이 AI 앱 만드는 플랫폼. 블록 쌓듯이 프롬프트 연결.
- **페이만 기법 (Feynman Technique)**: 복잡한 걸 단순히 설명하며 이해 확인.
- **메타 프롬프트 (Meta Prompting)**: AI에게 프롬프트를 고치게 함. 자기 개선.
- **체이닝 (Chaining)**: 프롬프트 여러 개 연결. 단계별 작업.
- **환각 (Hallucination)**: AI가 거짓 정보 만듦. 구체적 프롬프트로 방지.
- **토큰 (Token)**: AI가 처리하는 단위. 비용과 속도 영향.

쉬운 언어로: 환각은 AI가 "꿈꾸는" 거예요. 깨우려면 사실만 물어보세요!
