Mistake: Output sometimes includes citations, sometimes 
doesn't.
Fix: "Always include 1-2 data points with source citations in 
parentheses at the end of each main point."
Establish clear, consistent formatting expectations for all outputs.

Fix: Add fallback: "If the user request is unclear or outside the

information about your financial situation to provide helpful 
guidance.'"
Always plan for edge cases and provide clear instructions for 
handling them.
 
Mistake: "Based on this, generate a solution."
Fix: "Based on this customer support ticket: {{support_ticket}}, 
generate a troubleshooting response for our mobile app user."
Always clearly reference variables and specify how they should 
be interpreted.

Identifying Issues with Vague or Inconsistent Outputs
 
 
Example: You asked for JSON but got Markdown or mixed 
formats
Fix: Be more explicit: "Return only valid JSON with no explanatory 
text before or after. Use the exact keys and structure shown in 
this example: {example}"
 
Example: Sometimes responses are formal, sometimes casual
Fix: Define tone precisely: "Write in a professional but approachable tone. Use short sentences and avoid technical 
jargon. Limit your response to 3 paragraphs."
 
Example: AI responds to the wrong part of a complex question
Fix: Add parsing instructions: "First identify the main request in 
the user's message. If there are multiple questions, address only 
the primary one and note that you're focusing on that aspect."
Example: Responses start with "Sure! Here's your report:" or "I'd
be happy to help!"
Fix: Be direct: "Start your response immediately with the content. 
Do not include introductory phrases, pleasantries, or offers to 
help."
