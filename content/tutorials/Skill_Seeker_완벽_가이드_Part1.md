상상해보세요. 여러분이 새로운 게임을 만들고 싶어서 **Godot**이라는 게임 엔진을 배우기로 했어요.

```
여러분: "좋아! 공식 문서를 읽어보자!"
*문서 사이트 접속*

여러분: "...뭐?! 😱"
```

**현실:**
- ❌ 40,000페이지를 다 읽을 수는 없어요
- ❌ 검색해도 원하는 내용 찾기 어려워요
- ❌ 예제는 있는데 내 상황에 딱 맞는 건 없어요
- ❌ 영어로 되어있어서 더 힘들어요

**우리가 원하는 것:**
- ✅ "이거 어떻게 해?" → AI가 바로 답해주기
- ✅ 내 수준에 맞는 예제 보여주기

바로 이럴 때 **Skill Seeker**가 등장합니다! 🦸‍♂️

## Skill Seeker가 뭔가요?

### 🎁 마법 같은 도구의 정체

**Skill Seeker**는 **"문서 사이트를 Claude AI가 이해할 수 있는 지식으로 변환해주는 자동화 도구"**입니다.

#### 🌟 마법의 동화로 이해하기

옛날에 **거대한 도서관**이 있었어요. (= 문서 사이트)
- 책이 40,000권이나 있었죠
- 책은 모두 영어로 되어있었어요
- 찾고 싶은 내용이 어디 있는지 몰라요

그런데 **마법사 Skill Seeker**가 나타나서:
1. 📚 모든 책을 읽고
2. 🏷️ 주제별로 정리하고
3. 📝 핵심 내용과 예제를 뽑아내고
4. 🎁 Claude AI가 이해할 수 있는 "선물 상자"로 포장했어요!

이제 Claude에게 물어보면:
```
여러분: "Godot에서 캐릭터 점프는 어떻게 만들어?"
Claude: "네! 제가 Godot Skill을 가지고 있어서 알려드릴게요!"
          *정확한 코드 예제와 설명 제공*
```

### 🔍 WHY (왜) → WHAT (무엇) → HOW (어떻게)

#### 1️⃣ WHY - 왜 필요한가요?

**문제 상황:**
```
개발자: "React 배우고 싶은데 문서가 180페이지네..."
        "Django는 500페이지..."
        "FastAPI는 300페이지..."
        "다 읽을 시간이 없어! 😭"
```

**해결책:**
```
Skill Seeker: "제가 20분 만에 다 읽고 정리해드릴게요!
               Claude한테 선물로 포장해서 전달하면
               이제부터 Claude가 전문가처럼 답변해줄 거예요! ✨"
```

#### 2️⃣ WHAT - 정확히 무엇인가요?

Skill Seeker는 **3가지 역할**을 합니다:

**역할 1: 자동 수집가 📚**
```python
# 이런 작업을 자동으로!
문서_사이트 = "https://react.dev/"
자동으로_모든_페이지_읽기()
자동으로_코드_예제_찾기()
자동으로_중요한_부분_표시하기()
```

**역할 2: 똑똑한 정리왕 🏷️**
```
읽은 내용을:
- "초보자용 가이드" 폴더에
- "API 레퍼런스" 폴더에
- "고급 기술" 폴더에
자동으로 분류!
```

**역할 3: 선물 포장 전문가 🎁**
```
정리된 내용을
→ Claude AI가 좋아하는 형식으로
→ .zip 파일로 예쁘게 포장
→ 업로드하면 끝!
```

#### 3️⃣ HOW - 어떻게 작동하나요?

**전체 과정을 피자 만들기에 비유하면:**

```
🍕 피자 만들기 과정:
1. 재료 준비 (도우, 토마토 소스, 치즈...)
2. 도우 펴기
3. 토핑 올리기
4. 오븐에 굽기
5. 완성! 🍕

🤖 Skill 만들기 과정:
1. 재료 준비 (문서 사이트 URL)
2. 문서 수집 (스크래핑)
3. 내용 정리 (카테고리별 분류)
4. AI 향상 (예제 추출, 설명 개선)
5. 완성! (Claude에 업로드) 🎉
```

## 어떻게 작동하나요?

### 🎬 실제 작동 과정 - 영화 촬영에 비유

**영화 제작 과정:**
```
1. 기획 단계: 어떤 영화 만들지 결정
2. 촬영 단계: 필요한 장면 찍기
3. 편집 단계: 불필요한 부분 자르고 정리
4. 후반 작업: 특수효과, 음악 추가
5. 개봉: 극장에 상영
```

**Skill Seeker 작동 과정:**

#### 📋 1단계: 기획 (설정 파일 만들기)

**비유:** 영화 기획서 작성
**실제:** `config.json` 파일 만들기

```json
{
  "name": "react",
  "base_url": "https://react.dev/",
  "description": "React UI 프레임워크 문서",
  "max_pages": 200
}
```

**쉬운 설명:**
- `name`: 영화 제목 (skill 이름)
- `base_url`: 촬영 장소 (문서 사이트 주소)
- `description`: 영화 줄거리 (이 skill이 뭔지)
- `max_pages`: 촬영할 장면 수 (읽을 페이지 개수)

#### 🎥 2단계: 촬영 (문서 스크래핑)

**비유:** 카메라로 장면 찍기
**실제:** 웹사이트에서 내용 가져오기

```python
# 이런 일이 자동으로 일어나요!
for 페이지 in 문서사이트.모든페이지:
    제목 = 페이지.제목_가져오기()
    내용 = 페이지.본문_가져오기()
    코드예제 = 페이지.코드_찾기()
    
    저장하기(제목, 내용, 코드예제)
```

**진행 화면:**
```
📥 페이지 수집 중...
[=====>          ] 23/180 (13%)
⏱️  예상 소요 시간: 15분
```

#### ✂️ 3단계: 편집 (내용 분류)

**비유:** 영화 편집 - 장면별로 정리
**실제:** 주제별로 문서 분류

```
읽은 페이지들을 분석해서:

📂 getting_started/
   - 설치 방법.md
   - 첫 프로젝트.md

📂 api_reference/
   - useState.md
   - useEffect.md

📂 advanced/
   - 성능 최적화.md
   - 커스텀 훅.md
```

#### ✨ 4단계: 후반 작업 (AI 향상)

**비유:** 특수효과와 음악 추가
**실제:** Claude AI로 내용 개선

```python
# AI가 자동으로:
- 최고의 코드 예제 5~10개 선별
- 초보자용 설명 추가
- 단계별 가이드 생성
- 주의사항과 팁 정리
```

**변화 예시:**

**BEFORE (AI 향상 전):**
```markdown
# React 소개
React는 UI 라이브러리입니다.
```

**AFTER (AI 향상 후):**
```markdown
# React 소개

## 🌱 초보자용 예제
React는 마치 레고 블록처럼 작은 부품(컴포넌트)을 
조합해서 웹사이트를 만드는 도구예요!

### 가장 간단한 예제
```jsx
function Hello() {
  return <h1>안녕하세요!</h1>;
}
```

### 🤔 생각해보기
- 이 코드에서 `Hello`는 무엇일까요?
- `return` 뒤에 있는 게 HTML처럼 보이는데...?

### ⚠️ 주의사항
- 컴포넌트 이름은 반드시 대문자로 시작해야 해요
```

#### 🎁 5단계: 개봉 (패키징 & 업로드)

**비유:** 극장 개봉
**실제:** Claude에 업로드

```
최종 결과물:
📦 react.zip
   ├── SKILL.md (핵심 가이드)
   ├── references/ (상세 문서)
   │   ├── index.md
   │   ├── getting_started.md
   │   └── api_reference.md
   └── (기타 파일들)

→ Claude AI 사이트에 업로드
→ 이제 Claude가 React 전문가가 되었어요! 🎉
```

### 📋 Step 1: Python 확인

**Python이 뭔가요?**
> 프로그래밍 언어의 한 종류예요. Skill Seeker는 Python으로 만들어졌어요.
> 마치 한국 요리를 하려면 젓가락과 숟가락이 필요한 것처럼,
> Python 프로그램을 실행하려면 Python이 설치되어 있어야 해요!

**확인 방법:**

```bash
# 터미널을 열고 입력:
python3 --version

# 결과 예시:
Python 3.11.5  ✅ 좋아요!
Python 3.9.0   ❌ 너무 낮아요 (3.10 이상 필요)
command not found  ❌ 설치 안 됨
```

**설치 필요하면:**
1. https://www.python.org/downloads/ 방문
2. 큰 노란색 "Download Python" 버튼 클릭
3. 다운로드된 파일 실행
4. **중요!** "Add Python to PATH" 체크박스 선택
5. "Install Now" 클릭

### 📋 Step 2: Git 확인

**Git이 뭔가요?**

> 마치 도서관에서 책을 빌려오는 사서 선생님 같은 역할!

```bash
# 확인:

# 결과:

**설치:**

2. "Download" 클릭
3. 설치 프로그램 실행
4. 모든 옵션 기본값으로 두고 "Next" 계속 클릭

# 1. 터미널 열기

```

#### 🌿 중급자 레벨: 가상환경 만들기

**가상환경(Virtual Environment)이 뭔가요?**
> 각 프로젝트만을 위한 "독립된 작업 공간"이에요.
> 비유: 각자의 방에서 공부하는 것
> - 내 방(가상환경A)에서는 Python 라이브러리 A 사용
> - 동생 방(가상환경B)에서는 Python 라이브러리 B 사용
> - 서로 충돌 안 나요!

```bash
# 1. 가상환경 생성
python3 -m venv venv

# 이게 뭔가요?
# python3 = Python 3 사용해줘
# -m venv = venv 모듈 실행해줘
# venv (마지막) = "venv"라는 이름으로 폴더 만들어줘

# 2. 가상환경 활성화
# Mac/Linux:
source venv/bin/activate

# Windows:
venv\Scripts\activate

# 성공하면 이렇게 보여요:
(venv) YourName@Computer Skill_Seekers %

# 👆 앞에 (venv)가 보이면 성공!
```

#### 🌳 고급자 레벨: 의존성 설치

**의존성(Dependencies)이 뭔가요?**
> 프로그램이 작동하려면 필요한 "다른 프로그램들"이에요.
> 비유: 케이크를 만들려면
> - 밀가루 필요 (= requests 라이브러리)
> - 계란 필요 (= beautifulsoup4 라이브러리)
> - 설탕 필요 (= pytest 라이브러리)

```bash
# 필수 라이브러리 설치
pip install requests beautifulsoup4 pytest

# 진행 화면:
Collecting requests
  Downloading requests-2.31.0...
Installing collected packages: requests, beautifulsoup4, pytest
Successfully installed! ✅
```

**각 라이브러리가 하는 일:**

| 라이브러리 | 역할 | 비유 |
|----------|------|------|
| `requests` | 웹사이트에서 데이터 가져오기 | 우편 배달부 📬 |
| `beautifulsoup4` | HTML 파싱 (분석) | 번역가 📖 |
| `pytest` | 테스트 자동화 | 품질 검사원 ✅ |

## 용어 사전

### 🔤 초보자를 위한 쉬운 용어 설명

#### A

**API (Application Programming Interface)**
```
어려운 설명: 프로그램 간 통신 인터페이스
쉬운 설명: 식당의 메뉴판

비유:
- 손님(당신) → 웨이터(API) → 주방(프로그램)
- 메뉴판에 있는 것만 주문 가능
- 주방이 어떻게 요리하는지 몰라도 OK!

예시:
"날씨 API"를 사용하면
→ 복잡한 기상 데이터 수집 방법 몰라도
→ "오늘 서울 날씨 알려줘" 요청만 하면
→ 결과를 받을 수 있어요!
```

#### C

**CLI (Command Line Interface)**
```
어려운 설명: 텍스트 기반 사용자 인터페이스
쉬운 설명: 글자로만 소통하는 프로그램

비유:
- GUI (클릭클릭) = 터치스크린 키오스크
- CLI (타이핑) = 문자 메시지로 주문

예시:
python3 doc_scraper.py --name react
         ↑ 프로그램         ↑ 옵션
"doc_scraper야, react라는 이름으로 실행해줘!"
```

**Config (Configuration)**
```
어려운 설명: 프로그램 설정 파일
쉬운 설명: 프로그램에게 주는 "주문서"

비유:
커피 주문서:
- 이름: 아메리카노

#### D

**Documentation (문서)**
```
어려운 설명: 소프트웨어 사용 설명서
쉬운 설명: 프로그램 사용법을 알려주는 책

비유:
- IKEA 가구 조립 설명서
- 요리 레시피북
- 게임 공략집

예시:
React 문서 = React 사용법 설명서
- 설치 방법
- 기본 사용법
- 고급 기능
- 예제 코드
```

**Dependency (의존성)**
```
어려운 설명: 프로그램 실행에 필요한 외부 라이브러리
쉬운 설명: 케이크를 만들 때 필요한 재료들

비유:
케이크 만들기:
✅ 밀가루 (필수!)
✅ 계란 (필수!)
✅ 설탕 (필수!)
❌ 밀가루 없으면 케이크 못 만들어요

프로그램 실행:
✅ requests (필수!)
✅ beautifulsoup4 (필수!)
❌ 라이브러리 없으면 프로그램 못 돌려요
```

#### E

**Enhancement (향상)**

어려운 설명: AI를 활용한 콘텐츠 품질 개선
쉬운 설명: 프로 요리사가 레시피를 더 맛있게 만들어주는 것

BEFORE:
"물 끓여서 라면 넣기"

```

#### G

**Git**
```
어려운 설명: 분산 버전 관리 시스템
쉬운 설명: 코드를 저장하고 공유하는 도서관

비유:
- GitHub = 온라인 도서관
- git clone = 책 빌리기
- git pull = 책 업데이트 내용 받기
- git push = 내가 쓴 내용 도서관에 기부

예시:
git clone [주소]
→ "이 프로젝트 복사본 주세요!"
```

#### M

**MCP (Model Context Protocol)**
```
어려운 설명: AI 모델 간 컨텍스트 공유 프로토콜
쉬운 설명: Claude에게 새로운 기술을 가르쳐주는 방법

비유:
학생 Claude가 수학만 알고 있었는데
MCP로 영어 교과서를 전달하면
→ 이제 영어도 할 수 있어요!

Skill Seeker + MCP:
→ Claude에게 "React 기술" 가르치기
→ 이제 Claude가 React 질문에 답할 수 있어요!
```

**Package (패키지)**
```
어려운 설명: 배포 가능한 형태로 압축된 소프트웨어
쉬운 설명: 선물 포장

비유:
상자에 담기 전:
📄 SKILL.md
📂 references/
📂 examples/

.zip으로 포장 후:

```

**Python**
```
어려운 설명: 고급 프로그래밍 언어
쉬운 설명: 컴퓨터와 대화하는 언어

비유:
- 한국어 = 사람끼리 대화
- Python = 컴퓨터와 대화
- 배우기 쉬운 외국어 같은 느낌!

예시:
print("안녕하세요")
→ 화면에 "안녕하세요" 출력

name = "철수"

```

#### S

**Scraping (스크래핑)**
```
어려운 설명: 웹 페이지에서 데이터 추출
쉬운 설명: 웹사이트 내용을 복사해오는 것

비유:
도서관에서 책 내용 필기하기
- 손으로 쓰기 (느림) ❌
- 복사기 사용 (빠름) ✅
→ Scraping = 자동 복사기!

예시:
웹사이트: "React는 UI 라이브러리입니다"
스크래핑: 이 문장을 파일로 저장
저장됨.txt: "React는 UI 라이브러리입니다"
```

**Skill**
```
어려운 설명: Claude AI 확장 기능 모듈
쉬운 설명: Claude에게 가르쳐주는 "전문 분야"

비유:
선생님(Claude)이 국어만 가르칠 수 있었는데
- "React Skill" 배움 → 이제 React도 가르쳐요
- "Python Skill" 배움 → Python도 가르쳐요
- "Godot Skill" 배움 → 게임 개발도 가르쳐요

효과:
❌ Skill 없을 때: "잘 모르겠어요..."
✅ Skill 있을 때: "제가 자세히 알려드릴게요! 😊"
```

#### T

**Terminal (터미널)**
```
어려운 설명: 명령줄 인터페이스 프로그램
쉬운 설명: 컴퓨터에게 문자로 명령하는 창

비유:
- 일반 앱 = 그림이 있는 메뉴판 (클릭클릭)
- 터미널 = 글자만 있는 주문서 (타이핑)

예시:
ls → "현재 폴더 파일 목록 보여줘"
cd Desktop → "Desktop 폴더로 이동해줘"
python3 test.py → "test.py 실행해줘"
```

#### V

**Virtual Environment (가상환경)**
```
어려운 설명: 격리된 Python 실행 환경
쉬운 설명: 각 프로젝트만의 독립된 작업실

비유:
집에 방이 3개:
- 방1 (프로젝트A): Python 3.10 + 라이브러리 A
- 방2 (프로젝트B): Python 3.11 + 라이브러리 B
- 방3 (프로젝트C): Python 3.9 + 라이브러리 C
→ 서로 간섭 안 해요!

왜 필요해요?
프로젝트A는 requests 2.0 필요
프로젝트B는 requests 3.0 필요
→ 같이 설치하면 충돌! 💥
→ 가상환경으로 분리! ✅
```

#### Z

**`.zip` 파일**
```
어려운 설명: 압축된 아카이브 파일
쉬운 설명: 여러 파일을 하나로 묶은 꾸러미

비유:
여행 가방:
- 옷
- 세면도구
- 책
전부 따로 들고 가기 힘들어요!
→ 가방 하나에 쏙! 💼

.zip 파일:
- SKILL.md
- references/ 폴더
- examples/ 폴더
전부 따로 업로드하기 힘들어요!
→ react.zip 하나로 쏙! 📦
```
