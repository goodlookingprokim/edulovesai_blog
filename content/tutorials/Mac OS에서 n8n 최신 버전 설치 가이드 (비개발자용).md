# Mac OS에서 n8n 최신 버전 설치 가이드 (비개발자용)

**사전 참고:** 과거 macOS용 n8n 데스크톱 앱(DMG 설치파일)이 제공되었으나 현재는 유지보수가 중단되어 다운로드할 수 없습니다. 따라서 GUI에 익숙한 사용자를 위해 **Docker Desktop**을 활용한 방법을 우선 권장하며, 터미널 명령어 사용을 최소화합니다. 또한 Node.js와 **npm**을 통한 설치 방법도 함께 안내합니다. 두 방법 모두 n8n 최신 버전을 설치하며, 자동 실행 및 백그라운드 실행 설정 방법을 포함합니다.

## 방법 1: Docker Desktop을 이용한 설치 (GUI 기반, 추천)

1. **Docker Desktop 설치:** Docker를 사용하면 복잡한 환경 설정 없이 n8n을 손쉽게 실행할 수 있습니다. 먼저 Mac용 Docker Desktop을 설치합니다. [Docker 공식 사이트](https://www.docker.com/products/docker-desktop/)에서 DMG 파일을 다운로드해 설치하거나, Homebrew가 있다면 다음 명령어로 설치할 수 있습니다:  
     
   brew install \--cask docker  
     
   설치 후 **Docker Desktop**을 실행하여 Docker 데몬(고래 아이콘)이 **실행 중**인지 확인합니다. 터미널에서 `docker --version` 명령으로 Docker가 정상 설치되었는지도 확인할 수 있습니다.  
     
2. **n8n 컨테이너 다운로드 및 실행:** Docker Desktop이 실행 중이라면, 터미널에서 다음 명령어를 입력하여 n8n Docker 이미지를 다운로드하고 컨테이너를 실행합니다:  
     
   docker run \-d \--name n8n \-p 5678:5678 \-v \~/.n8n:/home/node/.n8n n8nio/n8n  
     
   * `-d` 옵션은 **백그라운드(detached) 실행**을 의미합니다. 이 옵션을 사용하면 터미널을 닫아도 컨테이너가 계속 실행됩니다.  
   * `--name n8n`은 컨테이너 이름을 "n8n"으로 지정하는 것으로, 나중에 관리할 때 편리합니다.  
   * `-p 5678:5678`는 **호스트의 포트 5678**을 컨테이너의 포트 5678에 매핑합니다. n8n의 기본 웹 인터페이스 포트가 5678이므로, 실행 후 브라우저에서 `localhost:5678`로 접근할 수 있게 됩니다.  
   * `-v ~/.n8n:/home/node/.n8n`는 **볼륨 마운트** 옵션으로, 호스트 사용자의 홈 디렉토리에 있는 `~/.n8n` 폴더를 컨테이너 내부의 `/home/node/.n8n` 경로에 연결합니다. 이를 통해 워크플로우, 자격증명 등 n8n의 데이터를 호스트에 영구 보존할 수 있습니다. (컨테이너를 재시작하거나 이미지 업그레이드해도 데이터 손실이 없도록 설정)

   

3. **자동 시작 설정:** 위 명령어로 n8n 컨테이너를 실행하면, n8n이 백그라운드에서 계속 실행됩니다. 추가로, Mac 재부팅이나 로그아웃 후에도 n8n이 자동으로 실행되도록 설정할 수 있습니다.  
     
   * **Docker Desktop 자동 실행:** Docker Desktop 앱 메뉴의 \*\*Preferences(설정)\*\*에서 "로그인 시 Docker Desktop 실행(Start Docker Desktop when you log in)" 옵션을 활성화합니다. 이 옵션을 켜두면 Mac 로그인 시 Docker 데몬이 자동으로 시작됩니다.  
       
   * **컨테이너 자동 재시작:** n8n 컨테이너를 실행할 때 `--restart` 정책을 지정하면 Docker 데몬 시작 시 컨테이너를 자동으로 다시 실행할 수 있습니다. 예를 들어 앞서 실행한 명령에 `--restart unless-stopped` 옵션을 추가하여 실행하면, 수동으로 컨테이너를 중지하지 않는 한 Mac 재부팅 시 n8n 컨테이너가 자동으로 시작됩니다. 전체 명령은 다음과 같습니다:  
       
     docker run \-d \--restart unless-stopped \--name n8n \-p 5678:5678 \-v \~/.n8n:/home/node/.n8n n8nio/n8n

     
   이로써 사용자는 일일이 터미널을 열지 않고도 시스템 부팅과 함께 n8n 서버를 항상 켤 수 있습니다.  
     
4. **설치 확인 및 n8n 접속:** Docker 컨테이너가 정상적으로 실행되었다면, 웹 브라우저를 열고 \*\*`http://localhost:5678`\*\*에 접속합니다. n8n의 워크플로우 에디터 UI가 나타나면 설치 성공입니다. 이제 n8n 최신 버전을 로컬에서 무료로 사용할 수 있으며, Docker를 통해 백그라운드에서 안정적으로 구동됩니다. (참고로 2025년 기준 n8n의 최신 버전은 1.x 대이며, Docker `n8nio/n8n:latest` 태그로 항상 최신 안정 버전이 설치됩니다.)

**Tip:** 이후 n8n을 업데이트하려면 최신 이미지를 받고 컨테이너를 다시 생성하면 됩니다. 예를 들어 `docker pull n8nio/n8n:latest`로 이미지를 업데이트한 뒤 기존 컨테이너를 삭제(`docker rm n8n`), 다시 위의 `docker run -d ...` 명령어를 실행하면 최신 버전으로 구동됩니다. 또한, `docker ps`로 실행 중인 컨테이너 목록을 보고 `docker stop n8n` / `docker start n8n`으로 n8n을 중지/재개할 수 있습니다.

## 방법 2: Node.js와 npm을 이용한 설치 (터미널 기반, 비개발자도 따라하기 쉬운 방법)

Docker 사용이 어렵거나, 별도의 컨테이너 없이 직접 n8n을 실행하고 싶다면 \*\*Node.js (npm)\*\*로 설치할 수 있습니다. 약간의 터미널 사용이 필요하지만, 차근차근 따라 하면 비개발자도 쉽게 설치할 수 있습니다.

1. **Node.js 설치 (버전 18 이상):** n8n은 Node.js 런타임에서 실행되므로, 먼저 Node.js를 설치해야 합니다. **Node 18 LTS** 이상의 버전이 필요하며, 최신 LTS 버전을 권장합니다.  
     
   * **Homebrew로 설치:** Mac에 Homebrew가 설치되어 있다면 터미널에서 다음을 실행하세요:  
       
     brew install node  
       
     명령 실행 후 `node -v`와 `npm -v`로 Node.js와 npm이 정상 설치되었는지 버전을 확인합니다.  
       
   * **공식 패키지로 설치:** Homebrew를 사용하지 않는 경우, [Node.js 공식 다운로드 페이지](https://nodejs.org)에서 macOS용 설치 패키지를 받아 **Node.js 최신 LTS 버전**을 설치하세요. 설치 마법사를 완료한 후 터미널을 열어 `node -v` 명령으로 버전이 표시되는지 확인합니다 (설치 후 npm도 함께 제공됩니다).

   

2. **n8n 최신 버전 설치:** Node.js 환경이 갖춰졌으면, 터미널에 다음 명령을 입력하여 n8n을 전역(global) 설치합니다:  
     
   npm install \-g n8n  
     
   npm이 n8n 패키지를 네트워크에서 내려받아 설치하며, 완료까지 수 분 정도 소요될 수 있습니다. 설치가 끝나면 `n8n --version` 명령으로 n8n 버전을 확인해볼 수 있습니다 (최신 버전 번호가 출력되면 성공).  
     
3. **n8n 실행 및 접속:** n8n이 설치되었으면 바로 실행할 수 있습니다. 터미널에서 다음을 입력하세요:  
     
   n8n start  
     
   해당 명령을 실행하면 터미널에 n8n 서버 로그가 나타나면서 백엔드가 구동됩니다. 이제 웹 브라우저로 \*\*`http://localhost:5678`\*\*에 접속하면 n8n 에디터 화면을 볼 수 있습니다. (기본 설정상 로컬호스트에서 누구나 접근 가능하므로, 필요 시 추후 환경변수로 기본 인증을 거는 것도 고려하세요.)  
     
4. **터미널을 닫아도 계속 실행되도록 설정 (PM2 이용):** 위 단계까지는 터미널 세션에서 n8n이 실행 중인 상태입니다. 이 터미널을 닫거나 Mac을 재부팅하면 n8n 프로세스가 종료되므로, **PM2**라는 Node.js용 프로세스 매니저를 사용하여 n8n을 백그라운드 서비스로 등록합니다. PM2를 사용하면 터미널을 열지 않아도 n8n이 자동으로 실행되고 유지됩니다. 설정 방법은 다음과 같습니다.  
     
   * **PM2 전역 설치:** 터미널에 다음을 입력하여 PM2를 설치합니다:  
       
     npm install \-g pm2  
       
     (한번만 설치하면 되며, PM2 자체도 Node.js로 동작합니다.)  
       
   * **n8n 프로세스 PM2로 시작:** PM2 설치가 완료되면, n8n을 PM2로 실행시킵니다:  
       
     pm2 start n8n  
       
     이 명령을 실행하면 PM2가 n8n 프로세스를 백그라운드에서 관리하게 됩니다. `pm2 list` 명령으로 n8n 프로세스가 PM2에 등록되어 **online** 상태로 표시되는지 확인할 수 있습니다. 이제 터미널을 닫아도 n8n이 계속 실행됩니다.  
       
   * **Mac 부팅 시 자동 시작 설정:** n8n을 항상 켜두고 사용하기 위해, PM2에 **시스템 시작 시 자동으로 실행**하도록 설정합니다. 아래 두 개의 명령을 차례로 실행하세요:  
       
     pm2 startup  
       
     pm2 save  
       
     첫 번째 `pm2 startup` 명령은 현재 사용자 환경에 맞는 PM2 런치데몬(Launch Daemon) 스크립트를 생성합니다. 실행 후 터미널에 나타나는 출력 지시에 따라 추가 명령을 한 번 더 실행해야 할 수도 있습니다 (예: `sudo ... pm2 startup ...` 형태의 명령을 안내해 줍니다). 두 번째 `pm2 save` 명령은 현재 관리 중인 프로세스 목록을 저장하여, 이후 부팅 시 자동으로 복원하도록 설정합니다. 이 과정을 마치면 Mac이 재부팅되거나 로그인할 때 PM2가 n8n 프로세스를 자동으로 다시 시작해줍니다.

     
   이제 n8n이 진정한 백그라운드 서비스로 동작하며, Mac을 켜는 것만으로 n8n이 실행됩니다. (참고: PM2로 관리 중인 프로세스를 중지하거나 재시작하고 싶다면 `pm2 stop n8n` / `pm2 restart n8n` 명령을 사용할 수 있습니다.)  
     
5. **설치 및 실행 확인:** `pm2 save`까지 마쳤다면 설정 완료입니다. Mac을 재부팅한 후 터미널을 전혀 건드리지 않고 브라우저로 `localhost:5678`에 접속해 보세요. n8n 로그인 화면 또는 에디터 UI가 나타나면 성공입니다. 이렇게 설치된 n8n은 **최신 버전**이며, 이후 새로운 버전이 나오면 `npm update -g n8n`으로 손쉽게 업그레이드할 수 있습니다.

## 참고 및 추가 설정

* **기본 인증 활성화:** 로컬에 설치된 n8n은 기본적으로 인증 없이 웹에 접근이 가능합니다. 개인 PC에서만 사용한다면 큰 문제가 없지만, 보안을 위해 **기본 인증**을 거는 것이 좋습니다. Docker 사용자의 경우 환경변수 `N8N_BASIC_AUTH_ACTIVE=true`와 아이디/패스워드를 설정하여 컨테이너를 실행하면 되고, PM2를 사용하는 경우 `N8N_BASIC_AUTH_ACTIVE=true` 등의 환경변수를 PM2에 등록하거나 `.env` 파일을 사용하면 됩니다. 이 설정을 하면 웹 브라우저에서 n8n에 접속할 때 로그인 창이 나타나게 됩니다.  
    
* **업데이트 및 유지관리:** Docker 방식이든 npm 방식이든, n8n는 주기적으로 업데이트가 이루어집니다. 새로운 기능을 사용하고 보안 패치를 적용하려면 정기적으로 업데이트하는 것이 좋습니다. Docker는 앞서 언급한 대로 이미지만 교체하면 되고, npm 설치의 경우 PM2로 구동 중인 n8n을 업데이트할 때는 `pm2 stop n8n` → `npm install -g n8n@latest` → `pm2 start n8n` 순으로 수행하면 됩니다.

以上와 같은 과정으로 **비개발자도 Mac OS에서 n8n 최신 버전을 손쉽게 설치**할 수 있습니다. Docker Desktop을 이용하면 GUI 환경에서 컨테이너를 관리할 수 있어 편리하며, Node.js 방법은 시스템에 직접 n8n을 설치하지만 가벼운 개발 지식만으로도 설정이 가능합니다. 두 방식 모두 n8n을 백그라운드에서 상시 실행하도록 구성할 수 있으므로, 개인용 자동화 워크플로우 서버로 활용할 수 있습니다. 이제 자유롭게 n8n으로 다양한 워크플로우 자동화를 시도해보세요\! 🚀
