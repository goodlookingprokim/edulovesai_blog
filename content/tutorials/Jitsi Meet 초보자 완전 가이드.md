## 들어가며 - 화상회의의 새로운 세계

### 📖 이야기로 시작하기

여러분은 친구들과 온라인으로 모임을 갖거나, 원격 수업을 듣거나, 재택근무를 하면서 화상회의를 사용해본 적이 있나요?

민수는 학교 동아리 회의를 위해 화상회의 프로그램이 필요했습니다. 하지만 문제가 있었죠.

```
문제 1: Zoom은 40분 제한이 있어요 😢
문제 2: 프로그램 설치가 귀찮아요 😫
문제 3: 개인정보가 걱정돼요 😰
문제 4: 서버가 해외에 있어서 느려요 🐌
```

그때 민수는 **Jitsi Meet**를 발견했습니다!

```
해결 1: 시간 제한 없음! ⏰
해결 2: 웹 브라우저만 있으면 OK! 🌐
해결 3: 오픈소스라 투명해요! 🔓
해결 4: 내 서버에 직접 설치 가능! 🖥️
```

자, 이제 Jitsi Meet의 세계로 함께 떠나볼까요?

## Jitsi Meet란 무엇인가

### 🎯 핵심 정의 (3단계로 이해하기)

#### 초등학생에게 설명한다면
"인터넷으로 친구들과 얼굴 보면서 이야기할 수 있는 무료 프로그램이에요. 마치 전화기로 영상통화 하는 것처럼요!"

#### 중학생에게 설명한다면
"Zoom이나 Google Meet처럼 화상회의를 할 수 있는 프로그램인데, 무료로 쓸 수 있고 소스코드가 공개되어 있어서 누구나 자신만의 서버를 만들 수 있어요."

#### 고등학생/성인에게 설명한다면
"WebRTC 기술을 기반으로 한 오픈소스 화상회의 플랫폼으로, HD 화질의 실시간 영상/음성 통신, 화면 공유, 채팅 등의 기능을 제공하며, 자체 서버 구축이 가능한 완전한 화상회의 솔루션입니다."

## 왜 Jitsi Meet를 배워야 할까

### 💡 학습 단계별 이유

#### 🥉 초보자 레벨
**목표**: Jitsi Meet를 사용자로서 활용하기

"나는 왜 Jitsi Meet를 써야 할까요?"

1. **완전 무료**: 시간 제한 없이 무료로 사용
2. **쉬운 사용**: 링크 하나만 공유하면 끝
3. **설치 불필요**: 브라우저만 있으면 OK
4. **개인정보 보호**: 계정 없이도 사용 가능

**실제 사례**:
```
상황: 학교 조별 과제 회의
문제: Zoom 40분 제한으로 회의가 끊김
해결: Jitsi Meet로 3시간 회의 진행
결과: 추가 비용 없이 프로젝트 완성! 🎉
```

#### 🥈 중급자 레벨
**목표**: Jitsi Meet를 커스터마이징하여 활용하기

"내 웹사이트에 화상회의 기능을 추가하고 싶어요!"

1. **SDK 제공**: 내 웹사이트에 쉽게 통합
2. **커스터마이징**: 로고, 색상 등 변경 가능
3. **API 활용**: 자동화 및 확장 가능
4. **무료 라이선스**: 상업적 사용도 가능

**실제 사례**:
```
상황: 온라인 교육 플랫폼 개발
요구사항: 화상 수업 기능 필요
문제: 타 서비스는 월 수백만원 비용
해결: Jitsi Meet SDK 통합
결과: 개발 비용만으로 기능 구현! 💰
```

#### 🥇 고급자 레벨
**목표**: 자체 Jitsi Meet 서버 구축 및 운영

"회사/학교 전용 화상회의 시스템을 만들고 싶어요!"

1. **완전한 통제**: 모든 데이터가 내 서버에
2. **보안 강화**: 외부 서버 경유 없음
3. **성능 최적화**: 내 네트워크에 맞게 조정
4. **비용 절감**: 대규모 사용 시 경제적

**실제 사례**:
```
상황: 중소기업의 재택근무 전환
요구사항: 100명 동시 화상회의
기존 솔루션: 월 500만원 이상
Jitsi Meet 도입: 서버 비용 월 50만원
결과: 연간 5,400만원 절감! 🚀
```

## Jitsi Meet 사용하기 - 기초편

### 🚀 첫 시작: 10초 만에 회의 시작하기

#### 📝 단계별 가이드 (초보자)

**STEP 1: 웹사이트 방문**
```
1. 웹 브라우저를 엽니다 (크롬, 사파리, 파이어폭스 등)
2. 주소창에 입력: meet.jit.si
3. Enter 키를 누릅니다
```

**STEP 2: 회의 이름 정하기**
```
화면에 "Start a new meeting" 입력창이 보입니다

좋은 회의 이름 예시:
✅ MyTeamMeeting123
✅ ProjectDiscussion0113
✅ FamilyVideoCall

나쁜 회의 이름 예시:
❌ 123 (너무 짧음)
❌ meeting (너무 흔함)
❌ 회의 (한글은 URL에 부적합)
```

**STEP 3: 회의 시작!**
```
1. 회의 이름을 입력합니다
2. "Start meeting" 버튼을 클릭합니다
3. 카메라/마이크 권한을 허용합니다
4. 완료! 🎉
```

### 🎬 화면 구성 이해하기

```
┌─────────────────────────────────────────────────────────┐
│  [<] Jitsi Meet          MyMeeting123    [최소화] [종료] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│            📹 상대방 비디오 화면                         │
│                                                         │
│  ┌─────────────────┐                                   │
│  │  😊 내 비디오   │  ← 작은 화면: 내 모습              │
│  └─────────────────┘                                   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [🎤] [📹] [✋] [💬] [👥] [📺] [⚙️] [📞]  ← 컨트롤바    │
└─────────────────────────────────────────────────────────┘

아이콘 설명:
🎤 = 마이크 켜기/끄기
📹 = 카메라 켜기/끄기
✋ = 손들기
💬 = 채팅
👥 = 참가자 목록
📺 = 화면 공유
⚙️ = 설정
📞 = 통화 종료
```

### 🎯 주요 기능 사용하기

#### 1️⃣ 마이크 조절하기

**초보자 버전**:
```
마이크 아이콘(🎤)을 클릭하면:
- 빨간색 = 음소거 상태 (내 목소리가 안 들림)
- 흰색 = 켜진 상태 (내 목소리가 들림)

💡 팁: 말하지 않을 때는 음소거해두세요!
      배경 소음이 다른 사람들에게 방해가 됩니다.
```

**중급자 버전**:
```
마이크 버튼을 길게 누르면:
- 다른 마이크 선택 가능
- 음량 조절 가능
- 노이즈 억제 설정 가능

💡 팁: "Start Muted" 옵션을 켜면
      회의 입장 시 자동으로 음소거됩니다.
```

**고급자 버전**:
```
키보드 단축키:
- Ctrl + D (Windows) / Cmd + D (Mac): 음소거 토글
- M 키: 빠른 음소거
- Space 바 (길게): 일시적으로 음소거 해제

API를 통한 제어:
api.executeCommand('toggleAudio');
```

#### 2️⃣ 화면 공유하기

**시나리오 1: 프레젠테이션 공유**
```
상황: PPT 발표를 하고 싶어요

단계:
1. 화면 공유 버튼(📺) 클릭
2. 공유 옵션 선택:
   - "전체 화면" 선택
   - 발표 자료가 있는 화면 선택
3. "공유" 버튼 클릭
4. 발표 시작!

💡 팁: 발표 전에 불필요한 창은 모두 닫으세요!
      알림도 끄는 것이 좋습니다.
```

**시나리오 2: 특정 프로그램만 공유**
```
상황: 크롬 브라우저만 보여주고 싶어요

단계:
1. 화면 공유 버튼(📺) 클릭
2. "애플리케이션 윈도우" 탭 선택
3. "Chrome" 선택
4. "공유" 버튼 클릭

💡 팁: 다른 프로그램은 안 보이니 안심!
      개인 메시지나 민감한 정보 보호됨.
```

#### 3️⃣ 채팅 사용하기

**기본 채팅**:
```
채팅 버튼(💬) 클릭 → 오른쪽에 채팅창 열림

사용 예시:
"안녕하세요!" → 전체 메시지
링크 공유: https://example.com → 자동으로 클릭 가능한 링크
```

**프라이빗 채팅**:
```
1. 참가자 목록(👥) 클릭
2. 특정 사람 이름 클릭
3. "Private message" 선택
4. 1:1 대화 시작

💡 팁: 프라이빗 메시지는 다른 사람에게 안 보입니다!
```

#### 4️⃣ 손들기 기능

**언제 사용할까?**
```
✅ 질문하고 싶을 때
✅ 의견을 내고 싶을 때
✅ 다음 발언권을 받고 싶을 때

손들기 버튼(✋) 클릭하면:
- 내 이름 옆에 손 아이콘 표시
- 참가자 목록에서 손 든 사람 확인 가능
- 발표자가 나를 지목할 수 있음
```

#### 5️⃣ 가상 배경 설정

**초보자 가이드**:
```
1. 설정 버튼(⚙️) 클릭
2. "Virtual Background" 선택
3. 기본 제공 배경 중 선택
4. "Apply" 클릭

제공되는 배경:
- 블러 (배경 흐리게)
- 오피스
- 자연 풍경
- 추상 패턴
```

**중급자 가이드**:
```
내 이미지 업로드:
1. "Virtual Background" 메뉴
2. "+" 버튼 클릭
3. 이미지 파일 선택 (JPG, PNG)
4. 적용

💡 팁:
- 이미지 크기: 1920x1080 권장
- 파일 크기: 2MB 이하 권장
- 단색 배경보다 패턴이 있는 배경 추천
```

### 👥 사람들 초대하기

#### 방법 1: 링크 공유 (가장 쉬움!)

```
1. 회의 링크 복사하기:
   - 주소창의 URL 복사
   - 예: https://meet.jit.si/MyMeeting123

2. 링크 공유하기:
   - 카카오톡으로 전송
   - 이메일로 전송
   - 문자로 전송

3. 친구들이 링크 클릭!
   - 자동으로 회의에 참가됨
```

**실제 예시**:
```
보내는 메시지:
"안녕! 오늘 3시에 화상회의 해요.
아래 링크로 들어오세요!

https://meet.jit.si/ProjectMeeting0113

카메라랑 마이크 켜고 들어오면 돼요 😊"
```

#### 방법 2: 회의 정보 공유

```
회의 정보:
- 서버: meet.jit.si

- 시간: 2025년 10월 13일 오후 3시

참가 방법:
1. meet.jit.si 접속
2. "MyMeeting123" 입력
3. "Join" 클릭
```

#### 방법 3: 초대 버튼 사용

```
1. 참가자 아이콘(👥) 클릭
2. "Invite more people" 버튼
3. 이메일 주소 입력
4. 초대 이메일 자동 발송!
```

```
비유:
일반 설치 = 집을 직접 짓기 (복잡!)
```

```bash

# - git clone: 깃허브에서 코드 복사
# - cd: 디렉토리로 이동

cp env.example .env
./gen-passwords.sh

# - gen-passwords.sh: 보안 비밀번호 자동 생성

docker-compose up -d

# 설명:
# - docker-compose up: 컨테이너 시작
# - -d: 백그라운드에서 실행
```

## Jitsi as a Service 활용하기

### 🌐 JaaS란 무엇인가?

**간단한 설명**:
```
JaaS = Jitsi as a Service
= "서비스로서의 Jitsi"

즉, Jitsi의 장점은 그대로 +
서버 관리는 8x8 회사가 해줌!

비유:
자체 서버 = 자가용 (내가 관리)
JaaS = 택시/카셰어링 (회사가 관리)
```

### 💰 비용 구조

**무료 플랜**:
```
제공 내용:
- 월 10,000분 무료
- 기본 기능 사용
- 커스터마이징 제한적

계산 예시:
- 10명 × 1시간 회의 = 600분
- 월 16회 정도 회의 가능
- 소규모 팀에 적합!
```

**유료 플랜**:
```
종량제:
- 추가 1분당 $0.0X (정확한 가격은 웹사이트 확인)
- 사용한 만큼만 지불

월정액:
- 무제한 사용
- 고급 기능 제공
- 전용 지원

💡 장점:
- 서버 관리 필요 없음
- 자동 업데이트
- 안정적인 성능 보장
```

### 🚀 JaaS 시작하기

**STEP 1: 계정 만들기**
```
1. https://jaas.8x8.vc 접속
2. "Sign up" 버튼 클릭
3. 이메일 주소 입력
4. 비밀번호 설정
5. 이메일 인증
6. 완료!
```

**STEP 2: API 키 발급**
```
1. 대시보드 로그인
2. "API Keys" 메뉴
3. "Generate new key" 클릭
4. 키 이름 입력 (예: "MyWebsite")
5. API 키 복사 (안전하게 보관!)
```

**STEP 3: 웹사이트에 통합**
```html
<!DOCTYPE html>
<html>
<head>
    <title>우리 회사 회의실</title>
    <script src='https://8x8.vc/vpaas-magic-cookie-YOUR-TENANT-ID/external_api.js'></script>
</head>
<body>
    <div id="meet"></div>

    <script>
        // Jitsi Meet 초기화
        const domain = '8x8.vc';
```

### 🔒 Jitsi Meet는 얼마나 안전한가?

#### 기본 보안 수준

```
보안 요소 분석:

1. 암호화 통신 ✅
   - 모든 데이터는 암호화됨 (HTTPS)
   - 영상/음성도 암호화 전송

2. 오픈소스 ✅
   - 코드가 공개됨
   - 누구나 검증 가능
   - 백도어 위험 낮음

3. 계정 불필요 ✅
   - 개인정보 수집 최소화
   - 이메일/전화번호 불필요

4. 데이터 저장
   - 기본적으로 저장 안 함
   - 녹화 시에만 저장 (선택사항)
```

### 🛡️ End-to-End 암호화 (E2EE)

**E2EE란?**
```
일반 암호화:
나 → [암호화] → 서버 → [복호화] → 상대방
        ↑ 서버는 내용을 볼 수 있음!

End-to-End 암호화:
나 → [암호화] ────────────→ [복호화] → 상대방
              서버는 내용을 못 봄!

비유:
일반 우편 = 우체국 직원이 볼 수 있음
E2EE = 자물쇠로 잠긴 상자, 받는 사람만 열쇠 있음
```

**E2EE 활성화 방법**:

```markdown
STEP 1: 회의 시작
1. 회의 입장
2. 모든 참가자 입장 대기

STEP 2: E2EE 켜기
1. 보안 아이콘(🔒) 클릭
2. "Enable E2E Encryption" 토글
3. 경고 메시지 확인:
   "일부 기능이 비활성화됩니다"
4. "Enable" 버튼 클릭

STEP 3: 모든 참가자 확인
1. 각 참가자의 아이콘 확인
2. 녹색 자물쇠 = E2EE 활성
3. 빨간색 = E2EE 비활성

주의사항:
⚠️ E2EE 활성화 시 사용 불가:
   - 서버 측 녹화
   - 전화 접속
   - 라이브 스트리밍
```

#### 문제 1: "상대방 목소리가 안 들려요!"

**체크리스트**:
```markdown
□ 내 볼륨 확인
   - 컴퓨터 볼륨 크기 확인
   - 헤드폰/스피커 연결 확인

□ 상대방 음소거 확인
   - 상대방 마이크 아이콘이 빨간색?
   - 음소거 상태일 수 있음

□ 브라우저 권한 확인
   - 주소창 옆 아이콘 클릭
   - 마이크 권한 "허용"으로 변경

□ 오디오 장치 선택
   - 설정 → "Audio"
   - 올바른 스피커 선택
```

**단계별 해결**:
```markdown
STEP 1: 기본 확인
1. 다른 탭에서 YouTube 재생 테스트
2. 소리 나오면 → Jitsi 설정 문제
3. 안 나오면 → 컴퓨터 설정 문제

STEP 2: Jitsi 설정
1. 설정(⚙️) 클릭
2. "Audio" 탭
3. "Output device" 드롭다운
4. 다른 장치 선택해보기

STEP 3: 브라우저 재시작
1. Jitsi 탭 닫기
2. 브라우저 완전 종료
3. 다시 시작
4. 회의 재입장

STEP 4: 다른 브라우저 시도
- Chrome 안 되면 Firefox 시도
- Safari 안 되면 Chrome 시도
```

```markdown
5. 모니터링 대시보드 구축
6. 사용자 교육 자료 제작

배우는 것:
- 대규모 시스템 설계
- 부하 분산
- 엔터프라이즈 통합
- 프로젝트 관리
```

## 용어 사전

### 🔤 기술 용어

#### A-E

**API (Application Programming Interface)**
```
초급: 프로그램들이 서로 대화하는 방법
중급: 소프트웨어 간 통신을 위한 규칙과 정의
고급: 특정 기능이나 데이터에 접근하기 위한
      프로토콜, 도구, 정의의 집합

예시:
날씨 앱이 기상청 서버에 "오늘 날씨 알려줘"라고 요청
→ API를 통해 데이터 받아옴
```

**대역폭 (Bandwidth)**
```
초급: 인터넷 속도
중급: 일정 시간 동안 전송 가능한 데이터양
고급: 네트워크에서 주어진 시간 동안 전송할 수 있는
      최대 데이터 전송률 (bps, Mbps)

비유:
수도관의 두께 = 대역폭
물의 양 = 데이터
```

**컨테이너 (Container)**
```
초급: 프로그램을 포장한 상자
중급: 애플리케이션과 모든 필요한 것을 포함한 패키지
고급: OS 수준의 가상화를 이용하여 격리된 환경에서
      애플리케이션을 실행하는 기술

Docker 컨테이너 = 가장 유명한 컨테이너 시스템
```

**도메인 (Domain)**
```
초급: 웹사이트 주소 (예: google.com)
중급: IP 주소를 사람이 읽기 쉬운 이름으로 변환
고급: 인터넷에서 컴퓨터나 네트워크를 식별하기 위한
      고유한 이름 체계

예시:
meet.jit.si → 도메인
64.233.177.113 → IP 주소 (컴퓨터는 이걸 씀)
```

**E2EE (End-to-End Encryption)**
```
초급: 나랑 상대방만 볼 수 있는 암호
중급: 데이터가 송신자에서 수신자까지 암호화된 상태 유지
고급: 통신 양 끝단에서만 암호화/복호화가 일어나며,
      중간 서버는 내용을 알 수 없는 암호화 방식

WhatsApp, Signal이 E2EE 사용
```

#### F-O

**방화벽 (Firewall)**
```
초급: 나쁜 사람들이 못 들어오게 하는 문지기
중급: 네트워크 트래픽을 감시하고 제어하는 보안 시스템
고급: 미리 정의된 보안 규칙에 따라 들어오고 나가는
      네트워크 트래픽을 모니터링하고 제어하는 시스템

집의 현관문 = 방화벽
초인종 누르면 확인하고 문 열어줌
```

**HTTPS (HTTP Secure)**
```
초급: 안전한 웹사이트 (자물쇠 표시)
중급: 암호화된 HTTP 통신
고급: SSL/TLS 프로토콜을 사용하여 데이터를
      암호화하는 HTTP의 보안 버전

주소창에 자물쇠 🔒 = HTTPS
```

**IP 주소 (IP Address)**
```
초급: 인터넷상의 집 주소
중급: 네트워크에 연결된 기기의 고유 식별자
고급: 인터넷 프로토콜을 사용하는 네트워크에서
      기기를 식별하기 위한 숫자 주소

예시:
IPv4: 192.168.0.1 (옛날 방식)
IPv6: 2001:0db8:85a3::8a2e:0370:7334 (최신)
```

**지연시간 (Latency)**
```
초급: 말하고 나서 상대방이 듣기까지 걸리는 시간
중급: 데이터 전송 요청부터 응답까지의 시간
고급: 패킷이 출발지에서 목적지까지 도달하는 데
      걸리는 시간 (RTT - Round Trip Time)

낮을수록 좋음!
게임: <50ms (매우 좋음)
화상회의: <150ms (괜찮음)
```

**오픈소스 (Open Source)**
```
초급: 소스 코드를 누구나 볼 수 있는 프로그램
중급: 코드가 공개되어 자유롭게 사용, 수정, 배포 가능
고급: 특정 라이선스 하에 소스 코드를 공개하여
      협업적 개발과 투명성을 촉진하는 소프트웨어

Jitsi Meet = 오픈소스!
Windows = 오픈소스 아님 (proprietary)
```

#### P-Z

**P2P (Peer-to-Peer)**
```
초급: 서버 거치지 않고 직접 연결
중급: 중앙 서버 없이 기기 간 직접 통신
고급: 분산 네트워크에서 각 노드가 동등한 위치에서
      자원을 공유하는 통신 방식

비유:
일반 방식: A → 서버 → B (우체국 경유)
P2P: A ↔ B (직접 전달)

장점: 빠르고, 서버 부하 적음
단점: 방화벽 문제 가능
```

**포트 (Port)**
```
초급: 집의 여러 개 문 (현관, 뒷문...)
중급: 특정 프로그램으로 데이터를 보내는 번호
고급: TCP/IP 네트워크에서 프로세스를 구분하기 위한
      논리적 연결 지점 (0-65535)

예시:
HTTP: 80번 포트
HTTPS: 443번 포트
Jitsi: 443, 10000 포트 등
```

**SSL/TLS**
```
초급: 인터넷을 안전하게 만드는 기술
중급: 네트워크 통신을 암호화하는 프로토콜
고급: Transport Layer Security - 클라이언트와 서버 간
      데이터를 암호화하여 안전한 통신을 보장

HTTPS의 'S'가 바로 SSL/TLS!
```

**SDK (Software Development Kit)**
```
초급: 프로그램 만들 때 쓰는 도구 모음
중급: 특정 플랫폼이나 프레임워크를 위한 개발 도구 세트
고급: API, 라이브러리, 문서, 샘플 코드 등을 포함한
      소프트웨어 개발을 위한 통합 도구 모음

Jitsi Meet SDK = Jitsi 기능을 내 앱에 추가하는 도구
```

**WebRTC**
```
초급: 브라우저에서 바로 영상통화 하는 기술
중급: 웹 브라우저 간 실시간 통신을 가능케 하는 기술
고급: 플러그인 없이 웹 브라우저 및 모바일 앱에서
      음성, 영상, 데이터를 P2P로 전송하는 오픈소스 프로젝트

Jitsi Meet의 핵심 기술!
Zoom, Google Meet도 WebRTC 사용
```

```
회의를 시작한 사람

- 참가자 추방
- 회의 잠금
- 설정 변경
```

```
회의에 들어온 사람들

역할:
- 발언하기
- 듣기
- 채팅하기
```

```
회의를 관리하는 사람
= 진행자, 사회자

- 발언 순서 정하기
- 시간 관리
- 분위기 조성
```

```
대기실

작동:
1. 참가자가 로비에서 대기
2. 호스트가 승인
3. 회의실 입장!
```

```
음소거 = 내 목소리 안 들림

단축키:
- Ctrl + D (Windows)
- Cmd + D (Mac)
- M 키
```

**스크린 쉐어 (Screen Share)**
```
화면 공유
= 내 컴퓨터 화면을 다른 사람에게 보여주기

활용:
- 프레젠테이션
- 코드 설명
- 동영상 함께 보기
```
