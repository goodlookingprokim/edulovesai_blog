## 이 가이드는 누구를 위한 것인가요?

### ✅ 이런 분들께 추천합니다

#### 📚 프로그래밍 입문자
- **상황**: "코딩을 배우고 싶은데 뭐부터 해야 할지 모르겠어요"
- **이 도구가 도움이 되는 이유**: AI가 친절한 선생님처럼 코드를 설명해줍니다

#### 🤔 코드 이해가 어려운 분
- **상황**: "다른 사람이 쓴 코드를 봐도 무슨 말인지 모르겠어요"
- **이 도구가 도움이 되는 이유**: 복잡한 코드를 쉬운 말로 풀어서 설명해줍니다

#### 🐛 에러 메시지가 두려운 분
- **상황**: "빨간 글씨만 보면 머리가 아파요"
- **이 도구가 도움이 되는 이유**: 에러를 한국어로 설명하고 해결 방법을 알려줍니다

#### 💻 터미널이 낯선 분
- **상황**: "검은 화면이 무서워요"
- **이 도구가 도움이 되는 이유**: 터미널을 친근한 대화 창으로 만들어줍니다

## Kilo Code CLI란 무엇인가요?

### 🎯 일상 비유로 이해하기

#### 비유 1: AI 개인 코딩 선생님
```
상황을 상상해보세요...

당신은 수학 문제를 풀고 있습니다.
옆에 친절한 선생님이 앉아 계십니다.

"선생님, 이 문제 어떻게 푸는 거예요?"
→ 선생님이 차근차근 설명해줍니다.

"여기서 실수한 것 같은데, 어디가 틀렸을까요?"
→ 선생님이 틀린 부분을 찾아 고쳐줍니다.

Kilo Code CLI는 바로 이런 선생님입니다!
단, 수학이 아니라 '코딩'을 가르쳐주는 선생님이죠.
```

#### 비유 2: 번역기 + 도우미
```
외국어 책을 읽는다고 생각해보세요.

어려운 단어가 나올 때마다
→ 번역기에게 물어봅니다.

문장 구조가 이해 안 될 때
→ 번역기가 쉽게 풀어서 설명해줍니다.

Kilo Code CLI는 '코드 번역기'입니다!
복잡한 코드를 쉬운 말로 번역해줍니다.
```

### 🔍 정확한 정의

**Kilo Code CLI**는:
1. **터미널에서 실행**되는 프로그램입니다
2. **AI를 활용**하여 코드 작성을 도와줍니다
3. **대화하듯이** 질문하고 답변을 받을 수 있습니다
4. **수백 개의 AI 모델**을 사용할 수 있습니다

## 왜 Kilo Code를 배워야 할까요?

### 💡 초보자에게 특히 좋은 이유

#### 1️⃣ 학습 속도가 빨라집니다

**기존 방법** (도구 없이 배우기):
```
1. 에러 발생
2. 구글 검색
3. Stack Overflow에서 비슷한 질문 찾기
4. 영어로 된 답변 읽기
5. 내 코드에 적용해보기
6. 또 에러 발생
7. 1번으로 돌아가기
⏰ 소요 시간: 30분~1시간
```

**Kilo Code 사용**:
```
1. 에러 발생
2. Kilo Code에게 물어보기
3. 한국어로 설명 듣기
4. 제안된 해결책 적용하기
⏰ 소요 시간: 5분
```

#### 2️⃣ 혼자서도 문제를 해결할 수 있습니다

**실제 이야기**:
```
민수는 코딩 부트캠프에 다니고 있습니다.
밤 11시, 과제를 하다가 막혔습니다.

선택지:
A) 내일 아침까지 기다려서 선생님께 질문하기
B) 친구에게 카톡으로 물어보기 (답장이 올까...?)
C) Kilo Code에게 즉시 물어보고 해결하기 ✅

민수는 C를 선택했고, 5분 만에 문제를 해결했습니다!
```

#### 3️⃣ 실수가 두렵지 않게 됩니다

**심리적 장벽 제거**:
```
사람에게 질문할 때:
😰 "이런 걸 물어봐도 될까?"
😓 "바보 같은 질문이면 어쩌지?"
😔 "이미 세 번이나 물어봤는데..."

AI에게 질문할 때:
😊 "100번 물어봐도 괜찮아!"
🤗 "어떤 질문이든 친절하게 답해줘!"
💪 "실수해도 부끄럽지 않아!"
```

## 준비물 확인하기

### 🛠️ 필요한 것들

#### 1. 컴퓨터 (Mac, Windows, Linux)
```
✅ Mac: macOS 10.15 이상
✅ Windows: Windows 10 이상
✅ Linux: Ubuntu 18.04 이상
```

#### 2. Node.js (노드 제이에스)
> **용어 설명**: JavaScript를 컴퓨터에서 실행할 수 있게 해주는 프로그램

**비유**:
- 게임을 하려면 게임기가 필요하듯이
- Node.js 프로그램을 실행하려면 Node.js가 필요합니다

**설치 확인하기**:
```bash
# 터미널을 열고 이 명령어를 입력하세요
node --version

# 이렇게 나오면 이미 설치되어 있습니다:
# v18.17.0 (숫자는 다를 수 있습니다)

# 이렇게 나오면 설치가 필요합니다:
# command not found: node
```

**없다면 설치하기**:
1. https://nodejs.org 접속
2. "LTS" 버전 다운로드 (추천)
3. 다운로드한 파일 실행
4. "다음", "동의", "설치" 버튼만 누르면 끝!

#### 3. 인터넷 연결
```
🌐 필수! AI와 대화하려면 인터넷이 필요합니다
📶 안정적인 Wi-Fi 권장
```

#### 4. 텍스트 에디터 (선택사항)
```
코드를 보고 수정하려면 필요합니다:
- VS Code (추천! 무료)
- Sublime Text
- Atom
- 메모장도 괜찮아요!
```

## Kilo Code 설치하기 단계별

### 🌱 초보자도 따라할 수 있는 상세 가이드

#### Step 1: 터미널 열기

**Mac 사용자**:
```
방법 1) Spotlight 사용
1. Command(⌘) + Space 누르기
2. "터미널" 입력
3. Enter 누르기

방법 2) 직접 찾기
1. Finder 열기
2. "응용 프로그램" → "유틸리티"
3. "터미널" 더블 클릭
```

**Windows 사용자**:
```
방법 1) 검색 사용
1. Windows 키 누르기
2. "cmd" 또는 "명령 프롬프트" 입력
3. Enter 누르기

방법 2) PowerShell 사용 (추천)
1. Windows 키 + X
2. "Windows PowerShell" 선택
```

**Linux 사용자**:
```
방법 1) 단축키
Ctrl + Alt + T

방법 2) 메뉴에서 찾기
"Terminal" 또는 "터미널" 검색
```

#### Step 2: 설치 명령어 입력

```bash
npm install -g @kilocode/cli
```

**명령어 분석** (무슨 뜻일까요?):
```
npm          → Node Package Manager (노드 프로그램 설치 도구)
install      → 설치하라는 뜻
-g           → 'global'의 약자, 컴퓨터 어디서든 사용 가능하게
@kilocode/cli → 설치할 프로그램 이름
```

**실제로 보이는 화면**:
```
$ npm install -g @kilocode/cli

downloading... ████████████████░░░░ 75%
```

#### Step 3: 기다리기 ⏰

설치 중에 이런 것들이 보일 거예요:
```
✓ 패키지 다운로드 중...
✓ 의존성 설치 중...
✓ 설정 파일 생성 중...
✓ 완료!
```

**소요 시간**:
- 빠른 인터넷: 1-2분
- 느린 인터넷: 3-5분

**⚠️ 만약 에러가 난다면?**
```
에러 메시지: "permission denied" 또는 "권한이 없습니다"

해결 방법:
Mac/Linux: sudo npm install -g @kilocode/cli
(비밀번호 입력 필요)

Windows: 관리자 권한으로 cmd 실행 후 다시 시도
```

## 터미널이란 무엇인가요?

### 🖥️ 쉬운 설명

#### 비유: 마법의 주문을 외우는 곳
```
옛날 옛적, 컴퓨터는 마우스가 없었습니다.
모든 것을 '글자로 명령'해야 했죠.

터미널은 그때의 방식입니다.
하지만 아직도 쓰는 이유가 있어요!

왜냐하면...
- 빠르고 정확하기 때문
- 자동화하기 쉽기 때문
- 개발자들이 선호하기 때문
```

#### 실생활 비유
```
마우스로 파일 10개 복사하기:
1. 마우스로 파일 선택
2. 드래그
3. 폴더로 이동
4. 붙여넣기
5. 1-4 반복 10번
⏰ 1분 소요

터미널로 파일 10개 복사하기:
cp file*.txt /destination/
⏰ 1초 소요!
```

### 📝 기본 터미널 명령어

#### 필수 명령어 5가지

**1. `pwd` (Print Working Directory)**
```bash
pwd
# 출력 예시: /Users/yourname/Documents
```
**뜻**: "지금 내가 어디에 있는지 보여줘"
**비유**: 내비게이션에서 "현재 위치" 보기

**2. `ls` (List)**
```bash
ls
# 출력 예시:
# file1.txt  file2.txt  폴더1  폴더2
```
**뜻**: "여기 있는 파일과 폴더를 보여줘"
**비유**: 서랍 열어서 안에 뭐가 있나 보기

**3. `cd` (Change Directory)**
```bash
cd Documents
# Documents 폴더로 이동
```
**뜻**: "이 폴더로 이동해"
**비유**: 방에서 거실로 이동하기

**4. `mkdir` (Make Directory)**
```bash
mkdir my-project
# my-project라는 폴더 생성
```
**뜻**: "새 폴더 만들어"
**비유**: 새 서랍 만들기

**5. `clear`**
```bash
clear
# 화면이 깨끗해집니다
```
**뜻**: "화면 지워"
**비유**: 칠판 지우기

## CLI가 뭔가요?

### 💬 Command Line Interface의 약자

#### 쉬운 설명
```
CLI = 명령줄 인터페이스

'인터페이스'란?
→ 사람과 컴퓨터가 소통하는 방법

두 가지 종류:
1. GUI (그래픽): 마우스로 클릭 (우리가 익숙한 방식)
2. CLI (글자): 키보드로 명령 타이핑
```

#### 비교 표

| 특징 | GUI (그래픽) | CLI (명령줄) |
|------|-------------|-------------|
| **예시** | 바탕화면, 아이콘 | 터미널, 검은 화면 |
| **사용 방법** | 마우스 클릭 | 키보드 타이핑 |
| **장점** | 직관적, 쉬움 | 빠름, 정확함 |
| **단점** | 느림, 반복작업 힘듦 | 배우기 어려움 |
| **비유** | 그림책 읽기 | 소설책 읽기 |

#### 왜 개발자들은 CLI를 좋아할까?
```
이유 1: 빠르다
마우스로 클릭 10번 vs 명령어 한 줄

이유 2: 정확하다
"여기를... 아니 저기를... 아 잘못 클릭했네" ❌
명령어는 항상 똑같이 실행됩니다 ✅

이유 3: 자동화할 수 있다
같은 작업을 100번 반복? 명령어 하나로 끝!

이유 4: 프로처럼 보인다 😎
(농담 아니에요! 개발자들 사이에서 인정받습니다)
```

## AI가 코드를 어떻게 도와주나요?

### 🤖 AI 코딩 도우미의 원리

#### 비유: 초능력 있는 친구
```
상황을 상상해보세요...

당신은 레고를 조립하고 있습니다.
옆에 레고 전문가 친구가 있습니다.

당신: "이 부품을 어디에 끼워야 하지?"
친구: "아! 그건 저기 빨간 블록 옆이야!"

당신: "이렇게 만들고 싶은데 어떻게 하지?"
```

## 코드 설명 받아보기

### 📖 이해하기 어려운 코드 해석하기

#### 실전 예제 1: JavaScript 함수 이해하기

**준비물**: 설명받고 싶은 코드 파일

**코드 예시** (app.js):
```javascript
function greet(name) {
    const timeOfDay = new Date().getHours();
    if (timeOfDay < 12) {
        return `Good morning, ${name}!`;
    } else if (timeOfDay < 18) {
        return `Good afternoon, ${name}!`;
    } else {
        return `Good evening, ${name}!`;
    }
}
```

**질문하기**:
```bash
kilocode ask "이 코드 설명해줘" --file app.js
```

**AI의 설명** (예상):
```
이 코드는 '시간대별 인사말' 함수예요!

📖 전체 흐름:
1. greet라는 함수 정의
2. 현재 시간 확인
3. 시간대에 따라 다른 인사말 반환

🔍 자세한 설명:

1줄: function greet(name) {
→ "greet"라는 함수를 만들어요
→ "name"을 입력받아요

2줄: const timeOfDay = new Date().getHours();
→ 현재 시각의 '시(hour)'를 가져와요
→ 예: 오전 9시면 9, 오후 3시면 15

3-5줄: if (timeOfDay < 12) { ... }
→ 시간이 12시 이전이면 (오전)
→ "Good morning, [이름]!" 반환

6-8줄: else if (timeOfDay < 18) { ... }
→ 시간이 12시~18시 사이면 (오후)
→ "Good afternoon, [이름]!" 반환

9-11줄: else { ... }
→ 그 외의 시간이면 (저녁)
→ "Good evening, [이름]!" 반환

💡 사용 예시:
greet("철수")
// 오전 10시라면: "Good morning, 철수!"
// 오후 3시라면: "Good afternoon, 철수!"
// 저녁 8시라면: "Good evening, 철수!"
```

#### 실전 예제 2: Python 리스트 컴프리헨션

**코드**:
```python
numbers = [1, 2, 3, 4, 5]
squared = [x**2 for x in numbers if x % 2 == 0]
print(squared)
```

**질문**:
```bash

```

**AI의 설명** (예상):
```
이 코드는 '짝수만 골라서 제곱하기'예요!

🎯 결과부터 보면:
[4, 16]

왜냐하면:
- 1은 홀수라서 제외
- 2는 짝수 → 2² = 4
- 3은 홀수라서 제외
- 4는 짝수 → 4² = 16
- 5는 홀수라서 제외

📝 한 줄씩 뜯어보기:

1줄: numbers = [1, 2, 3, 4, 5]
→ 숫자 리스트 만들기

2줄: squared = [x**2 for x in numbers if x % 2 == 0]
이것을 풀어서 설명하면...

[x**2          → x의 제곱
 for x         → 각각의 x에 대해
 in numbers    → numbers 리스트에서
 if x % 2 == 0]→ x가 짝수인 것만

일반 for문으로 쓰면:
squared = []
for x in numbers:
    if x % 2 == 0:  # 짝수면
        squared.append(x**2)  # 제곱해서 추가

리스트 컴프리헨션이 더 짧고 깔끔하죠?

3줄: print(squared)
→ 결과 출력: [4, 16]
```

## 용어 사전 (Glossary)

### 💡 초보자를 위한 쉬운 용어 설명

#### A

**API (Application Programming Interface)**
- **사전적 정의**: 프로그램 간 소통하는 규칙
- **쉬운 설명**: 식당의 메뉴판과 같아요!
  ```
  당신(손님) ← 메뉴판 → 주방

  메뉴판을 보고:
  "치킨 1개 주세요!" → 주방에서 만들어줌

  API를 사용해:
  "날씨 정보 주세요!" → 서버에서 보내줌
  ```

**AI (Artificial Intelligence, 인공지능)**
- **사전적 정의**: 사람처럼 생각하는 컴퓨터
- **쉬운 설명**: 매우 똑똑한 로봇 친구
  - 질문하면 답해줘요
  - 패턴을 학습해요
  - 사람처럼 대화할 수 있어요

#### C

**CLI (Command Line Interface)**
- **사전적 정의**: 명령줄 인터페이스
- **쉬운 설명**: 키보드로만 컴퓨터와 대화하는 방식
  ```
  GUI (그래픽): 🖱️ 마우스 클릭
  CLI (명령줄): ⌨️ 키보드 타이핑
  ```

**Code (코드)**
- **사전적 정의**: 컴퓨터가 이해하는 명령어
- **쉬운 설명**: 컴퓨터에게 보내는 '지시서'
  ```
  사람 세계: "문 좀 닫아줄래?"
  컴퓨터 세계: door.close()
  ```

**Compiler (컴파일러)**
- **사전적 정의**: 코드를 기계어로 번역하는 프로그램
- **쉬운 설명**: 통역사와 같아요!
  ```
  사람 말 → 통역사 → 외국어
  코드 → 컴파일러 → 기계어
  ```

#### D

**Debug (디버그)**
- **사전적 정의**: 버그(오류)를 찾아서 고치는 것
- **쉬운 설명**: 코드 탐정이 되는 것!
  ```
  1. 뭔가 이상해 (버그 발견)
  2. 어디가 문제지? (원인 찾기)
  3. 이렇게 고치면 돼! (수정)
  4. 잘 작동한다! (해결)
  ```

#### E

**Error (에러)**
- **사전적 정의**: 프로그램이 잘못 실행될 때 나는 오류
- **쉬운 설명**: 컴퓨터가 "이해 못 했어요!" 하는 것
  ```
  종류:
  - SyntaxError: 문법 틀림 (맞춤법 오류)
  - TypeError: 타입이 안 맞음 (사과에 나누기 하기)
  - NameError: 존재하지 않는 이름 (없는 변수 부르기)
  ```

#### F

**Function (함수)**
- **사전적 정의**: 특정 작업을 수행하는 코드 묶음
- **쉬운 설명**: 요리 레시피와 같아요!
  ```
  레시피: "계란말이 만들기"
  1. 계란 깨기
  2. 섞기
  3. 프라이팬에 굽기

  함수: makeOmelette()
  1. eggs.crack()
  2. eggs.mix()
  3. pan.cook()

  필요할 때마다 makeOmelette() 호출!
  ```

#### I

**IDE (Integrated Development Environment)**
- **사전적 정의**: 통합 개발 환경
- **쉬운 설명**: 개발자의 작업실
  ```
  VS Code, PyCharm 등

  포함된 것들:
  - 코드 에디터 (글 쓰는 곳)
  - 디버거 (오류 찾기 도구)
  - 터미널 (명령어 실행)
  - 기타 편리한 도구들
  ```

#### L

**LLM (Large Language Model)**
- **사전적 정의**: 대형 언어 모델
- **쉬운 설명**: 엄청나게 많은 글을 읽고 학습한 AI
  ```
  예시: ChatGPT, Claude, Gemini

  할 수 있는 것:
  - 대화하기
  - 글 쓰기
  - 코드 설명
  - 번역
  - 요약
  ```

**Loop (반복문)**
- **사전적 정의**: 같은 작업을 반복하는 코드
- **쉬운 설명**: "이거 10번 해줘!" 같은 것
  ```
  사람한테 시키기:
  "1부터 10까지 말해봐"
  → "하나, 둘, 셋..." (힘들다!)

  반복문 사용:
  for i in range(1, 11):
      print(i)
  → 컴퓨터가 자동으로 출력!
  ```

#### N

**Node.js (노드 제이에스)**
- **사전적 정의**: JavaScript 실행 환경
- **쉬운 설명**: JavaScript를 브라우저 밖에서도 쓸 수 있게 해주는 프로그램
  ```
  원래: JavaScript는 웹브라우저에서만 가능
  Node.js 설치 후: 내 컴퓨터 어디서든 가능!
  ```

**npm (Node Package Manager)**
- **사전적 정의**: 노드 패키지 관리자
- **쉬운 설명**: 프로그램 다운로드 하는 앱스토어
  ```
  스마트폰: 앱스토어에서 앱 다운로드
  개발자: npm에서 패키지 다운로드

  명령어: npm install [패키지명]
  ```

#### P

**Package (패키지)**
- **사전적 정의**: 재사용 가능한 코드 묶음
- **쉬운 설명**: 누군가 만들어둔 도구 상자
  ```
  직접 만들기: 100시간
  패키지 사용: 1분

  예시:
  - lodash: 편리한 함수들
  - axios: 웹 요청 보내기
  - express: 웹서버 만들기
  ```

**Parameter (매개변수)**
- **사전적 정의**: 함수에 전달되는 값
- **쉬운 설명**: 함수에게 주는 '재료'
  ```
  함수: 주스 만들기
  매개변수: 과일 종류

  makeJuice("사과")  → 사과주스
  makeJuice("바나나") → 바나나주스

  def makeJuice(fruit):  # fruit이 매개변수
      return f"{fruit} 주스"
  ```

#### S

**Syntax (문법)**
- **사전적 정의**: 프로그래밍 언어의 규칙
- **쉬운 설명**: 언어의 '맞춤법'
  ```
  한국어 문법: "나는 학교에 간다"
  틀린 문법: "나는 학교 간다에"

  Python 문법: print("Hello")
  틀린 문법: print "Hello"  (괄호 없음)
  ```

#### T

**Terminal (터미널)**
- **사전적 정의**: 명령어를 입력하는 프로그램
- **쉬운 설명**: 컴퓨터와 대화하는 '채팅창'
  ```
  검은 화면에
  하얀 글자로
  명령어를 입력하면
  컴퓨터가 실행해줍니다!
  ```

**Type (타입, 자료형)**
- **사전적 정의**: 데이터의 종류
- **쉬운 설명**: 값의 '종류'
  ```
  실생활:
  - 사과 (과일)
  - 자동차 (탈것)
  - 돈 (숫자)

  프로그래밍:
  - "Hello" (문자열, String)
  - 123 (숫자, Number)
  - True (참/거짓, Boolean)
  ```

#### V

**Variable (변수)**
- **사전적 정의**: 값을 저장하는 공간
- **쉬운 설명**: 이름표 붙은 상자
  ```
  age = 25

  'age'라는 상자에
  '25'라는 값을 담아요

  나중에:
  print(age)  → 25가 나와요!
  ```
