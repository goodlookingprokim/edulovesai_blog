## 주요 내용

### 라이브 코딩 목표
- SaaS 서비스의 핵심 기능을 실제로 구현
- 기능이 "말이 되게" 완성하는 것에 중점
- Cursor AI를 활용한 효율적인 개발 프로세스 시연

### 시청자 제보 서비스
- **서비스명**: WorkEasy (워크이지)
- **유사 서비스**: 워크잼(WorkJam)
- **주요 기능**:
  - 알바생 스케줄 자동 조정
  - 사내 채팅 기능
  - 근무 교대 시스템

## WorkEasy 프로젝트 소개

### 서비스 개요
- **목적**: 알바생들의 효율적인 근무 스케줄 관리
- **타겟**: 알바생이 많은 매장 (카페, 레스토랑, 편의점 등)
- **핵심 가치**: 근무 교대의 효율화

### 주요 기능 명세
1. **스케줄 관리**
   - 알바생별 근무 스케줄 조회
   - 매니저의 통합 스케줄 관리
   
2. **스케줄 변경 및 교환**
   - 알바생 간 근무 교대 요청
   - 자동 매칭 시스템
   
3. **계정 체계**
   - 매니저 계정: 스케줄 생성 및 관리 권한
   - 알바생 계정: 스케줄 조회 및 교대 요청 권한
   
4. **부가 기능**
   - 교육 지침 자료 공유
   - 사내 공지사항
   - PWA 설치 시 푸시 알림

### 사용자 플로우 설계
```
매니저 가입 → 매장 생성 → 알바생 계정 발급 → 
알바생 초대 → 스케줄 등록 → 근무 교대 요청/승인
```

## EasyNext 프레임워크 활용

### EasyNext란?
- 방송 진행자가 개발한 Next.js 기반 프레임워크
- 빠른 프로젝트 초기 설정 지원
- SaaS 개발에 최적화된 보일러플레이트

### 설치 및 설정

#### 1. 사전 요구사항
```bash
# Node.js 버전 확인 (v20 이상 필요)
node -v
```

#### 2. EasyNext 설치
```bash
# EasyNext CLI 설치
npm install -g easynext

# 또는 npx 사용
npx create-easynext@latest
```

#### 3. 프로젝트 생성
```bash
# 프로젝트 생성 명령어
easynext create workeasy

# 폴더 이동
cd workeasy
```

#### 4. 프로젝트 실행
```bash
# 개발 서버 실행
npm run dev
```

## 구현 방법

### Cursor 활용 팁

#### 1. 프로젝트 열기
- Cursor 실행 → Open Project → workeasy 폴더 선택

#### 2. 주요 패널 소개
- **파일 탐색기**: 좌측 패널에서 프로젝트 파일 관리
- **AI 채팅 패널**: 우측에서 AI에게 기능 요청
- **터미널**: Ctrl+J (Windows) 또는 Cmd+J (Mac)로 열기

#### 3. AI 활용 예시
```
"로그인 기능 만들어 줘"
"알바생 스케줄 관리 페이지 구현해 줘"
"근무 교대 요청 시스템 만들어 줘"
```

### 개발 환경 구성

#### 필수 도구
- Node.js (v20 이상)
- Cursor AI Editor
- Git (버전 관리용)

#### 터미널 명령어
```bash
# 프로젝트 실행
npm run dev

# 의존성 설치
npm install

# 빌드
npm run build
```
